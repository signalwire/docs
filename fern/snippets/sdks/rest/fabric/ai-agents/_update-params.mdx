<ParamField path="prompt" type="object" toc={true}>
  Defines the AI agent's personality, goals, behaviors, and instructions for handling conversations.
  The prompt establishes how the agent should interact with callers, what information it should gather,
  and how it should respond to various scenarios. It is recommended to write prompts using markdown formatting.
</ParamField>

<ParamField path="global_data" type="object" toc={true}>
  A key-value object for storing data that persists throughout the AI session.
  Can be set initially in the SWML script or modified during the conversation using the `set_global_data` action.
  The `global_data` object is accessible everywhere in the AI session: prompts, AI parameters,
  and SWML returned from SWAIG functions. Access properties using template strings (e.g., `${global_data.property_name}`).
</ParamField>

<ParamField path="hints" type="array" toc={true}>
  Hints help the AI agent understand certain words or phrases better. Words that can commonly be
  misinterpreted can be added to the hints to help the AI speak more accurately.
</ParamField>

<ParamField path="languages" type="array" toc={true}>
  An array of JSON objects defining supported languages in the conversation.
</ParamField>

<ParamField path="params" type="object" toc={true}>
  A JSON object containing parameters as key-value pairs (e.g., temperature, top_p).
</ParamField>

<ParamField path="post_prompt" type="object" toc={true}>
  The final set of instructions and configuration settings to send to the agent.
</ParamField>

<ParamField path="post_prompt_url" type="string" toc={true}>
  The URL to which to send status callbacks and reports.
  Authentication can also be set in the URL in the format of `username:password@url`.
</ParamField>

<ParamField path="pronounce" type="array" toc={true}>
  An array of JSON objects to clarify the AI's pronunciation of words or expressions.
</ParamField>

<ParamField path="SWAIG" type="object" toc={true}>
  SWAIG (SignalWire AI Gateway) function configuration. An array of JSON objects to create
  user-defined functions/endpoints that can be executed during the dialogue.
</ParamField>
