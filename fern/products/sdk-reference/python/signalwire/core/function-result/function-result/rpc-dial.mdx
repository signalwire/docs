# `rpc_dial`

Dial out to a number with a destination SWML URL using execute_rpc.

This is commonly used in call screening scenarios where you place a caller
on hold and dial out to a human, with the dest_swml specifying what agent
handles the outbound leg.

## Signature

```python
rpc_dial(
    to_number: str,
    from_number: str,
    dest_swml: str,
    device_type: str = 'phone'
) -> FunctionResult
```

## Parameters

<ParamField path="to_number" type="str" required>
  Phone number to dial (E.164 format)
</ParamField>
<ParamField path="from_number" type="str" required>
  Caller ID to use (E.164 format)
</ParamField>
<ParamField path="dest_swml" type="str" required>
  URL to the SWML that handles the outbound call
</ParamField>
<ParamField path="device_type" type="str" default="'phone'">
  Device type, typically "phone" (default: "phone")
</ParamField>

`FunctionResult` — self for method chaining
