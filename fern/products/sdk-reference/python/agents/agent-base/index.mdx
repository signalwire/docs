# `agent_base`

Base class for all SignalWire AI Agents.

This class extends SWMLService and provides enhanced functionality for building agents including:

- Prompt building and customization
- SWML rendering
- SWAIG function definition and execution
- Web service for serving SWML and handling webhooks
- Security and session management

Subclassing options:

1. Simple override of get\_prompt() for raw text
2. Using prompt\_\* methods for structured prompts
3. Declarative PROMPT\_SECTIONS class attribute

### add\_pre\_answer\_verb

Add a verb to run before the call is answered.

Pre-answer verbs execute while the call is still ringing. Only certain
verbs are safe to use before answering:

> \[!NOTE]
> cond, if, eval, set, unset, hangup, send\_sms, sleep,
> stop\_record\_call, stop\_denoise, stop\_tap

Verbs with auto\_answer option (play, connect): Must include
"auto\_answer": False in config to prevent automatic answering.

> \[!NOTE]
>
> ### Play ringback tone before answering
>
> agent.add\_pre\_answer\_verb("play", \{
> "urls": \["ring:us"],
> "auto\_answer": False
> \})

### add\_swaig\_query\_params

This is particularly useful for preserving dynamic configuration state
across SWAIG callbacks. For example, if your dynamic config adds skills

### on\_debug\_event

Register a handler for debug webhook events.

Use as a decorator to receive real-time debug events from the AI module
during calls. Requires enable\_debug\_events() to be called first.

> \[!NOTE]
> event\_type (str): The event label (e.g. "barge", "llm\_error", "session\_start")
> data (dict): The full event payload including call\_id, label, and event-specific fields

The handler may be sync or async.

> \[!NOTE]
> @agent.on\_debug\_event
> def handle(event\_type, data):
> if event\_type == "barge":
> print(f"Barge detected: \{data.get('barge\_elapsed\_ms')\}ms")

#### Signature

```python
on_debug_event(handler: Callable) -> Callable
```

#### Parameters

<ParamField path="handler" type="Callable" required>
  Callback function with signature (event\_type: str, data: dict)
</ParamField>

#### Returns

`Callable` — The handler function (unchanged), for use as a decorator
