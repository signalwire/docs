### preprocess\_query

Advanced query preprocessing with language detection, POS tagging, synonym expansion, and vectorization

#### Signature

```python
preprocess_query(
    query: str,
    language: str = 'en',
    pos_to_expand: Optional[List[str]] = None,
    max_synonyms: int = 5,
    debug: bool = False,
    vector: bool = False,
    vectorize_query_param: bool = False,
    nlp_backend: str = None,
    query_nlp_backend: str = 'nltk',
    model_name: str = None,
    preserve_original: bool = True
) -> Dict[str, Any]
```

#### Parameters

<ParamField path="query" type="str" required>
  Input query string
</ParamField>

<ParamField path="language" type="str" default="'en'">
  Language code ('en', 'es', etc.) or 'auto' for detection
</ParamField>

<ParamField path="pos_to_expand" type="Optional[List[str]]" default="None">
  List of POS tags to expand with synonyms
</ParamField>

<ParamField path="max_synonyms" type="int" default="5">
  Maximum synonyms per word
</ParamField>

<ParamField path="debug" type="bool" default="False">
  Enable debug output
</ParamField>

<ParamField path="vector" type="bool" default="False">
  Include vector embedding in output
</ParamField>

<ParamField path="vectorize_query_param" type="bool" default="False">
  If True, just vectorize without other processing
</ParamField>

<ParamField path="nlp_backend" type="str" default="None">
  DEPRECATED - use query\_nlp\_backend instead
</ParamField>

<ParamField path="query_nlp_backend" type="str" default="'nltk'">
  NLP backend for query processing ('nltk' for fast, 'spacy' for better quality)
</ParamField>

<ParamField path="model_name" type="str" default="None" />

<ParamField path="preserve_original" type="bool" default="True" />

#### Returns

`Dict[str, Any]` — Dict containing processed query, language, POS tags, and optionally vector
