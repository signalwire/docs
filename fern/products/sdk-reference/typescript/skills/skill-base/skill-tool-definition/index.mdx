# `SkillToolDefinition`

Definition of a SWAIG tool provided by a skill.

## Signature

```typescript
interface SkillToolDefinition
```

## Properties

<ParamField path="description" type="string" required>
  Human-readable description of what the tool does, shown to the AI.
</ParamField>

<ParamField path="fillers" type="Record<string, string[]>">
  Filler phrases spoken while the tool executes, keyed by language.
</ParamField>

<ParamField path="handler" type="SwaigHandler" required>
  Handler function invoked when the tool is called.
</ParamField>

<ParamField path="isHangupHook" type="boolean">
  When true, the SignalWire platform automatically invokes this tool when the call ends (hangup), regardless of whether the AI explicitly calls it. Equivalent to Python's `is_hangup_hook=True` in `define_tool()`. The flag is serialised as `"is_hangup_hook": true` in the SWAIG JSON.
</ParamField>

<ParamField path="name" type="string" required>
  Unique tool name used in SWAIG function registration.
</ParamField>

<ParamField path="parameters" type="Record<string, unknown>">
  Parameter schema for the tool, keyed by parameter name.
</ParamField>

<ParamField path="required" type="string[]">
  List of parameter names that are required.
</ParamField>

<ParamField path="secure" type="boolean">
  Whether the tool requires secure (authenticated) invocation.
</ParamField>

<ParamField path="skip_fillers" type="boolean">
  If true, suppress filler phrases while the tool executes.
</ParamField>

<ParamField path="wait_for_fillers" type="boolean">
  If true, wait for any in-flight fillers to complete before executing the tool.
</ParamField>
