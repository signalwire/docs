# `SwaigFunctionOptions`

Configuration options for creating a SwaigFunction.

## Signature

```typescript
interface SwaigFunctionOptions
```

## Properties

<ParamField path="description" type="string" required>
  Human-readable description of what the tool does, shown to the AI.
</ParamField>

<ParamField path="extraFields" type="Record<string, unknown>">
  Additional fields to merge directly into the SWAIG function definition. **Python equivalent:** `**extra_swaig_fields` kwargs on the constructor. In Python these are passed as bare keyword arguments and merged directly into the output dict via `function_def.update(self.extra_swaig_fields)`. In TypeScript the same fields are collected under this single options key and merged identically in `toSwaig()` — the wire format is identical.
</ParamField>

<ParamField path="fillers" type="Record<string, string[]>">
  Language-keyed filler phrases spoken while the tool executes.
</ParamField>

<ParamField path="handler" type="SwaigHandler" required>

</ParamField>

<ParamField path="isTypedHandler" type="boolean">
