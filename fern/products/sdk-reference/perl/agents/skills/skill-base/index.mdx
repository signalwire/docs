# `SkillBase`

SignalWire::Skills::SkillBase is the Perl port of the Python reference's `SkillBase`. Every built-in or custom skill extends it. A skill declares metadata (`skill_name`, `skill_description`, `skill_version`), optional package / environment requirements, and overrides the abstract `setup` and `register_tools` hooks to wire its SWAIG tools onto the owning agent.

Construction fails fast (Moo `isa` constraints): `skill_name` / `skill_description` must be non-empty strings, `agent` must be a blessed object, `required_packages` / `required_env_vars` must be arrayrefs, and `params` / `swaig_fields` must be hashrefs.

**ATTRIBUTES**

`skill_name`, `skill_description`, `skill_version`, `supports_multiple_instances`, `required_packages`, `required_env_vars`, `agent`, `params`, `swaig_fields`.

**METHODS**

- `setup` / `register_tools` â€” abstract; a subclass **must** override both (the defaults `croak`).

- `define_tool(%opts)` â€” register a SWAIG tool on the agent, merging the skill's `swaig_fields` into the definition.

- `get_hints` / `get_global_data` / `get_prompt_sections` â€” default no-op contributions a subclass may override; `get_prompt_sections` honours the `skip_prompt` param.

- `cleanup` â€” teardown hook (no-op by default).

- `validate_env_vars` â€” true iff every `required_env_vars` entry is set in `%ENV`.

- `get_parameter_schema` â€” the JSON-schema-ish description of the config params the skill accepts.

- `get_instance_key` â€” the registry key for this skill instance (`skill_name`, optionally suffixed by `tool_name`).

**Synopsis**

package SignalWire::Skills::Builtin::MySkill;

```

sub setup          ($self) { ...; return 1; }
sub register_tools ($self) {
    $self->define_tool( name => 'do_thing', ... );
}
```
