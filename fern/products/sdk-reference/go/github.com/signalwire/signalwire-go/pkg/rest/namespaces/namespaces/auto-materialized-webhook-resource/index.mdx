# `AutoMaterializedWebhookResource`

AutoMaterializedWebhookResource is a Fabric webhook resource that is normally auto-created by the phone\_numbers.Set\*Webhook helpers. Exposed for backwards compatibility: list/get/update/delete work as usual, but Create now emits a deprecation warning because creating a webhook resource directly produces an orphan that isn't bound to any phone number.

### Create

<Warning>
  **Deprecated.** Creating a webhook Fabric resource directly produces an orphan not bound to any phone number. Use phone\_numbers.SetSwmlWebhook or phone\_numbers.SetCxmlWebhook instead; setting call\_handler on the phone number causes the server to auto-materialize the webhook resource. See porting-sdk's phone-binding.md.
</Warning>

Create sends a POST to create a new webhook resource.
