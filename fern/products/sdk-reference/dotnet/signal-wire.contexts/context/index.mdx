  <Card title="SetIsolated(bool)" href="/reference/dotnet/signal-wire.contexts/context/set-isolated-bool">
    Mark this context as isolated — entering it wipes conversation history. &lt;p>When &lt;code class="paramref">isolated&lt;/code> = true and the context is entered via change_context, the runtime wipes the conversation array. The model starts fresh with only the new context's system_prompt + step instructions, with no memory of prior turns.&lt;/p> &lt;p>&lt;b>EXCEPTION — reset overrides the wipe:&lt;/b> If the context also has a reset configuration (via &lt;xref href="SignalWire.Contexts.Context.SetConsolidate(System.Boolean)" data-throw-if-not-resolved="false">&lt;/xref> or &lt;xref href="SignalWire.Contexts.Context.SetFullReset(System.Boolean)" data-throw-if-not-resolved="false">&lt;/xref>), the wipe is skipped in favor of the reset behavior. Use reset with consolidate=true to summarize prior history into a single message instead of dropping it entirely.&lt;/p> &lt;p>Use cases: switching to a sensitive billing flow that should not see prior small-talk; handing off to a different agent persona; resetting after a long off-topic detour.&lt;/p>
