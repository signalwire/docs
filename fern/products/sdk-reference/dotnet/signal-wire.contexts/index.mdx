  <Card title="ContextBuilder" href="/docs/sdk-reference/reference/dotnet/signal-wire.contexts/context-builder">
    Builder for multi-step, multi-context AI agent workflows. &lt;p>A ContextBuilder owns one or more &lt;xref href="SignalWire.Contexts.Context" data-throw-if-not-resolved="false">&lt;/xref>s; each context owns an ordered list of &lt;xref href="SignalWire.Contexts.Step" data-throw-if-not-resolved="false">&lt;/xref>s. Only one context and one step is active at a time. Per chat turn, the runtime injects the current step's instructions as a system message, then asks the LLM for a response.&lt;/p> &lt;p>&lt;b>Native tools auto-injected by the runtime:&lt;/b> When a step (or its enclosing context) declares valid\_steps or valid\_contexts, the runtime auto-injects two native tools so the model can navigate the flow: &lt;code>next\_step&lt;/code> and &lt;code>change\_context&lt;/code>. A third native tool, &lt;code>gather\_submit&lt;/code>, is injected during gather\_info questioning. These three names are reserved: &lt;xref href="SignalWire.Contexts.ContextBuilder.Validate" data-throw-if-not-resolved="false">&lt;/xref> rejects any agent that defines a SWAIG tool with one of them. See &lt;xref href="SignalWire.Contexts.ReservedToolNames.Reserved" data-throw-if-not-resolved="false">&lt;/xref>.&lt;/p> &lt;p>&lt;b>Function whitelisting (&lt;xref href="SignalWire.Contexts.Step.SetFunctions(System.Object)" data-throw-if-not-resolved="false">&lt;/xref>):&lt;/b> Each step may declare a functions whitelist. The whitelist is applied in-memory at the start of each LLM turn. CRITICALLY: if a step does NOT declare a functions field, it INHERITS the previous step's active set. See &lt;xref href="SignalWire.Contexts.Step.SetFunctions(System.Object)" data-throw-if-not-resolved="false">&lt;/xref> for details and examples.&lt;/p>

  <Card title="ReservedToolNames" href="/docs/sdk-reference/reference/dotnet/signal-wire.contexts/reserved-tool-names">
    Reserved tool names auto-injected by the runtime when contexts/steps are present. User-defined SWAIG tools must not collide with these names. &lt;ul>&lt;li>&lt;code>next\_step&lt;/code> / &lt;code>change\_context&lt;/code> are injected when valid\_steps or valid\_contexts is set so the model can navigate the flow.&lt;/li>&lt;li>&lt;code>gather\_submit&lt;/code> is injected while a step's gather\_info is collecting answers.&lt;/li>&lt;/ul> &lt;xref href="SignalWire.Contexts.ContextBuilder.Validate" data-throw-if-not-resolved="false">&lt;/xref> rejects any agent that registers a user tool sharing one of these names — the runtime would never call the user tool because the native one wins.
