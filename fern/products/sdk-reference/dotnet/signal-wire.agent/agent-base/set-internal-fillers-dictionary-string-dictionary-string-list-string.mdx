# `SetInternalFillers(Dictionary<string, Dictionary<string, List<string>>>)`

Set internal fillers for native SWAIG functions.

&lt;p>Internal fillers are short phrases the AI agent speaks (via
TTS) while an internal/native function is running, so the caller
doesn't hear dead air during transitions or background work.&lt;/p>

&lt;p>Supported function names (match the SWAIGInternalFiller
schema): &lt;code>hangup&lt;/code>, &lt;code>check_time&lt;/code>, &lt;code>wait_for_user&lt;/code>,
&lt;code>wait_seconds&lt;/code>, &lt;code>adjust_response_latency&lt;/code>,
&lt;code>next_step&lt;/code>, &lt;code>change_context&lt;/code>, &lt;code>get_visual_input&lt;/code>,
&lt;code>get_ideal_strategy&lt;/code>. See
&lt;xref href="SignalWire.Agent.AgentBase.SupportedInternalFillerNames" data-throw-if-not-resolved="false">&lt;/xref>.&lt;/p>

&lt;p>Notably NOT supported: &lt;code>change_step&lt;/code>,
&lt;code>gather_submit&lt;/code>, or arbitrary user-defined SWAIG function
names. The runtime only honors fillers for the names listed above;
everything else is silently ignored at the SWML level. This method
warns at registration time if you pass an unknown name so you
catch the typo early.&lt;/p>
