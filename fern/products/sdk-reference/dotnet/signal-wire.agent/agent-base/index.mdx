# `AgentBase`

AI agent built on &lt;xref href="SignalWire.SWML.Service" data-throw-if-not-resolved="false">&lt;/xref>. Provides prompt management, SWAIG tool dispatch,
context switching, skill stubs, and a 5-phase SWML rendering pipeline.
All configuration methods return &lt;code>this&lt;/code> for fluent chaining.

public class AgentBase : Service

### HandleRequest(string, string, Dictionary\<string, string>, string?)

Override the base dispatch to enforce webhook signature validation on
POST requests targeting the signed routes (&lt;code>/&lt;/code>, &lt;code>/swaig&lt;/code>, &lt;code>/post\_prompt&lt;/code>) when &lt;xref href="SignalWire.Agent.AgentBase.SigningKey" data-throw-if-not-resolved="false">&lt;/xref> is configured.

&lt;p>Validation is gated behind Basic Auth: callers must already
satisfy the SWMLService basic-auth check (it always runs first in
&lt;xref href="SignalWire.SWML.Service.HandleRequest(System.String%2cSystem.String%2cSystem.Collections.Generic.Dictionary%7bSystem.String%2cSystem.String%7d%2cSystem.String)" data-throw-if-not-resolved="false">&lt;/xref>) before we even look at
signatures, matching Python where &lt;code>signing_key&lt;/code> is layered on
top of &lt;code>basic_auth&lt;/code>.&lt;/p>

&lt;p>On invalid signature: returns 403 directly without dispatching
to the agent's POST handler. On valid signature (or non-POST, or
non-signed route): delegates to &lt;xref href="SignalWire.SWML.Service.HandleRequest(System.String%2cSystem.String%2cSystem.Collections.Generic.Dictionary%7bSystem.String%2cSystem.String%7d%2cSystem.String)" data-throw-if-not-resolved="false">&lt;/xref>.
&lt;/p>

&lt;p>(Python parity: &lt;code>web_mixin._register_routes&lt;/code> wraps the
signed POST routes in a FastAPI &lt;code>Depends(sig_dep)&lt;/code> dependency
when &lt;code>signing_key&lt;/code> is set; this is the .NET equivalent.)&lt;/p>

**Modifiers:** `override`

### SetInternalFillers

Set internal fillers for native SWAIG functions.

&lt;p>Internal fillers are short phrases the AI agent speaks (via
TTS) while an internal/native function is running, so the caller
doesn't hear dead air during transitions or background work.&lt;/p>

&lt;p>Supported function names (match the SWAIGInternalFiller
schema): &lt;code>hangup&lt;/code>, &lt;code>check_time&lt;/code>, &lt;code>wait_for_user&lt;/code>,
&lt;code>wait_seconds&lt;/code>, &lt;code>adjust_response_latency&lt;/code>,

&lt;code>gather_submit&lt;/code>, or arbitrary user-defined SWAIG function
