# `SignalWire.REST`

## Classes

<CardGroup cols={2}>
  <Card title="CrudResource" href="/docs/sdk-reference/reference/dotnet/signal-wire.rest/crud-resource">
    Generic CRUD wrapper around an &lt;xref href="SignalWire.REST.HttpClient" data-throw-if-not-resolved="false">&lt;/xref> and a base API path. Provides List / Create / Get / Update / Delete for any REST resource that follows the standard SignalWire collection+item URL pattern.
  </Card>

  <Card title="CrudWithAddresses" href="/docs/sdk-reference/reference/dotnet/signal-wire.rest/crud-with-addresses" />

  <Card title="HttpClient" href="/docs/sdk-reference/reference/dotnet/signal-wire.rest/http-client">
    Low-level HTTP client for SignalWire REST APIs. Uses &lt;xref href="System.Net.Http.HttpClient" data-throw-if-not-resolved="false">&lt;/xref> with Basic Auth, and returns parsed JSON responses as dictionaries.
  </Card>

  <Card title="PaginatedIterator" href="/docs/sdk-reference/reference/dotnet/signal-wire.rest/paginated-iterator">
    Walks paged HTTP responses by following `links.next` cursors. Mirrors Python `signalwire.rest._pagination.PaginatedIterator` — constructor records inputs without fetching; iteration triggers the first fetch and continues until a page is returned without a `links.next` cursor.
  </Card>

  <Card title="RestClient" href="/docs/sdk-reference/reference/dotnet/signal-wire.rest/rest-client">
    Top-level SignalWire REST client. Provides lazy access to every API namespace (fabric, calling, phone\_numbers, datasphere, video, compat, etc.). Credentials can be supplied explicitly or pulled from environment variables.
  </Card>

  <Card title="SignalWireRestError" href="/docs/sdk-reference/reference/dotnet/signal-wire.rest/signal-wire-rest-error">
    Exception thrown when a SignalWire REST API call returns a non-2xx status or encounters a transport-level error.
