# `Purpose(string)`

Set the LLM-facing tool description (the "purpose"). PROMPT
ENGINEERING, not developer documentation.

&lt;p>The description string is rendered into the OpenAI tool
schema &lt;code>description&lt;/code> field on every LLM turn. The model
reads it to decide WHEN to call this tool. A vague
&lt;xref href="SignalWire.DataMap.DataMap.Purpose(System.String)" data-throw-if-not-resolved="false">&lt;/xref> is the #1 cause of "the model has the
right tool but doesn't call it" failures with data-map tools.&lt;/p>

&lt;p>&lt;b>Bad vs good:&lt;/b>&lt;/p>
&lt;pre>&lt;code class="lang-csharp">BAD : .Purpose("weather api")
GOOD: .Purpose("Get the current weather conditions and "
             + "forecast for a specific city. Use this "
             + "whenever the user asks about weather, "
             + "temperature, rain, or similar conditions in a "
             + "named location.")&lt;/code>&lt;/pre>
