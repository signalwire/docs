# `ValidateWebhookSignature(string, string?, string, string?)`

Validate a SignalWire webhook signature against both schemes.

**Throws:**
- System.ArgumentException: Thrown when &lt;code class="paramref">signingKey&lt;/code> is null or empty.

## Signature

```dotnet
public static bool ValidateWebhookSignature(string signingKey, string? signature, string url, string? rawBody)
```

## Parameters

<ParamField path="signingKey" type="string" required>
  Customer's Signing Key from the Dashboard. UTF-8 string, secret. Empty
or null raises &lt;xref href="System.ArgumentException" data-throw-if-not-resolved="false">&lt;/xref> — that's a programming
error, not a validation failure.
</ParamField>
<ParamField path="signature" type="string" required>
  The &lt;code>X-SignalWire-Signature&lt;/code> header value (or &lt;code>X-Twilio-Signature&lt;/code>
for cXML compat). Missing / empty returns &lt;code>false&lt;/code> without throwing.
</ParamField>
<ParamField path="url" type="string" required>
  The full URL SignalWire POSTed to (scheme, host, optional port, path,
query). Must match what the platform saw — see the
&lt;code>URL reconstruction&lt;/code> section of &lt;code>porting-sdk/webhooks.md&lt;/code>.
</ParamField>
<ParamField path="rawBody" type="string" required>
  The raw request body bytes as a UTF-8 string, BEFORE any JSON / form
parsing. Re-serialization breaks the Scheme A digest.
</ParamField>

## Returns

`bool` — &lt;code>true&lt;/code> if the signature matches either Scheme A (hex JSON) or
Scheme B (base64 form, with port-normalization variants and optional
bodySHA256 fallback). &lt;code>false&lt;/code> otherwise.
