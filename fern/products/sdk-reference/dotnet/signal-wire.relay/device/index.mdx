# `Device`

A typed RELAY device object — the &lt;code>\{type, params\}&lt;/code> shape that recurs
across &lt;code>connect&lt;/code> / &lt;code>refer&lt;/code> / &lt;code>dial&lt;/code> / &lt;code>tap&lt;/code> and the
ringback list.

**Remarks:** &lt;p>
Grounded in the RELAY wire schema &lt;code>relay-protocol/calling.connect.params.json&lt;/code>, where each device is an
object with a required &lt;code>type&lt;/code> (string) and a free-form &lt;code>params&lt;/code>
payload. This class types the &lt;em>shape&lt;/em> only: &lt;xref href="SignalWire.Relay.Device.Type" data-throw-if-not-resolved="false">&lt;/xref> stays
a &lt;code>string&lt;/code> because the discriminant set (&lt;code>phone&lt;/code>, &lt;code>sip&lt;/code>, &lt;code>webrtc&lt;/code>, …) is &lt;strong>not&lt;/strong> enumerated in any schema, so an
enum would risk rejecting a valid value.

&lt;/p>
&lt;p>
The reference and the rest of this port pass devices as raw
&lt;code>Dictionary&lt;string, object?&gt;&lt;/code> (e.g. &lt;xref href="SignalWire.Relay.Call.Device" data-throw-if-not-resolved="false">&lt;/xref>);
this is an additive typed convenience. &lt;xref href="SignalWire.Relay.Device.ToDict" data-throw-if-not-resolved="false">&lt;/xref> yields the
identical wire dictionary, so a &lt;xref href="SignalWire.Relay.Device" data-throw-if-not-resolved="false">&lt;/xref> can be used anywhere a
hand-built device dict is accepted with no change in emitted bytes.
&lt;/p>
&lt;example>
&lt;pre>&lt;code class="lang-csharp">var d = new Device("phone", new Dictionary&lt;string, object?&gt;
&#123;
    ["to_number"] = "+15551112222",
    ["from_number"] = "+15553334444",
&#125;);
// d.ToDict() is byte-identical to the hand-written
// &#123; ["type"] = "phone", ["params"] = &#123; ... &#125; &#125;&lt;/code>&lt;/pre>
&lt;/example>
