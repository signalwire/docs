### TryParse(string?, out CallState)

`false` (and `default`) for an unrecognised value — the
server may introduce new states, so callers fall back to the raw
string on &lt;xref href="SignalWire.Relay.Call.State" data-throw-if-not-resolved="false">&lt;/xref> rather than crashing.
