## Enums

### CallState

Lifecycle state of a RELAY &lt;xref href="SignalWire.Relay.Call" data-throw-if-not-resolved="false">&lt;/xref>, as a typed, compile-time
closed set.

**Remarks:** &lt;p>
The RELAY &lt;code>calling.call.state&lt;/code> event carries one of a known set of
call-state strings (&lt;code>created&lt;/code>, &lt;code>ringing&lt;/code>, &lt;code>answered&lt;/code>, &lt;code>ending&lt;/code>, &lt;code>ended&lt;/code>) — grounded in the Python reference's &lt;code>signalwire/relay/constants.py&lt;/code> (&lt;code>CALL\_STATES&lt;/code>) and mirrored by &lt;xref href="SignalWire.Relay.Constants.CallStateCreated" data-throw-if-not-resolved="false">&lt;/xref> .. &lt;xref href="SignalWire.Relay.Constants.CallStateEnded" data-throw-if-not-resolved="false">&lt;/xref>.
The reference exposes the state as a bare &lt;code>str&lt;/code>; this enum is a typed
alias over those strings so callers get autocompletion and a compile error
on a typo, while &lt;xref href="SignalWire.Relay.Call.State" data-throw-if-not-resolved="false">&lt;/xref> stays a string for parity and
forward-compatibility.

&lt;/p>
&lt;p>
This is the &lt;em>call&lt;/em> vocabulary and is deliberately
&lt;strong>distinct&lt;/strong> from &lt;xref href="SignalWire.Relay.DialState" data-throw-if-not-resolved="false">&lt;/xref> (the dial-outcome
vocabulary) and &lt;xref href="SignalWire.Relay.MessageState" data-throw-if-not-resolved="false">&lt;/xref> (the messaging-delivery
vocabulary). The three are never conflated — they index different wire
fields (&lt;code>call_state&lt;/code> vs &lt;code>dial_state&lt;/code> vs &lt;code>message_state&lt;/code>).
&lt;/p>
&lt;p>
Because the server emits these values and the set can grow, parse via
&lt;xref href="SignalWire.Relay.CallStateExtensions.TryParse(System.String%2cSignalWire.Relay.CallState%40)" data-throw-if-not-resolved="false">&lt;/xref>, which
returns &lt;code>false&lt;/code> for an unknown value rather than throwing — the string
arm on &lt;xref href="SignalWire.Relay.Call.State" data-throw-if-not-resolved="false">&lt;/xref> preserves any future server value.
&lt;/p>

### DialState

Outcome state of a RELAY dial attempt (&lt;code>calling.call.dial&lt;/code>), as a
typed, compile-time closed set.

**Remarks:** &lt;p>
The RELAY &lt;code>calling.call.dial&lt;/code> event carries a &lt;code>dial\_state&lt;/code> of &lt;code>dialing&lt;/code>, &lt;code>answered&lt;/code>, or &lt;code>failed&lt;/code> (see &lt;xref href="SignalWire.Relay.Client.HandleDialEvent(SignalWire.Relay.Event%2cSystem.Collections.Generic.Dictionary%7bSystem.String%2cSystem.Object%7d)" data-throw-if-not-resolved="false">&lt;/xref> and &lt;xref href="SignalWire.Relay.Constants.DialStateDialing" data-throw-if-not-resolved="false">&lt;/xref> .. &lt;xref href="SignalWire.Relay.Constants.DialStateFailed" data-throw-if-not-resolved="false">&lt;/xref>). It is the dial-&lt;em>outcome&lt;/em>
vocabulary and is deliberately &lt;strong>distinct&lt;/strong> from &lt;xref href="SignalWire.Relay.CallState" data-throw-if-not-resolved="false">&lt;/xref> (the per-call lifecycle) — a dial that resolves &lt;code>answered&lt;/code> transitions the winning &lt;xref href="SignalWire.Relay.Call" data-throw-if-not-resolved="false">&lt;/xref> to &lt;xref href="SignalWire.Relay.CallState.Answered" data-throw-if-not-resolved="false">&lt;/xref>, but the two fields are separate on the
wire.

&lt;/p>
&lt;p>
Both &lt;code>answered&lt;/code> and &lt;code>failed&lt;/code> are terminal dial outcomes (the dial
is resolved); &lt;code>dialing&lt;/code> is in-progress. Parse via
&lt;xref href="SignalWire.Relay.DialStateExtensions.TryParse(System.String%2cSignalWire.Relay.DialState%40)" data-throw-if-not-resolved="false">&lt;/xref>, which
tolerates an unknown server value.
&lt;/p>

### MessageState

Delivery state of a RELAY &lt;xref href="SignalWire.Relay.Message" data-throw-if-not-resolved="false">&lt;/xref> (SMS / MMS), as a typed,
compile-time closed set.

**Remarks:** &lt;p>
The RELAY &lt;code>messaging.state&lt;/code> event carries a &lt;code>message\_state&lt;/code> of &lt;code>queued&lt;/code>, &lt;code>initiated&lt;/code>, &lt;code>sent&lt;/code>, &lt;code>delivered&lt;/code>, &lt;code>undelivered&lt;/code>, or &lt;code>failed&lt;/code> — grounded in the Python reference's &lt;code>signalwire/relay/constants.py&lt;/code> (&lt;code>MESSAGE\_STATE\_\*&lt;/code>) and the wire
schema &lt;code>relay-protocol/messaging.state.event.json&lt;/code>; &lt;code>received&lt;/code> is
the additional inbound state (&lt;xref href="SignalWire.Relay.Constants.MessageStateReceived" data-throw-if-not-resolved="false">&lt;/xref>).
The reference exposes the state as a bare &lt;code>str&lt;/code>; this enum is a typed
alias so callers get autocompletion and a compile-time check, while &lt;xref href="SignalWire.Relay.Message.State" data-throw-if-not-resolved="false">&lt;/xref> stays a string for parity and forward-compat.

&lt;/p>
&lt;p>
This is the &lt;em>messaging&lt;/em> vocabulary and is deliberately
&lt;strong>distinct&lt;/strong> from &lt;xref href="SignalWire.Relay.CallState" data-throw-if-not-resolved="false">&lt;/xref> and
&lt;xref href="SignalWire.Relay.DialState" data-throw-if-not-resolved="false">&lt;/xref> (the voice vocabularies) — it indexes the
&lt;code>message_state&lt;/code> wire field, never &lt;code>call_state&lt;/code>/&lt;code>dial_state&lt;/code>.
&lt;/p>
&lt;p>
Parse via &lt;xref href="SignalWire.Relay.MessageStateExtensions.TryParse(System.String%2cSignalWire.Relay.MessageState%40)" data-throw-if-not-resolved="false">&lt;/xref>,
which tolerates an unknown server value.
