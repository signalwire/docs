# `Service`

A SWML service that manages a Document, provides schema-driven verb methods,
handles HTTP requests with Basic authentication, and supports routing callbacks.

### Run()

Start a blocking HTTP(S) server bound to &lt;xref href="SignalWire.SWML.Service.Host" data-throw-if-not-resolved="false">&lt;/xref>:&lt;xref href="SignalWire.SWML.Service.Port" data-throw-if-not-resolved="false">&lt;/xref>.
Each incoming request is dispatched through &lt;xref href="SignalWire.SWML.Service.HandleRequest(System.String%2cSystem.String%2cSystem.Collections.Generic.Dictionary%7bSystem.String%2cSystem.String%7d%2cSystem.String)" data-throw-if-not-resolved="false">&lt;/xref>;
the response status / headers / body are written back to the client.

Mirrors Python's SWMLService.run() — examples and the porting-sdk
audit harness call this directly.

Transport selection mirrors Python's &lt;code>SecurityConfig&lt;/code> /
uvicorn &lt;code>ssl\_certfile&lt;/code>/&lt;code>ssl\_keyfile&lt;/code> path:

&lt;ul>&lt;li>Plain HTTP uses System.Net.HttpListener (BCL, no extra deps).&lt;/li>&lt;li>HTTPS (when &lt;code>SWML_SSL_ENABLED&lt;/code> is truthy and
  &lt;code>SWML_SSL_CERT_PATH&lt;/code>/&lt;code>SWML_SSL_KEY_PATH&lt;/code> point at a valid
  PEM cert+key) uses Kestrel, because HttpListener cannot terminate TLS
  on Linux (cert binding requires http.sys / netsh, Windows-only).&lt;/li>&lt;/ul>

Server stops on Ctrl-C or when the process is killed.

**Modifiers:** `virtual`
