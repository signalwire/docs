# `ParameterSchema`

Fluent, type-safe builder for a SWAIG tool's &lt;code>parameters&lt;/code> — the
JSON-Schema &lt;code>properties&lt;/code> map passed as the third argument to &lt;xref href="SignalWire.SWML.Service.DefineTool(System.String%2cSystem.String%2cSystem.Collections.Generic.Dictionary%7bSystem.String%2cSystem.Object%7d%2cSystem.Func%7bSystem.Collections.Generic.Dictionary%7bSystem.String%2cSystem.Object%7d%2cSystem.Collections.Generic.Dictionary%7bSystem.String%2cSystem.Object%7d%2cSignalWire.SWAIG.FunctionResult%7d%2cSystem.Boolean)" data-throw-if-not-resolved="false">&lt;/xref> /
SignalWire.Agent.AgentBase.DefineTool.

**Remarks:** &lt;p>
Defining a SWAIG tool's parameters in the Python reference (and, until
now, in this port) means hand-writing an untyped &lt;code>Dictionary\<string, object>&lt;/code> of nested dictionaries — a raw
JSON-Schema blob:

&lt;/p>
&lt;pre>&lt;code class="lang-csharp">new Dictionary&lt;string, object&gt;
&#123;
    ["service"] = new Dictionary&lt;string, object&gt;
    &#123;
        ["type"] = "string",
        ["description"] = "The service",
        ["required"] = true,
    &#125;,
&#125;;&lt;/code>&lt;/pre>
&lt;p>
&lt;xref href="SignalWire.SWAIG.ParameterSchema" data-throw-if-not-resolved="false">&lt;/xref> produces the &lt;strong>byte-identical&lt;/strong>
&lt;code>properties&lt;/code> map type-safely:
&lt;/p>
&lt;pre>&lt;code class="lang-csharp">ParameterSchema.Create()
    .String("service", "The service")
    .String("date", "YYYY-MM-DD")
    .Enum("fmt", typeof(RecordFormat), "format")  // → enum: ["wav","mp3","mp4"]
    .Required("service", "date")
    .Build();&lt;/code>&lt;/pre>
&lt;p>
This is a &lt;strong>typed convenience over the same wire output, not a new
format&lt;/strong>. &lt;xref href="SignalWire.SWAIG.ParameterSchema.Build" data-throw-if-not-resolved="false">&lt;/xref> returns exactly the dictionary you
would have hand-written, so it drops straight into the existing untyped
&lt;code>DefineTool&lt;/code> path — which keeps working unchanged. There is no Python
reference counterpart (Python builds the dict by hand); this is a .NET-only
additive ergonomics layer.
&lt;/p>
&lt;p>
&lt;strong>Required.&lt;/strong> &lt;xref href="SignalWire.SWAIG.ParameterSchema.Required(System.String%5b%5d)" data-throw-if-not-resolved="false">&lt;/xref> marks properties
with the inline &lt;code>["required"] = true&lt;/code> flag this port's hand-written
tool params already use (e.g. &lt;code>MathSkill&lt;/code>, &lt;code>SpiderSkill&lt;/code>,
&lt;code>InfoGathererSkill&lt;/code>). For callers that prefer the top-level JSON-Schema
&lt;code>required: [...]&lt;/code> array instead (the shape &lt;code>DataMap&lt;/code> and the Python
&lt;code>SWAIGFunction&lt;/code> emit), &lt;xref href="SignalWire.SWAIG.ParameterSchema.RequiredNames" data-throw-if-not-resolved="false">&lt;/xref> exposes the same
names as an ordered &lt;code>List&lt;string&gt;&lt;/code>.
&lt;/p>
&lt;p>
&lt;strong>Closed sets.&lt;/strong> Enum(string, Type, string?)
integrates the Tier-1 typed enums (&lt;xref href="SignalWire.SWAIG.RecordFormat" data-throw-if-not-resolved="false">&lt;/xref>,
&lt;xref href="SignalWire.SWAIG.RecordDirection" data-throw-if-not-resolved="false">&lt;/xref>, &lt;xref href="SignalWire.SWAIG.TapDirection" data-throw-if-not-resolved="false">&lt;/xref>,
&lt;xref href="SignalWire.SWAIG.Codec" data-throw-if-not-resolved="false">&lt;/xref>, …) by reflecting over the enum's members and emitting
each member's canonical wire string (its &lt;code>ToWireName&lt;/code>) into the schema
&lt;code>enum: [...]&lt;/code> list — so the closed set is defined once, in the enum,
rather than re-typed as a string list at every call site. An explicit
Enum(string, IEnumerable&lt;string&gt;, string?) overload remains
for genuinely-open or ad-hoc value sets.
&lt;/p>
