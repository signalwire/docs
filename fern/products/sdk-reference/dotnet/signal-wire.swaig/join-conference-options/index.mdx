# `JoinConferenceOptions`

Typed options bag for the convenience overload &lt;xref href="SignalWire.SWAIG.FunctionResult.JoinConference(System.String%2cSignalWire.SWAIG.JoinConferenceOptions)" data-throw-if-not-resolved="false">&lt;/xref>.

**Remarks:** &lt;p>
Mirrors the 18 optional parameters of the Python reference
`join_conference(name, muted=False, beep="true", ...)` one-for-one, with
the four closed-set arguments (`beep`, `record`, `trim`, the two
callback methods) surfaced as the typed enums &lt;xref href="SignalWire.SWAIG.ConferenceBeep" data-throw-if-not-resolved="false">&lt;/xref>, &lt;xref href="SignalWire.SWAIG.ConferenceRecord" data-throw-if-not-resolved="false">&lt;/xref>, &lt;xref href="SignalWire.SWAIG.ConferenceTrim" data-throw-if-not-resolved="false">&lt;/xref>, and &lt;xref href="SignalWire.SWAIG.CallbackMethod" data-throw-if-not-resolved="false">&lt;/xref> for
editor autocompletion and compile-time checking.

&lt;/p>
&lt;p>
This record is a .NET-idiomatic convenience (a single options object instead
of 18 positional arguments). The flat, all-string
&lt;xref href="SignalWire.SWAIG.FunctionResult.JoinConference(System.String%2cSystem.Boolean%2cSystem.String%2cSystem.Boolean%2cSystem.Boolean%2cSystem.String%2cSystem.Int32%2cSystem.String%2cSystem.String%2cSystem.String%2cSystem.String%2cSystem.String%2cSystem.String%2cSystem.String%2cSystem.String%2cSystem.String%2cSystem.String%2cSystem.Object)" data-throw-if-not-resolved="false">&lt;/xref>
overload remains the parity-bearing signature against the Python reference;
the convenience overload delegates straight to it via each enum's
`ToWireName()`, so the emitted `join_conference` action is identical.
&lt;/p>
&lt;p>
Every default matches the Python reference exactly, so an unset options object
collapses to the simple-form bare conference-name string just like the flat
overload with no arguments.
&lt;/p>
