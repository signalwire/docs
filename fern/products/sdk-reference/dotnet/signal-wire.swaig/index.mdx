### Codec

compile-time-checked closed set.

**Remarks:** &lt;p>
The Python reference validates this argument explicitly
(&lt;code>tap(... codec ...)&lt;/code> raises &lt;code>ValueError&lt;/code> unless the value is &lt;code>"PCMU"&lt;/code> or &lt;code>"PCMA"&lt;/code>), so it is a genuine closed set rather than a
free-form string. The wire strings are upper-case (&lt;code>"PCMU"&lt;/code> / &lt;code>"PCMA"&lt;/code>).

&lt;/p>
&lt;p>
This is the two-value SWAIG &lt;code>tap&lt;/code> codec set and is deliberately
&lt;strong>distinct&lt;/strong> from the larger RELAY &lt;code>connect&lt;/code>/&lt;code>stream&lt;/code>
codec superset (&lt;code>&#123;PCMU, PCMA, OPUS, G729, G722, VP8, H264&#125;&lt;/code>, optionally
comma-joined): only &lt;code>PCMU&lt;/code>/&lt;code>PCMA&lt;/code> are valid here, so this type must
not be reused for those RELAY params (which stay strings).
FunctionResult.Tap(string, TapDirection, Codec, string, int, string?)
accepts this enum OR a string: the enum gives editor autocompletion and turns
a typo into a compile error, while the string overload preserves parity with
the Python reference (which takes a bare &lt;code>str&lt;/code>).
&lt;/p>
&lt;p>
Each member maps to its canonical wire value via
&lt;xref href="SignalWire.SWAIG.CodecExtensions.ToWireName(SignalWire.SWAIG.Codec)" data-throw-if-not-resolved="false">&lt;/xref>; the enum is purely a typed
alias over those strings, so the emitted SWML is identical to passing the
string directly.
&lt;/p>
&lt;example>
