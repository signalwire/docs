### Codec

Media codec for &lt;xref href="SignalWire.SWAIG.FunctionResult.Tap(System.String%2cSystem.String%2cSignalWire.SWAIG.TapDirection%2cSignalWire.SWAIG.Codec%2cSystem.Int32%2cSystem.String)" data-throw-if-not-resolved="false">&lt;/xref>, as a typed,
compile-time-checked closed set.

**Remarks:** &lt;p>
The Python reference validates this argument explicitly
(`tap(... codec ...)` raises `ValueError` unless the value is
`"PCMU"` or `"PCMA"`), so it is a genuine closed set rather than a
free-form string. The wire strings are upper-case (`"PCMU"` / `"PCMA"`).

&lt;/p>
&lt;p>
This is the two-value SWAIG `tap` codec set and is deliberately
&lt;strong>distinct&lt;/strong> from the larger RELAY `connect`/`stream`
codec superset (`&#123;PCMU, PCMA, OPUS, G729, G722, VP8, H264&#125;`, optionally
comma-joined): only `PCMU`/`PCMA` are valid here, so this type must
not be reused for those RELAY params (which stay strings).
FunctionResult.Tap(string, TapDirection, Codec, string, int, string?)
accepts this enum OR a string: the enum gives editor autocompletion and turns
a typo into a compile error, while the string overload preserves parity with
the Python reference (which takes a bare `str`).
&lt;/p>
&lt;p>
Each member maps to its canonical wire value via
&lt;xref href="SignalWire.SWAIG.CodecExtensions.ToWireName(SignalWire.SWAIG.Codec)" data-throw-if-not-resolved="false">&lt;/xref>; the enum is purely a typed
alias over those strings, so the emitted SWML is identical to passing the
string directly.
&lt;/p>
&lt;example>
&lt;pre>&lt;code class="lang-csharp">result.Tap("rtp://1.2.3.4:5000", TapDirection.Both, Codec.Pcma);  // typed, autocompleted
result.Tap("rtp://1.2.3.4:5000", codec: "PCMA");                   // string still works (parity)&lt;/code>&lt;/pre>
&lt;/example>
