### ExecuteRpc(string, Dictionary\<string, object>?, string?, string?)

Execute an RPC method on a call using SWML.

**Remarks:** Parity with the Python reference
`execute_rpc(method, params=None, call_id=None, node_id=None)`: the
rpc params dict is keyed `{method, call_id?, node_id?, params?}` —
`call_id`/`node_id` are TOP-LEVEL siblings of `method`/`params`,
NOT nested inside `params` — and the `{execute_rpc: …}` verb is
wrapped in a full SWML document under the `SWML` action key. There is
no `jsonrpc` envelope, and method strings are bare (e.g. `"dial"`,
not `"calling.dial"`). `params` is omitted when empty.

### ExecuteSwml(object, bool)

### Pay(string, string, string?, string, int, int, bool, object?, int, string, string?, string, string, string, string?, string, List\<Dictionary\<string, string>>?, List\<Dictionary\<string, object>>?, string?)

**Remarks:** Full parity with the Python reference &lt;code>pay(payment\_connector\_url, input\_method="dtmf", status\_url=None,

`input` (NOT `input_method`); numeric and boolean fields are

### RecordCall

Start background call recording using SWML — canonical, full-arity
overload with the two closed-set arguments (&lt;code class="paramref">format&lt;/code> / &lt;code class="paramref">direction&lt;/code>) as the typed &lt;xref href="SignalWire.SWAIG.RecordFormat" data-throw-if-not-resolved="false">&lt;/xref> / &lt;xref href="SignalWire.SWAIG.RecordDirection" data-throw-if-not-resolved="false">&lt;/xref> enums.

**Remarks:** Full parity with the Python reference &lt;code>record\_call(control\_id, stereo, format, direction, terminators, beep,
input\_sensitivity, initial\_timeout, end\_silence\_timeout, max\_length,
status\_url)&lt;/code>, in the same parameter order. The Python reference
validates the bare-string `format`/`direction` against the closed
sets \{wav,mp3,mp4\} / \{speak,listen,both\}; this overload surfaces those
knowable sets as enums so a bad value is a compile error rather than a
runtime `ValueError` (a same-arity bare-string overload preserves the
Python `str` path — see &lt;xref href="SignalWire.SWAIG.FunctionResult.RecordCall(System.String%2cSystem.Boolean%2cSystem.String%2cSystem.String%2cSystem.String%2cSystem.Boolean%2cSystem.Double%2cSystem.Nullable%7bSystem.Double%7d%2cSystem.Nullable%7bSystem.Double%7d%2cSystem.Nullable%7bSystem.Double%7d%2cSystem.String)" data-throw-if-not-resolved="false">&lt;/xref>).
The `record_call` verb is wrapped in a SWML document
(`{version, sections: {main: [{record_call: ...}]}}`) and emitted under
the `SWML` action key — there is no bare top-level `record_call`
action and no invented `initiator` key. `stereo`, `format`,
`direction`, `beep`, and `input_sensitivity` are ALWAYS
emitted (Python emits them unconditionally); the remaining parameters are
emitted only when set.
