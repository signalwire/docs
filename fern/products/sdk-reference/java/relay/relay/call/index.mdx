# `Call`

Represents a RELAY call with 30+ methods organized by category.

&lt;p>
Methods are grouped into:
&lt;ul>
  &lt;li>&lt;b>Simple fire-and-response&lt;/b>: answer, hangup, pass, hold, etc.&lt;/li>
  &lt;li>&lt;b>Action-based&lt;/b>: play, record, detect, collect, etc.

(return `Action`)&lt;/li>

  &lt;li>&lt;b>Connection&lt;/b>: connect, disconnect, transfer&lt;/li>
  &lt;li>&lt;b>Conference&lt;/b>: joinConference, leaveConference&lt;/li>
  &lt;li>&lt;b>AI&lt;/b>: ai, aiMessage, aiHold, aiUnhold, amazonBedrock&lt;/li>
&lt;/ul>

Event routing: the `RelayClient` routes events to calls by `call_id`.
Each call maintains an `actions` map keyed by `control_id` for action events.

### getCallState

The current call state as a typed `CallState`, &lt;em>alongside&lt;/em>
the raw string `#getState()` (which stays canonical for parity and
forward-compat).

Returns `Optional.empty()` when the live wire state
is not one of the five known `CallState` values — the set mirrors
server-emitted values that can grow, so an unrecognised state is tolerated
here rather than crashing the caller. The present value always agrees with
`getState()`: `getCallState().get().getValue().equals(getState())`.

#### Signature

```java
public Optional<CallState> getCallState()
```

#### Returns

`Optional<CallState>` — the typed state, or empty if the raw state is unknown/unset.
