# `RelayClient`

RELAY WebSocket connection manager.

&lt;p>
Manages WebSocket connections to the SignalWire RELAY service using JSON-RPC 2.0.
Implements the four correlation mechanisms:
&lt;ol>
  &lt;li>JSON-RPC id -> CompletableFuture for RPC response matching&lt;/li>
  &lt;li>call_id -> Call for event routing&lt;/li>
  &lt;li>control_id -> Action per Call for action event routing&lt;/li>
  &lt;li>tag -> CompletableFuture&lt;Call&gt; for dial correlation&lt;/li>
&lt;/ol>
&lt;p>
Also handles:
&lt;ul>
  &lt;li>Event ACK for every `signalwire.event`&lt;/li>
  &lt;li>Ping/pong for `signalwire.ping`&lt;/li>
  &lt;li>Exponential backoff reconnection&lt;/li>
  &lt;li>Authorization state for fast reconnection&lt;/li>
  &lt;li>Server-initiated disconnect with restart flag&lt;/li>
  &lt;li>Dynamic context subscription via receive/unreceive&lt;/li>
  &lt;li>Message tracking by message_id&lt;/li>
&lt;/ul>

&lt;pre>client = RelayClient.builder()
    .project("project-id")
    .token("api-token")
    .space("example.signalwire.com")
    .contexts(List.of("default"))
    .build();

client.onCall(call -> &#123;
    call.answer();
    var action = call.play(List.of(Map.of("type", "tts",
        "params", Map.of("text", "Hello!"))));
    action.waitForCompletion();
    call.hangup(););

client.run();
&#125;&lt;/pre>

`AutoCloseable` entry point so the client can be used in a
try-with-resources block:

&lt;pre>&#123;@code
try (var client = RelayClient.builder()...build()) &#123;
    client.connect();
    // ...

`AutoCloseable` entry point so the client can be used in a
try-with-resources block:

&lt;pre>(var client = RelayClient.builder()...build()) &#123;
    client.connect();
    // ... use the client ... // close() runs here: WebSocket shut down, worker pool released
&#125;&lt;/pre>

Releases the same resources as `#disconnect()` — closes the
RELAY WebSocket, releases the `#run()` latch, and shuts the
worker `java.util.concurrent.ExecutorService` down — so the
client is the rough Java parallel of Python's
with RelayClient(...) context manager. Idempotent: a
second call is a harmless no-op.

**Decorators:** `@Override`
