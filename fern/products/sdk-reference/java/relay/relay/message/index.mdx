# `Message`

Represents an SMS/MMS message with state tracking.

&lt;p>
Messages are simpler than calls: they have a `message_id`, progress through
states (queued, initiated, sent, delivered), and support completion waiting.
&lt;p>
State progression:
&lt;ul>
  &lt;li>Success: queued -> initiated -> sent -> delivered&lt;/li>
  &lt;li>Failure: queued -> initiated -> failed/undelivered&lt;/li>
&lt;/ul>
Terminal states: delivered, undelivered, failed

### getMessageState

The current message state as a typed `MessageState`, &lt;em>alongside&lt;/em> the raw string `#getState()` (which stays
canonical for parity and forward-compat).

Returns `Optional.empty()`
when the live wire state is not one of the seven known
`MessageState` values — the set mirrors server-emitted values that
can grow, so an unrecognised state is tolerated here rather than crashing
the caller. The present value always agrees with `getState()`:
`getMessageState().get().getValue().equals(getState())`.

#### Signature

```java
public Optional<MessageState> getMessageState()
```

#### Returns

`Optional<MessageState>` — the typed state, or empty if the raw state is unknown/unset.
