# `Device`

A typed view of the RELAY params\} device object that recurs as
a raw Object> across `Call#connect`, `Call#refer`,
`RelayClient#dial` and `Call#tap`.

&lt;p>The wire shape is fixed (extracted from switchblade's
`PublicCallConnectParams` / `PublicCallDialParams` /
`PublicCallReferParams` / `PublicCallTapParams`, mirrored in
`porting-sdk/relay-protocol/calling.*.params.json`): every device is an
object with a required string `type` discriminant and a `params`
sub-object whose shape depends on the type (a `phone` carries
`to_number`/`from_number`, a `sip` carries `to`/
`headers`, etc.).

This class types the &lt;em>shape&lt;/em> — the
params\} envelope — so callers stop hand-rolling the two-key
map and get an explicit constructor + accessors.

&lt;p>&lt;strong>`type` stays a `String`.&lt;/strong> The discriminant is
NOT schema-enumerated: the wire schema constrains it only to
&#123;"type": "string"&#125;, and the platform accepts device types the
SDK may not know about. Typing it as an enum would reject valid values, so it
is deliberately left open (the convention names `phone`/`sip`/
`webrtc` — see `Constants#DEVICE_TYPE_PHONE` etc. — but those are
conveniences, not a closed set). `params` is left an arbitrary
Object> for the same reason: it is type-dependent and not
a single fixed schema.

&lt;p>&lt;strong>Additive — the raw-map path stays.&lt;/strong> Every method that
takes a device still accepts a raw Object>; this class is
a convenience on top. `#toMap()` produces a map that is
&lt;em>byte-for-byte identical&lt;/em> to the hand-written
…, "params": …&#125; (insertion order `type` then
`params`, matching every existing call site), so routing a device
through `Device` serialises exactly as the raw map does.

&lt;pre>typed:
Device phone = Device.of("phone", Map.of(
    "to_number", "+15551112222", "from_number", "+15553334444"));
call.connect(List.of(List.of(phone.toMap())), null);

// raw map still works (parity):
call.connect(List.of(List.of(Map.of(
    "type", "phone",
    "params", Map.of("to_number", "+15551112222",
                     "from_number", "+15553334444")))), null);&lt;/pre>

&lt;p>Python uses a bare `dict` for this object, so there is no reference
equivalent — this is a documented Java addition.

### toMap

Serialise to the wire shape: a two-key map …, "params": …\}
with `type` inserted first, identical to the hand-written map every
existing call site builds.

The returned map is fresh and mutable (callers
routinely nest it inside a `devices` list), and `params` is a
defensive copy so later mutation of the returned map cannot corrupt this
`Device`.

#### Signature

```java
public Map<String,Object> toMap()
```

#### Returns

`Map<String,Object>` — a new params\} map ready to pass to `connect`/`refer`/`dial`/`tap`.
