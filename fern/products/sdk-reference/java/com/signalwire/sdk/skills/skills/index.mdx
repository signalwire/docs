## Enums

### SkillName

Built-in skill names as a typed, compile-time-checked closed set.

&lt;p>java.util.Map)
(and the matching `removeSkill` / `hasSkill` overloads) accept
this enum &lt;em>or&lt;/em> a plain `String`.

The enum gives editor
autocompletion and makes a typo fail at compile time (a bare string like
`"datetiem"` only fails at runtime, on the server). Strings keep
parity with the Python reference (which uses bare `str`) and still
allow custom / third-party skills that aren't built in:

&lt;pre>// typed, autocompleted
agent.addSkill("datetime");                  // string still works (parity)
agent.addSkill("my_custom_skill");           // open set: custom skills ok&lt;/pre>

&lt;p>Each constant's value is the canonical wire string
(the key the skill registers under in `SkillRegistry`), so routing a
call through the enum is byte-for-byte identical to passing that string.
