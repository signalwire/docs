# `LambdaAgentHandler`

AWS Lambda adapter for a SignalWire `AgentBase`.

&lt;p>Translates API Gateway (REST v1 or HTTP v2) / Lambda Function URL
 events into the same dispatch logic the in-process HTTP server uses,
 without depending on the `aws-lambda-java-events` typed models
 at runtime (the handler accepts plain Object> so
 the SDK stays dependency-light).

 &lt;p>&lt;b>Usage&lt;/b> \u2014 in your Lambda handler class:
 &lt;pre>class MyHandler implements RequestHandler&lt;Map&lt;String, Object>, Map&lt;String, Object>> \{
     private final LambdaAgentHandler delegate;

```
 public MyHandler() {
     AgentBase agent = AgentBase.builder()
             .name("my-agent")
             .route("/")
             .build();
     agent.setPromptText("You are helpful.");
     this.delegate = new LambdaAgentHandler(agent);
```

     public Map&lt;String, Object> handleRequest(Map&lt;String, Object> event, Context ctx) \{
         return delegate.handle(event).toMap();
     }
 }
 }&lt;/pre>

 &lt;p>The adapter dispatches based on the request path and the agent's
 configured route:
 &lt;ul>
   &lt;li>/&lt;route>          \u2192 returns rendered SWML&lt;/li>
   &lt;li>/&lt;route>/swaig    \u2192 executes the named SWAIG tool&lt;/li>
   &lt;li>/&lt;route>/post_prompt \u2192 invokes the summary callback&lt;/li>
   &lt;li>/&lt;route>/mcp      \u2192 JSON-RPC 2.0 (if MCP enabled)&lt;/li>
   &lt;li>/health, `/ready` \u2192 health probes&lt;/li>
 &lt;/ul>

 &lt;p>All webhook URLs generated inside the SWML document are built from
 the Lambda's Function URL (via `LambdaUrlResolver`) or the
 `SWML_PROXY_URL_BASE` override, with the agent's route always
 layered on top. This is the same invariant the HTTP server path
 enforces.

**Modifiers:** `final`
