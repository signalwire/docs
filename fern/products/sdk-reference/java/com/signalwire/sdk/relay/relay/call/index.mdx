# `Call`

Represents a RELAY call with 30+ methods organized by category.

&lt;p>
 Methods are grouped into:
 &lt;ul>
   &lt;li>&lt;b>Simple fire-and-response&lt;/b>: answer, hangup, pass, hold, etc.&lt;/li>
   &lt;li>&lt;b>Action-based&lt;/b>: play, record, detect, collect, etc. (return `Action`)&lt;/li>
   &lt;li>&lt;b>Connection&lt;/b>: connect, disconnect, transfer&lt;/li>
   &lt;li>&lt;b>Conference&lt;/b>: joinConference, leaveConference&lt;/li>
   &lt;li>&lt;b>AI&lt;/b>: ai, aiMessage, aiHold, aiUnhold, amazonBedrock&lt;/li>
 &lt;/ul>

 Event routing: the `RelayClient` routes events to calls by `call_id`.
 Each call maintains an `actions` map keyed by `control_id` for action events.
