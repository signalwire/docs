# `get_execution_mode`

Cross-language SDK contract for serverless / deployment-mode detection.
Mirrors signalwire.core.logging_config.get_execution_mode in the Python reference. Order of precedence (FIRST match wins):

GATEWAY_INTERFACE -> "cgi"
AWS_LAMBDA_FUNCTION_NAME or LAMBDA_TASK_ROOT -> "lambda"
FUNCTION_TARGET, K_SERVICE, or GOOGLE_CLOUD_PROJECT -> "google_cloud_function"
AZURE_FUNCTIONS_ENVIRONMENT, FUNCTIONS_WORKER_RUNTIME, or AzureWebJobsStorage -> "azure_function"
otherwise -> "server"


The detected mode as a canonical lower-case string. One of "cgi", "lambda", "google_cloud_function", "azure_function", or "server".
