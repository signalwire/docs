[validate-webhook-signature]: /docs/server-sdks/reference/typescript/core/security/validate-webhook-signature

Verify a SignalWire webhook signature, accepting either a raw body string or
pre-parsed form params. This is the drop-in replacement for the legacy
`@signalwire/compatibility-api` `validateRequest`. For new code that has access
to the raw body, prefer [`validateWebhookSignature()`][validate-webhook-signature].

```typescript {1}
validateRequest(signingKey: string, signature: string, url: string, paramsOrRawBody: string | Record<string, unknown> | Array<[string, unknown]> | null | undefined): boolean
```

## **Parameters**

<ParamField path="signingKey" type="string" required={true} toc={true}>
  Your SignalWire Signing Key. An empty or non-string value throws an `Error`.
</ParamField>

<ParamField path="signature" type="string" required={true} toc={true}>
  The signature header value. A missing or empty value returns `false`.
</ParamField>

<ParamField path="url" type="string" required={true} toc={true}>
  The full URL SignalWire POSTed to.
</ParamField>

<ParamField path="paramsOrRawBody" type="string | Record<string, unknown> | Array<[string, unknown]> | null | undefined" required={true} toc={true}>
  Either the raw body string, or pre-parsed form params as a record, a `Map`, or
  an array of `[key, value]` tuples.
</ParamField>

## **Returns**

`boolean` -- `true` when the signature matches; `false` otherwise. The
comparison is constant-time.

## **Example**

```typescript {6}
import { validateRequest } from '@signalwire/sdk';

// Pre-parsed form params (compat surface)
const params = { From: '+15551234567', To: '+15559876543' };

const ok = validateRequest(
  process.env.SIGNALWIRE_SIGNING_KEY!,
  signature,
  'https://example.com/webhook',
  params,
);
```
