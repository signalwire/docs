Register a callback for debug events. Pair this with [`EnableDebugEvents`][enable-debug-events] so the SignalWire platform actually emits events.

<Note>
The Go SDK does not auto-register a `/debug_events` HTTP route — debug-event webhooks need to be routed to a handler you control before this callback can be invoked. See [`EnableDebugEvents`][enable-debug-events] for the wiring expectations and the upstream SDK gap.
</Note>

<Note>
The Go callback receives a single `event map[string]any` argument — the full event payload including `event_type`, `call_id`, `label`, and event-specific fields. Python splits these into `(event_type, data)`; in Go, read `event["event_type"]` from the same map.
</Note>

## **Parameters**

<ParamField path="cb" type="DebugEventHandler" required={true} toc={true}>
  Callback signature: `func(event map[string]any)`. The map carries the entire decoded payload.
</ParamField>

## **Returns**

[`*AgentBase`][ref-agentbase] — returns the receiver for method chaining.

## **Example**

```go {12-22}
package main

import (
	"log"

	"github.com/signalwire/signalwire-go/pkg/agent"
)

func main() {
	a := agent.NewAgentBase(agent.WithName("debug-agent"), agent.WithRoute("/debug"))
	a.SetPromptText("You are a helpful assistant.")
	a.EnableDebugEvents(1)
	a.OnDebugEvent(func(event map[string]any) {
		eventType, _ := event["event_type"].(string)
		switch eventType {
		case "llm_error":
			log.Printf("LLM error: %v", event)
		case "barge":
			log.Printf("Barge detected: %v ms", event["barge_elapsed_ms"])
		case "step_change":
			log.Printf("Step changed to: %v", event["step"])
		}
	})

	_ = a.Serve()
}
```
