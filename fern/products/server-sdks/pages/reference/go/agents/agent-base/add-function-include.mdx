Add a remote SWAIG function include so the agent can call tools hosted on an
external server. The remote endpoint is contacted at session start and the
listed functions become available to the AI just like locally defined tools.

<Info>
This maps to the SWML [`ai.swaig.includes`][ai-swaig-includes] array. See the
[SWML SWAIG includes reference][swml-swaig-includes-reference] for details.
</Info>

<Note>
`functions` and `metaData` are only added to the include map when non-empty.
To replace all includes in one call, use [`SetFunctionIncludes`][set-function-includes].
</Note>

## **Parameters**

<ParamField path="url" type="string" required={true} toc={true}>
  URL of the remote SWAIG server that hosts the functions.
</ParamField>

<ParamField path="functions" type="[]string" required={true} toc={true}>
  List of function names to include from the remote server. Omitted from the
  rendered include when empty.
</ParamField>

<ParamField path="metaData" type="map[string]any" toc={true}>
  Optional metadata passed alongside the function include. Useful for auth
  tokens or context delivered to the remote server. Omitted when empty.
</ParamField>

## **Returns**

[`*AgentBase`][ref-agentbase] — returns the receiver for fluent chaining.

## **Example**

```go {12-16}
package main

import "github.com/signalwire/signalwire-go/pkg/agent"

func main() {
	a := agent.NewAgentBase(
		agent.WithName("assistant"),
		agent.WithRoute("/assistant"),
	)
	a.SetPromptText("You are a helpful assistant.")

	a.AddFunctionInclude(
		"https://tools.example.com/swaig",
		[]string{"lookup_order", "cancel_order"},
		map[string]any{"auth_token": "secret-token"},
	)

	_ = a.Serve()
}
```
