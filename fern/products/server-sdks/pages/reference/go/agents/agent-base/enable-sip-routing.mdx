Enable SIP-based routing for this agent. The agent registers a routing
callback on its underlying SWML service at `path` (or `/sip` if `path` is
empty). When SignalWire posts a SIP call to that endpoint, the callback
extracts the SIP username from the request body, lower-cases it, and checks
it against the agent's registered username set.

When `autoMap` is `true`, [`AutoMapSipUsernames`][auto-map-sip-usernames] is
invoked to derive common usernames from the agent's name and route. To add
explicit usernames, call [`RegisterSipUsername`][register-sip-username] before
or after this method.

## **Parameters**

<ParamField path="autoMap" type="bool" required={true} toc={true}>
  When `true`, automatically derive and register SIP usernames from the
  agent's name and route. Pass `false` if you have already populated the
  username set with [`RegisterSipUsername`][register-sip-username].
</ParamField>

<ParamField path="path" type="string" required={true} toc={true}>
  URL path for the SIP routing endpoint. Pass an empty string to use the
  default `/sip`.
</ParamField>

## **Returns**

[`*AgentBase`][ref-agentbase] — returns the receiver for fluent chaining.

## **Example**

```go {11}
package main

import "github.com/signalwire/signalwire-go/pkg/agent"

func main() {
	a := agent.NewAgentBase(
		agent.WithName("support"),
		agent.WithRoute("/support"),
	)
	a.SetPromptText("You are a helpful assistant.")
	a.EnableSipRouting(true, "/sip")

	_ = a.Serve()
}
```
