[on-debug-event]: /docs/server-sdks/reference/go/agents/agent-base/on-debug-event

Set the debug-events verbosity level. When the level is greater than `0`, the
agent renders `debug_events: <level>` into the `ai` verb of the generated SWML
so the SignalWire platform begins emitting debug-event webhooks during calls.

To react to those events programmatically, register a callback with
[`OnDebugEvent`][on-debug-event]. Without a registered handler the events are
still emitted by the platform but the agent itself does nothing with them.

<Note>
Unlike the Python SDK, the Go SDK does **not** auto-register a `/debug_events`
HTTP route or auto-populate `debug_webhook_url`. Wire those up explicitly via
your HTTP mux if you need to receive the platform's POSTs in-process.
</Note>

## **Parameters**

<ParamField path="level" type="int" required={true} toc={true}>
  Debug-event verbosity. `0` disables the feature.

  - `1` — high-level events: barge, errors, session start/end, step changes
  - `2+` — adds high-volume events: every LLM request/response,
    `conversation_add`
</ParamField>

## **Returns**

[`*AgentBase`][ref-agentbase] — returns the receiver for fluent chaining.

## **Example**

```go {11}
package main

import "github.com/signalwire/signalwire-go/pkg/agent"

func main() {
	a := agent.NewAgentBase(
		agent.WithName("debug-agent"),
		agent.WithRoute("/debug"),
	)
	a.SetPromptText("You are a helpful assistant.")
	a.EnableDebugEvents(1)

	_ = a.Serve()
}
```
