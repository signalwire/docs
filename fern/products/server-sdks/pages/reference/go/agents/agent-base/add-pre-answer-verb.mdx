Add a verb to run before the call is answered. Pre-answer verbs execute while
the call is still ringing.

<Note>
The Go SDK appends the verb verbatim — no add-time validation runs. Whether a
given verb is accepted before answer is determined by the SWML runtime (and,
when `WithSchemaValidation` is enabled, the SWML schema). See the
[SWML reference][swml-reference] for the verbs that are valid before
`answer`.
</Note>

To remove every pre-answer verb, use
[`ClearPreAnswerVerbs`][clear-pre-answer-verbs].

## **Parameters**

<ParamField path="verbName" type="string" required={true} toc={true}>
  SWML verb name (e.g. `"play"`, `"sleep"`, `"request"`).
</ParamField>

<ParamField path="config" type="map[string]any" required={true} toc={true}>
  Verb configuration map. See the [SWML reference][swml-reference] for
  verb-specific parameters.
</ParamField>

## **Returns**

[`*AgentBase`][ref-agentbase] — returns the receiver for fluent chaining.

## **Example**

```go {12-15}
package main

import "github.com/signalwire/signalwire-go/pkg/agent"

func main() {
	a := agent.NewAgentBase(
		agent.WithName("support"),
		agent.WithRoute("/support"),
	)
	a.SetPromptText("You are a helpful assistant.")

	a.AddPreAnswerVerb("play", map[string]any{
		"urls":        []string{"ring:us"},
		"auto_answer": false,
	})

	_ = a.Serve()
}
```
