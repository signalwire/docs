# Browser SDK Web Components

The Browser SDK contains a suite of flexible web components that can be used to create UI.

---

## Components

Ready-to-use call UI elements. Most components consume context from a parent `<sw-call-media>` automatically.

| Component | Description |
|-----------|-------------|
| [`<sw-audio-level>`](./components/sw-audio-level) | Real-time audio level visualizer — animated bar graph |
| [`<sw-call-controls>`](./components/sw-call-controls) | Mute audio, mute video, screen share, and hangup buttons |
| [`<sw-call-media>`](./components/sw-call-media) | Remote video container and root context provider — wrap all other call components inside it |
| [`<sw-call-status>`](./components/sw-call-status) | Displays call state label and elapsed duration timer |
| [`<sw-call-widget>`](./components/sw-call-widget) | All-in-one call widget — handles client init, dialing, media, controls, and optional AI transcript |
| [`<sw-click-to-call>`](./components/sw-click-to-call) | Single-button widget that dials on click and shows mute/hangup once connected |
| [`<sw-device-selector>`](./components/sw-device-selector) | Drop-up buttons for microphone, camera, and speaker selection with optional live preview |
| [`<sw-dialpad>`](./components/sw-dialpad) | DTMF keypad with optional call button |
| [`<sw-directory>`](./components/sw-directory) | Searchable contact list with dial actions from the SDK directory |
| [`<sw-local-camera>`](./components/sw-local-camera) | Local camera preview with a muted overlay |
| [`<sw-participant-controls>`](./components/sw-participant-controls) | Per-participant moderation controls: mute, video mute, remove, volume, pin |
| [`<sw-participants>`](./components/sw-participants) | Renders positioned overlays for each remote participant |
| [`<sw-self-media>`](./components/sw-self-media) | Local video preview — consumes `callContext` from a parent `<sw-call-media>` |

---

## Context

Lit context keys and reactive controllers used to share state down the component tree.

| Name | Description |
|------|-------------|
| [`callContext`](./context/call-context) | Lit context key for sharing a `Call` object — provided by `<sw-call-media>` |
| [`transcriptContext`](./context/transcript-context) | Lit context key for sharing live transcript state — provided by `TranscriptController` |
| [`TranscriptController`](./context/TranscriptController) | Reactive controller that subscribes to call signaling events and drives the AI transcript state machine |

---

## UI Primitives

Low-level layout and control components used to build custom interfaces.

| Component | Description |
|-----------|-------------|
| [`<one-layout>`](./ui/one-layout) | Responsive split layout: video pane + controls bar + animated transcript panel |
| [`<responsive-container>`](./ui/responsive-container) | Sizes itself to a responsive modal footprint with 16:9 aspect ratio |
| [`<sw-ctrl-button>`](./ui/sw-ctrl-button) | Plain action button for control bars |
| [`<sw-ctrl-dropup>`](./ui/sw-ctrl-dropup) | Button that opens a menu above itself with selected item indicator |
| [`<sw-ctrl-toggle>`](./ui/sw-ctrl-toggle) | Two-state toggle button — parent owns state |
| [`<sw-modal>`](./ui/sw-modal) | Wraps a native `<dialog>` with bounce-in/out animations and body scroll lock |
| [`<sw-prompt>`](./ui/sw-prompt) | Confirm/alert dialog — use the element directly or the `showPrompt` helper |
| [`<sw-transcript-view>`](./ui/sw-transcript-view) | Renders AI transcript bubbles — must be inside a host that uses `TranscriptController` |
