Call controls split across three objects:

- **`Call`** owns session-level actions: hangup, transfer, hold, lock,
  layout, send DTMF.
- **`SelfParticipant`** (`call.self`) owns *your* state: mute,
  deaf, hand raise, screen share, volume, audio processing, end call,
  remove self.
- **`Participant`** (entries in `call.participants$`) owns moderation
  actions on *other* members — mute them, change their position,
  remove them. Subject to `call.self.capabilities.member` permissions.

The pattern that runs through everything: **mutator method + matching
`$` observable**. Call the method, listen on the observable for UI
state. Don't try to track local state in a `let isMuted` variable —
the server is the source of truth and may flip it (a moderator can
mute you).

## Mute / unmute (self)

```js
call.self$.subscribe((self) => {
  if (!self) return;

  muteBtn.onclick = () => self.toggleMute();

  self.audioMuted$.subscribe((muted) => {
    muteBtn.classList.toggle("muted", muted);
    muteBtn.textContent = muted ? "Unmute" : "Mute";
  });
});
```

If you need explicit [`mute()`] / [`unmute()`] (e.g. to force a state
regardless of current), they're available too:

```js
await self.mute();        // force muted
await self.unmute();      // force unmuted
await self.muteVideo();   // video off
await self.unmuteVideo(); // video on
```

The toggles are idempotent — calling `toggleMute()` while a network
request is in flight is safe; the SDK serializes them.

## Deaf

"Deaf" means: stop playing remote audio to *me*. Other participants
can still hear each other. Useful when the user wants to step away
without leaving the room. Different from mute (which is about what
*you* send).

```js
self.toggleDeaf();
self.deaf$.subscribe((deaf) => deafBtn.classList.toggle("active", deaf));
```

## Hand raise

```js
self.toggleHandraise();
self.handraised$.subscribe((up) => handBtn.classList.toggle("active", up));

// And to see who *else* has their hand up:
call.participants$.subscribe((participants) => {
  for (const p of participants) {
    p.handraised$.subscribe((up) => updateAvatarBadge(p.id, up));
  }
});
```

The `call.raiseHandPriority$` observable indicates whether the room
prioritizes raised hands in the layout (server-side setting).

## Sending DTMF

```js
// Single digit (call as the user presses)
await call.sendDigits("5");

// Or batch
await call.sendDigits("1234#");
```

Send digits only after `status$` is `'connected'` — earlier and the
remote side may not accept them yet. `sendDigit` capability
(`call.self.capabilities.sendDigit`) controls whether the room
permits DTMF.

## Hold / resume

```js
await call.toggleHold();
```

Hold pauses local media transmission (mic + camera) without
disconnecting the call. The remote side hears silence; the call
resumes with the same media tracks when you toggle again.

This is different from `toggleMute()` — mute disables a single track,
hold pauses the whole call. There's no `held$` observable in v4;
track it in your own state if needed.

## Lock / unlock

```js
await call.toggleLock();
call.locked$.subscribe((locked) => lockBtn.classList.toggle("active", locked));
```

A locked room rejects new joiners until it's unlocked. Permission
gated by `call.self.capabilities.lock`.

## Transfer

```js
await call.transfer({
  destination: "/private/specialist",
});
```

Transfer hands the call off to another destination. The platform
handles the actual cutover; `status$` reflects the transitions.

## Local audio pipeline (advanced)

The standard mute / volume controls are server-mixed. For
**client-side** mic processing — push-to-talk, a local gain knob,
a VAD speaking indicator — use the local pipeline on the `Call`:

```js
// 0..200 percent, 100 = unity
call.setLocalMicrophoneGain(150);
call.localMicrophoneGain$.subscribe(/* ... */);

// Real-time level meter (0..1 RMS at ~30fps)
call.localAudioLevel$.subscribe((level) => {
  meter.style.width = `${level * 100}%`;
});

// Speaking indicator (VAD with hysteresis)
call.localSpeaking$.subscribe((speaking) => {
  selfTile.classList.toggle("speaking", speaking);
});

// Push-to-talk
call.enablePushToTalk();
document.addEventListener("keydown", (e) => {
  if (e.code === "Space") call.setPushToTalkActive(true);
});
document.addEventListener("keyup", (e) => {
  if (e.code === "Space") call.setPushToTalkActive(false);
});
call.disablePushToTalk(); // when done
```

The pipeline is lazy — it engages on first use, so there's no cost
if you don't call any of these.

## Moderation: acting on another participant

`call.participants$` exposes each `Participant`. With the right
capability, you can mute or remove them:

```js
call.participants$.subscribe((participants) => {
  // Note: this includes the self participant — filter if you don't
  // want to expose moderation actions against yourself.
  for (const p of participants.filter((p) => p.id !== call.self?.id)) {
    bindModerationUi(p);
  }
});

function bindModerationUi(p) {
  // Check capabilities first
  call.self?.capabilities.member$.subscribe((member) => {
    muteOthersBtn.disabled = !member.muteAudio.on;
    kickBtn.disabled       = !member.remove;
  });

  muteOthersBtn.onclick = () => p.mute();
  kickBtn.onclick       = () => p.remove();
}
```

See [Capabilities](/docs/browser-sdk/v4/guides/capabilities) for the full
permission model — only show moderation UI when the server actually
grants the action, since trying without the right capability will
fail server-side.

## Putting it together

A minimal control bar:

```js
call.self$.subscribe((self) => {
  if (!self) return;

  // Buttons
  muteBtn.onclick   = () => self.toggleMute();
  videoBtn.onclick  = () => self.toggleMuteVideo();
  deafBtn.onclick   = () => self.toggleDeaf();
  handBtn.onclick   = () => self.toggleHandraise();
  shareBtn.onclick  = () =>
    self.screenShareStatus === "started"
      ? self.stopScreenShare()
      : self.startScreenShare();
  hangupBtn.onclick = () => call.hangup();

  // Observables driving button state
  self.audioMuted$.subscribe((m)   => muteBtn.classList.toggle("muted", m));
  self.videoMuted$.subscribe((m)   => videoBtn.classList.toggle("muted", m));
  self.deaf$.subscribe((d)         => deafBtn.classList.toggle("active", d));
  self.handraised$.subscribe((h)   => handBtn.classList.toggle("active", h));
  self.screenShareStatus$.subscribe((s) =>
    shareBtn.classList.toggle("active", s === "started"));
});
```

That's the full muscle of any call UI. Everything else (layouts,
device pickers, recording badge) is decoration over the same
observable pattern.
