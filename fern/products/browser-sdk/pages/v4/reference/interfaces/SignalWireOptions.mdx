Options for constructing a [SignalWire](/docs/browser-sdk/v4/reference/signalwire).

## **Properties**

<ParamField path="debug" type="DebugOptions" toc={true}>
  Debug options for verbose SDK diagnostics (e.g., `{ logWsTraffic: true }`). See [`DebugOptions`](/docs/browser-sdk/v4/reference/interfaces/debug-options).
</ParamField>

<ParamField path="logger" type="SDKLogger | null" toc={true}>
  Custom logger implementation. Must implement the [SDKLogger](/docs/browser-sdk/v4/reference/interfaces/sdk-logger) interface. Pass `null` to restore the built-in logger. **Note:** Logger configuration is global — setting it on one instance affects all instances.
</ParamField>

<ParamField path="logLevel" type="LogLevel" toc={true}>
  Log level for the built-in logger. Default: `'warn'`. Set to `'debug'` for verbose SDK output. Has no effect when a custom `logger` is provided. **Note:** Logger configuration is global — setting it on one instance affects all instances. See [`LogLevel`](/docs/browser-sdk/v4/reference/type-aliases/log-level).
</ParamField>

<ParamField path="persistSession" type="boolean" toc={true}>
  Persist the session across page reloads. When `true`, credential, authorization state, and protocol are stored in `localStorage` (survives reload). The DPoP key pair is persisted in IndexedDB. On reload, the SDK restores the session from cache without calling `credentialProvider.authenticate()`. When `false` (default), session data lives in `sessionStorage` and is lost on reload. Both [`disconnect()`](/docs/browser-sdk/v4/reference/signalwire/disconnect) and [`destroy()`](/docs/browser-sdk/v4/reference/signalwire/destroy) end the session and clear the persisted resume state and attach records; credentials and device preferences survive. Use [`resetToDefaults()`](/docs/browser-sdk/v4/reference/signalwire/reset-to-defaults) for a full wipe, or [`unregister()`](/docs/browser-sdk/v4/reference/signalwire/unregister) to temporarily stop receiving inbound calls while keeping the session alive.
</ParamField>

<ParamField path="reconnectAttachedCalls" type="boolean" toc={true}>
  Whether to reconnect to previously attached calls.
</ParamField>

<ParamField path="savePreferences" type="boolean" toc={true}>
  Whether to save preferences.
</ParamField>

<ParamField path="skipConnection" type="boolean" toc={true}>
  Skip automatic WebSocket connection on construction.
</ParamField>

<ParamField path="skipDeviceMonitoring" type="boolean" toc={true}>
  Skip monitoring media device changes.
</ParamField>

<ParamField path="skipRegister" type="boolean" toc={true}>
  Skip automatic user registration on construction.
</ParamField>

<ParamField path="storageImplementation" type="Storage" toc={true}>
  Custom storage implementation for persistence. See [`Storage`](/docs/browser-sdk/v4/reference/interfaces/storage).
</ParamField>

<ParamField path="webRTCApiProvider" type="WebRTCApiProvider" toc={true}>
  Custom WebRTC API provider. See [`WebRTCApiProvider`](/docs/browser-sdk/v4/reference/interfaces/webrtc-api-provider).
</ParamField>

<ParamField path="webSocketConstructor" type="NodeSocketAdapter | WebSocketAdapter" toc={true}>
  Custom WebSocket constructor. See [`NodeSocketAdapter`](/docs/browser-sdk/v4/reference/interfaces/node-socket-adapter) and [`WebSocketAdapter`](/docs/browser-sdk/v4/reference/type-aliases/web-socket-adapter).
</ParamField>
