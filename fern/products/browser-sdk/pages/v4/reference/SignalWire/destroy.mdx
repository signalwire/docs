```ts
destroy(): void
```
Destroys the client, clearing timers and releasing all resources.

Intentionally destroying the client also ends its session: the resume
state (authorization state and protocol) and the attach records are both
cleared, so the ended session's calls cannot be reattached after a later
[`connect`](/docs/browser-sdk/v4/reference/signalwire/connect). Credentials
and device preferences are preserved — use
[`resetToDefaults`](/docs/browser-sdk/v4/reference/signalwire/reset-to-defaults)
for a full wipe. To temporarily stop receiving inbound calls while keeping
the session alive, use
[`unregister`](/docs/browser-sdk/v4/reference/signalwire/unregister) instead.

## **Returns**

`void`

## **Examples**

```ts
client.destroy();
```
