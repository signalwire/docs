`WebRTCCall` represents an active call between the local user and one or
more remote participants. Instances are never constructed directly by
applications — they are returned by
[`SignalWire.dial`](/docs/browser-sdk/v4/reference/signalwire#dial) for outbound
calls, or surfaced through the inbound-call flow once the user has
[`register`](/docs/browser-sdk/v4/reference/signalwire#register)ed.

The class owns every aspect of an in-progress call: signaling
(`RTCPeerConnection`, Verto messages), local and remote media streams,
participant roster, layout, recording/streaming state, and network-quality
telemetry. State is exposed in two complementary forms — a snapshot getter
(e.g. [`locked`](/docs/browser-sdk/v4/reference/webrtc-call/locked$#locked))
for one-shot reads and an observable (e.g.
[`locked$`](/docs/browser-sdk/v4/reference/webrtc-call/locked$)) for reactive
UI binding. Event streams that have no snapshot equivalent end in `$` only
(e.g. [`memberJoined$`](/docs/browser-sdk/v4/reference/webrtc-call/member-joined$),
[`signalingEvent$`](/docs/browser-sdk/v4/reference/webrtc-call/signaling-event$)).

The available control methods depend on the capabilities granted by the server
at join time — inspect [`capabilities$`](#capabilities-1) before exposing a
control in your UI. Calling [`hangup`](#hangup) ends the call locally and
transitions [`status$`](#status-1) to `disconnected`; the instance is then safe
to release.

- `Destroyable`

## **Implements**

- `CallManager`

## **Constructors**

### Constructor

```ts
new WebRTCCall(clientSession, options, initialization, address?): WebRTCCall
```

#### Parameters

<ParamField path="clientSession" type="ClientSession" required toc={true}>
  Active client session created by `SignalWire`. Provides the signaling
  transport and authentication context for the call.
</ParamField>

<ParamField path="options" type="CallOptions" required toc={true}>
  Media and behavior options for this call (audio/video constraints,
  recovery, etc.). See [`CallOptions`](/docs/browser-sdk/v4/reference/interfaces/call-options).
</ParamField>

<ParamField path="initialization" type="CallInitialization" required toc={true}>
  Initialization payload supplied by the SDK when constructing the call.
  Carries direction, remote address, and any inbound-call metadata.
</ParamField>

<ParamField path="address" type="Address" toc={true}>
  Optional pre-resolved `Address` for the destination. When omitted, the
  address is derived from the dial string or inbound signaling.
</ParamField>

#### Returns

`WebRTCCall`

## **Properties**

<ParamField path="id" type="string" required toc={true}>
  Unique identifier for this call.
</ParamField>

<ParamField path="to" type="string" toc={true}>
  Destination URI this call was placed to.
</ParamField>
