Use this endpoint for the [Media](/compatibility-api/index) method to delete a message from your Project so it no longer appears in the Dashboard or on the API.
Any Media files that may be associated with this message are not deleted, and will still be available for access in the usual methods.

Messages in progress may not be deleted, and attempting to do so results in an error.

## Parameters

This endpoint takes no parameters.

## Request

<Tabs groupId="compatibility-sdks">

<Tab title="cURL">

```shell
curl https://example.signalwire.com/api/laml/2010-04-01/Accounts/{AccountSid}/Messages/{MessageSid}.json \
  -X DELETE \
  -u "YourProjectID:YourAuthToken"
```

</Tab>

<Tab title="Node.js">

```javascript
const { RestClient } = require('@signalwire/compatibility-api')
const client = RestClient('YourProjectID', 'YourAuthToken', { signalwireSpaceUrl: 'example.signalwire.com' })

client.messages('MessageSid')
      .remove()
      .then(message => console.log(message.sid))
      .done();
```

## Responses

<h4><span class="badge badge--success">204</span> NO CONTENT</h4>

A `204 No Content` response indicates that the Message was successfully deleted.
