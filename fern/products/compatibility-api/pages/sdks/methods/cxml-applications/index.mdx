cXML Applications are our serverless hosted service that allows you to easily serve static documents without setting up any infrastructure.

## The cXML Application Object

> Example response of a cXML Application

```json
{
  "sid": "5184b831-184f-4209-872d-ccdccc80f2f1",
  "date_created": "2019-11-26T20:00:00Z",
  "date_updated": "2020-05-05T20:00:00Z",
  "date_last_accessed": "2020-06-05T20:00:00Z",
  "account_sid": "b3877c40-da60-4998-90ad-b792e98472af",
  "name": "Death Star IVR",
  "contents": "<Response><Say>Hello!</Say></Response>",
  "request_url": "https://{SPACE}.signalwire.com/laml-bins/5184b831-184f-4209-872d-ccdccc80f2f1",
  "num_requests": 1337,
  "api_version": "2010-04-01",
  "uri": "/api/laml/2010-04-01/Accounts/b3877c40-da60-4998-90ad-b792e98472af/LamlBins/5184b831-184f-4209-872d-ccdccc80f2f1",
}
```

<ParamField path="account_sid" type="string">
  The unique identifier for the account this Application is associated with.
</ParamField>

<ParamField path="api_version" type="string">
  The version of the SignalWire API.
</ParamField>

<ParamField path="contents" type="string">
  The contents of the Application, this evaluate to valid XML, with additional support for mustache templating. [You can read more about cXML in our documentation](/compatibility-api/cxml).
</ParamField>

<ParamField path="date_created" type="string">
  The date and time, in ISO 8601 format, the Application was created.
</ParamField>

<ParamField path="date_last_accessed" type="string">
  The date and time, in ISO 8601 format, the Application was last accessed.
</ParamField>

<ParamField path="date_updated" type="string">
  The date and time, in ISO 8601 format, the Application was updated.
</ParamField>

<ParamField path="name" type="string">
  A friendly name given to the cXML Application to help distinguish and search for different Applications within your project.
</ParamField>

<ParamField path="num_requests" type="int">
  The number of times this Application has been accessed.
</ParamField>

<ParamField path="request_url" type="string">
  The unique URL to the raw contents of the cXML Application. Use this as the URL for configuring webhooks or anything needing the XML returned.
</ParamField>

<ParamField path="sid" type="string">
  The unique identifier of the cXML Application on SignalWire. This can be used to show, update, or delete the Application programmatically.
</ParamField>

<ParamField path="uri" type="string">
  The URL of this resource.
</ParamField>
