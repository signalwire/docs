Use this endpoint for the [Recordings](/compatibility-api/index) method to initiate a Recording for a [Call](/compatibility-api/calls/index).
You can use this endpoint to start recording a Call that is already in progress.
If you want to record the entire Call, you can specify recording params during [creation of the Call](/compatibility-api/calls/create).

<EndpointSchemaSnippet endpoint="POST /Accounts/{AccountSid}/Calls/{CallSid}/Recordings" />

## Request Examples

<Tabs groupId="compatibility-sdks">

<Tab title="cURL">

```shell
curl https://example.signalwire.com/api/laml/2010-04-01/Accounts/{AccountSid}/Calls/{CallSid}/Recordings.json \
  -X POST \
  -u "YourProjectID:YourAuthToken" \
  --data-urlencode "RecordingChannels=dual" \
  --data-urlencode "RecordingStatusCallbackEvent=completed,in-progress" \
  --data-urlencode "RecordingStatusCallback=http://your-application.com/callback" \
  --data-urlencode "Trim=do-not-trim" \
  --data-urlencode "RecordingTrack=both"
```

</Tab>
