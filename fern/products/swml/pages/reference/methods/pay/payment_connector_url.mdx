The `payment_connector_url` is a required URL that processes payment requests during the payment flow. It receives payment details once all required information has been gathered from the user, and it processes the final payment transaction, returning the results in the response.

<ParamField path="pay.payment_connector_url" type="string">
  The URL to which payment requests are sent after all required payment details have been provided.
  
  See [pay method][pay-method] for more details.
</ParamField>


## Payment connector URL request

SignalWire sends a `POST` request to the URL specified in the `payment_connector_url` parameter.
The connector URL receives these details once all required payment information has been collected from the user.

<ParamField path="transaction_id" type="string">
  The unique identifier for the transaction.
</ParamField>


<ParamField path="method" type="string">
  The payment method.
</ParamField>


<ParamField path="cardnumber" type="string">
  The card number.
</ParamField>


<ParamField path="cvv" type="string">
  The CVV.
</ParamField>


<ParamField path="postal_code" type="string">
  The postal code.
</ParamField>


<ParamField path="description" type="string">
  The description.
</ParamField>


<ParamField path="chargeAmount" type="string">
  The charge amount.
</ParamField>


<ParamField path="token_type" type="string">
  The token type.
</ParamField>


<ParamField path="expiry_month" type="string">
  The expiry month.
</ParamField>


<ParamField path="expiry_year" type="string">
  The expiry year.
</ParamField>


<ParamField path="currency_code" type="string">
  The currency code.
</ParamField>


### Request format

The request body is a JSON object containing the payment details.

```json title="Request Body Example"
{
  "transaction_id": "8c9d14d5-52ae-4e2e-b880-a14e6e1cda7d",
  "method": "credit-card",
  "cardnumber": "************1234",
  "cvv": "***",
  "postal_code": "123456",
  "description": "Payment description",
  "chargeAmount": "10.55",
  "token_type": "reusable",
  "expiry_month": "12",
  "expiry_year": "99",
  "currency_code": "usd"
}
```

## Response format

Your payment connector endpoint should respond with specific formats for both successful and failed transactions. 
The response format varies depending on whether you are processing a charge transaction or generating a token.

### Successful response

<Tabs>
<Tab title="Successful standard card charge">

For a successful charge transaction, return a `200` HTTP status code with a JSON response containing:

<ParamField path="charge_id" type="string">
  A unique identifier for the successful transaction.
</ParamField>


<ParamField path="error_code" type="null">
  Must be null for successful transactions.
</ParamField>


<ParamField path="error_message" type="null">
  Must be null for successful transactions.
</ParamField>


```json
{
  "charge_id": "ch_123456789",
  "error_code": null,
  "error_message": null
}
```

</Tab>
<Tab title="Successful tokenized card payment">

For a successful tokenization, return a `200` HTTP status code with a JSON response containing:

<ParamField path="token_id" type="string">
  A unique identifier for the stored payment method.
</ParamField>


<ParamField path="error_code" type="null">
  Must be null for successful transactions.
</ParamField>


<ParamField path="error_message" type="null">
  Must be null for successful transactions.
</ParamField>


```json
{
  "token_id": "tok_123456789",
  "error_code": null,
  "error_message": null
}
```

</Tab>
</Tabs>

### Error response

<Note>
The error response will trigger the corresponding [`payment-failed`][payment-step] prompt in your SWML application.
</Note>

<Tabs>
<Tab title="Failed standard card charge">

For failed charge transactions, return a non-200 HTTP status code with a JSON response containing:

```json
{
  "charge_id": null,
  "error_code": "insufficient_funds",
  "error_message": "Card has insufficient funds"
}
```

<ParamField path="charge_id" type="null">
  Must be null for failed transactions.
</ParamField>


<ParamField path="error_code" type="string">
  An error code identifying the type of failure.
</ParamField>


<ParamField path="error_message" type="string">
  A human-readable description of the error.
</ParamField>


</Tab>
<Tab title="Failed tokenized card payment">

For failed tokenization, return a non-200 HTTP status code with a JSON response containing:

```json
{
  "token_id": null,
  "error_code": "invalid_card",
  "error_message": "Card validation failed"
}
```

<ParamField path="token_id" type="null">
  Must be null for failed transactions.
</ParamField>


<ParamField path="error_code" type="string">
  An error code identifying the type of failure.
</ParamField>


<ParamField path="error_message" type="string">
  A human-readable description of the error.
</ParamField>


</Tab>
</Tabs>
