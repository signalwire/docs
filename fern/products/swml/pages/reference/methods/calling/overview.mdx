Calling SWML is the flavor of SWML used to handle voice calls — inbound calls arriving on a phone
number configured with a SWML calling handler, outbound REST-initiated calls that point at a SWML
URL, and re-fetches triggered by methods that hit an external URL.

For handling inbound SMS and MMS messages, see the
[Messaging SWML overview](/docs/swml/reference/messaging).

## Document structure

A Calling SWML document is a top-level `sections` map with named sections. Execution starts
at `sections.main`, and each section contains an array of methods that run sequentially.

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - answer: {}
    - play:
        url: "say:Hello!"
    - hangup: {}
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      { "answer": {} },
      { "play": { "url": "say:Hello!" } },
      { "hangup": {} }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>

### SWML object properties

<ParamField path="version" type="string" toc={true}>
  The SWML schema version. Currently only `"1.0.0"` is accepted. Optional — defaults to the
  current schema version when omitted.
</ParamField>

<ParamField path="sections" type="object" required={true} toc={true}>
  A map of named sections. Each value is an array of [methods](#methods) that run sequentially.
  Must contain at least a `main` section; additional named sections can be invoked via
  [`execute`](/docs/swml/reference/calling/execute),
  [`transfer`](/docs/swml/reference/calling/transfer), or
  [`goto`](/docs/swml/reference/calling/goto).
</ParamField>

<Indent>

<ParamField path="sections.main" type="array" required={true} toc={true}>
  The entry-point section. Execution starts here when the document loads. Each item in the
  array is one of the [methods](#methods) listed below.
</ParamField>


<ParamField path="sections.<name>" type="array" toc={true}>
  Any additional named section, invokable via `execute`, `transfer`, or `goto`. Each item in
  the array is one of the [methods](#methods) listed below.
</ParamField>

</Indent>

## Document-fetching webhook [#document-fetching-webhook]

When SignalWire fetches a Calling SWML document from an external URL, it sends a POST request
with the [inbound call webhook payload](/docs/apis/rest/swml-webhook/webhooks/inbound-call-webhook).
This applies on the initial inbound fetch and on every fetch triggered by methods that hit an
external URL — `execute` with a remote URL, [`transfer`](/docs/swml/reference/calling/transfer),
[`join_conference.wait_url`](/docs/swml/reference/calling/join-conference#properties),
[`enter_queue.wait_url`](/docs/swml/reference/calling/enter-queue#properties), and
[`connect.confirm`](/docs/swml/reference/calling/connect#properties).

<WebhookPayloadSnippet webhook="inboundCallWebhook" />

The payload includes:

- **`call`** — call state and routing data (call ID, from/to, type, headers, project/space IDs, etc.).
- **`vars`** — script variables set by prior SWML execution (via [`set`](/docs/swml/reference/calling/set)
  or method outputs). Empty on the initial inbound fetch; populated on subsequent fetches.
- **`params`** — parameters explicitly passed by the calling step (`execute.params` or
  `transfer.params`). Empty on the initial inbound fetch.
- **`envs`** — environment variables configured at the account or project level.

Your server must respond with a valid SWML document using one of these content types:
`application/json`, `application/yaml`, or `text/x-yaml`.

## Variable expansion

Calling SWML supports two substitution syntaxes in string values:

- **`${...}`** — JavaScript expressions evaluated by V8. Use for transformations, calculations,
  and conditional logic. See [Expressions](/docs/swml/reference/expressions).
- **`%{path}`** — pure path-based substitution. See [Variables](/docs/swml/reference/variables).

## Methods

A method is a single step in a SWML document — each item in a `sections.<name>` array invokes
one method. Each method's reference page documents its parameters, defaults, and any output
variables it sets. Methods below are grouped by what they do.
