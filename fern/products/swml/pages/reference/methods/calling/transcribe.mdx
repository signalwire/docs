[statuscallbacks]: #statuscallbacks

Start transcribing the call in the background. Execution continues to the next instruction while transcription runs. A recording of the call is produced, and its location is returned so you can retrieve the transcription. To stop it, use the [`transcribe_stop`](/docs/swml/reference/calling/transcribe-stop) method.

Only one transcription can be active on a call at a time. Starting another while one is already running will fail.

## **Properties**

<ParamField path="transcribe" type="object" toc={true}>
  An object that accepts the following properties.
</ParamField>

<Indent>

<ParamField path="transcribe.status_url" type="string" toc={true}>
  HTTP or HTTPS URL to deliver transcription status events. Learn more about [status callbacks][statuscallbacks].
</ParamField>

</Indent>

## **Variables**

Set by the method:

- **transcribe_result:** (out) `success` | `failed`.
- **transcribe_control_id:** (out) Identifier assigned to this transcription. `transcribe_stop` uses it to stop the right one.

## **StatusCallbacks**

When you set `status_url`, SignalWire sends a `calling.call.transcribe` event there twice: once with `state` set to `transcribing` when transcription starts, and once with `state` set to `finished` when it ends. The `finished` event also includes `duration`, `size`, `start_time`, and `end_time`.

<WebhookPayloadSnippet webhook="transcribeStatusCallback" />

See the [Transcribe status callback](/docs/apis/rest/calls/webhooks/transcribe-status-callback) webhook page for the full field reference.

## **Examples**

### Start a transcription

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - transcribe:
        status_url: https://example.com/transcribe-status
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "transcribe": {
          "status_url": "https://example.com/transcribe-status"
        }
      }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>

### Start a transcription with no parameters

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - transcribe: {}
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "transcribe": {}
      }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>
