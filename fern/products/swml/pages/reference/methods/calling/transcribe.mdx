[statuscallbacks]: #statuscallbacks

Start transcribing the call in the background. Execution continues to the next instruction while transcription runs. A recording of the call is produced, and its location is returned so you can retrieve the transcription. To stop it, use the [`transcribe_stop`](/docs/swml/reference/calling/transcribe-stop) method.

Only one transcription can be active on a call at a time. Starting another while one is already running will fail.

## **Properties**

<ParamField path="transcribe" type="object" toc={true}>
  An object that accepts the following properties. `transcribe` may also be invoked as a bare string with no parameters.
</ParamField>

<Indent>

<ParamField path="transcribe.status_url" type="string" toc={true}>
  HTTP or HTTPS URL to deliver transcription status events. Learn more about [status callbacks][statuscallbacks].
</ParamField>

</Indent>

## **Variables**

Set by the method:

- **transcribe_result:** (out) `success` | `failed`.
- **transcribe_control_id:** (out) Identifier assigned to this transcription. `transcribe_stop` uses it to stop the right one.

## **StatusCallbacks**

If a `status_url` is provided, a POST request will be sent to it with a JSON payload like the following:

<ParamField path="event_type" type="string" toc={true}>
  The type of event. Always `calling.call.transcribe` for this method.
</ParamField>


<ParamField path="event_channel" type="string" toc={true}>
  The channel for the event, includes the SWML session ID.
</ParamField>


<ParamField path="timestamp" type="number" toc={true}>
  Unix timestamp (float) when the event was generated.
</ParamField>


<ParamField path="project_id" type="string" toc={true}>
  The project ID associated with the call.
</ParamField>


<ParamField path="space_id" type="string" toc={true}>
  The Space ID associated with the call.
</ParamField>


<ParamField path="params" type="object" toc={true}>
  An object containing transcription-specific parameters.
</ParamField>

<Indent>

<ParamField path="params.call_id" type="string" toc={true}>
  The call ID.
</ParamField>


<ParamField path="params.node_id" type="string" toc={true}>
  The node handling the call.
</ParamField>


<ParamField path="params.control_id" type="string" toc={true}>
  The control ID for this transcription operation.
</ParamField>


<ParamField path="params.state" type="string" toc={true}>
  The current transcription state. **Valid values:** `transcribing`, `finished`.
</ParamField>


<ParamField path="params.url" type="string" toc={true}>
  URL where the call recording is hosted. Retrieve the transcription from here. Present when available.
</ParamField>


<ParamField path="params.recording_id" type="string" toc={true}>
  Identifier of the call recording. Present when available.
</ParamField>


<ParamField path="params.status_url" type="string" toc={true}>
  The callback URL the event was delivered to. Present when a `status_url` was set.
</ParamField>


<ParamField path="params.duration" type="number" toc={true}>
  Length of the recording in seconds. Present only when `state` is `finished`.
</ParamField>


<ParamField path="params.size" type="number" toc={true}>
  Size of the recording in bytes. Present only when `state` is `finished`.
</ParamField>


<ParamField path="params.start_time" type="number" toc={true}>
  Unix timestamp when the recording started. Present only when `state` is `finished`.
</ParamField>


<ParamField path="params.end_time" type="number" toc={true}>
  Unix timestamp when the recording ended. Present only when `state` is `finished`.
</ParamField>

</Indent>

## **Examples**

### Start a transcription

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - transcribe:
        status_url: https://example.com/transcribe-status
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "transcribe": {
          "status_url": "https://example.com/transcribe-status"
        }
      }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>

### Start a transcription with no parameters

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - transcribe
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      "transcribe"
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>
