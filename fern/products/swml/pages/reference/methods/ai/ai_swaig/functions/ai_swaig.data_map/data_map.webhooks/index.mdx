An array of objects that define external API calls.

<ParamField toc={true} path="data_map.webhooks" type="object[]" required={true}>
  An array of objects that accept the following properties.
</ParamField>


## **Properties**

<Warning title="Processing order">
The properties are processed in the following sequence:

1. `foreach`
2. `expressions` 
3. `output`

Only applicable when multiple of these properties are set in your configuration.
</Warning>

<ParamField toc={true} path="webhooks[].expressions" type="object">
  A list of expressions to be evaluated upon matching.
  
  See [`expressions`](/swml/methods/ai/swaig/functions/data_map/expressions) for additional details.
</ParamField>


<ParamField toc={true} path="webhooks[].error_keys" type="string | string[]">
  A string or array of strings that represent the keys to be used for error handling.
</ParamField>


<ParamField toc={true} path="webhooks[].url" type="string" required={true}>
  The endpoint for the external service or API. Authentication can also be set in the url in the format of `username:password@url`.
</ParamField>


<ParamField toc={true} path="webhooks[].foreach" type="object">
  Iterates over an array of objects and processes an output based on each element in the array. Works similarly to JavaScript's [forEach](https://www.w3schools.com/jsref/jsref_foreach.asp) method.
</ParamField>

This accepts the [`foreach properties`](./foreach.mdx#foreach-properties).

  See [`foreach`](/swml/methods/ai/swaig/functions/data_map/webhooks/foreach) for additional details.

<ParamField toc={true} path="webhooks[].headers" type="object">
  Any necessary headers for the API call.
</ParamField>


<ParamField toc={true} path="webhooks[].method" type="string" required={true}>
  The HTTP method (GET, POST, etc.) for the API call.
</ParamField>


<ParamField toc={true} path="webhooks[].input_args_as_params" type="boolean" default="false">
  A boolean to determine if the input [parameters](../../parameters/index.mdx) should be passed as parameters.
</ParamField>


<ParamField toc={true} path="webhooks[].params" type="object">
  An object of any necessary parameters for the API call. The key is the parameter name and the value is the parameter value.
</ParamField>


<ParamField toc={true} path="webhooks[].required_args" type="string | string[]">
  A string or array of strings that represent the [parameters](../../parameters/index.mdx) that are required to make the webhook request.
</ParamField>


<ParamField toc={true} path="webhooks[].output" type="object" required={true}>
  Defines the response or action to be taken when the webhook is successfully triggered.

  See [`output`](/swml/methods/ai/swaig/functions/data_map/output) for additional details.
</ParamField>


## **Runtime request**

When the AI triggers a function that uses `data_map.webhooks`, SignalWire sends a request to each configured `url`.

### URL substitution

The `url` field supports `%{args.<name>}` substitution, where `<name>` matches a property defined in the function's [`argument`](/swml/methods/ai/swaig/functions#properties) schema.
For example, `https://api.example.com/weather?city=%{args.location}` will substitute the value the AI extracted for `location`.

### Request body (POST)

When `method` is `POST`, the request body contains:

<ParamField path="call_id" type="string">
  The call ID for the current call.
</ParamField>


<ParamField path="vars" type="object">
  An object containing the function argument values extracted by the AI, keyed by argument name.
</ParamField>


### Raw JSON example

```json
{
  "call_id": "xxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "vars": {
    "location": "New York"
  }
}
```

### Response processing

The JSON response from the webhook is processed through the [`output`](/swml/methods/ai/swaig/functions/data_map/output) template.
Fields from the response can be referenced using `%{key}` syntax in the output's `response` string.
For example, if the webhook returns `{"temp": 72, "conditions": "sunny"}`, an output of
`"The weather is %{temp}°F with %{conditions}"` will produce `"The weather is 72°F with sunny"`.
