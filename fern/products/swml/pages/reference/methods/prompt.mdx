[statuscallbacks]: #statuscallbacks

Play a prompt and wait for input. The input can be received either as digits from the keypad, or from speech, or both depending on
what [properties](#properties) are set.

## **Properties**

<ParamField path="prompt" type="object" required={true}>
  An object that accepts the following properties.
</ParamField>

<Indent>

<ParamField path="prompt.play" type="string | string[]" required={true}>
  Either a [playable sound](#playable-sounds) or an array of [playable sounds](#playable-sounds)
</ParamField>


<ParamField path="prompt.volume" type="number" default="0">
  Volume gain to apply to played URLs. Allowed values from `-40.0` to `40.0`.
</ParamField>


<ParamField path="prompt.say_voice" type="string" default="Polly.Salli">
  Voice to use with `say:` for text to speech
</ParamField>


<ParamField path="prompt.say_language" type="string" default="en-US">
  Language to use with `say:` for text to speech
</ParamField>


<ParamField path="prompt.say_gender" type="string" default="female">
  Gender to use with `say:` for text to speech
</ParamField>


<ParamField path="prompt.max_digits" type="integer" default="1">
  Number of digits to collect
</ParamField>


<ParamField path="prompt.terminators" type="string">
  Digits that terminate digit collection
</ParamField>


<ParamField path="prompt.digit_timeout" type="number" default="5.0 seconds">
  Time in seconds to wait for next digit
</ParamField>


<ParamField path="prompt.initial_timeout" type="number" default="5.0 seconds">
  Time in seconds to wait for start of input
</ParamField>


<ParamField path="prompt.speech_timeout" type="number">
  Max time in seconds to wait for speech result
</ParamField>


<ParamField path="prompt.speech_end_timeout" type="number">
  Time in seconds to wait for end of speech utterance
</ParamField>


<ParamField path="prompt.speech_language" type="string">
  Language to detect speech in
</ParamField>


<ParamField path="prompt.speech_hints" type="string[]">
  Expected words to match
</ParamField>


<ParamField path="prompt.speech_engine" type="string">
  The engine selected for speech recognition. The engine must support the specified language. Valid values: `Google`, `Google.V2`, `Deepgram`. Default is not set (SignalWire picks the engine).
</ParamField>


<ParamField path="prompt.status_url" type="string">
  HTTP or HTTPS URL to deliver prompt status events. Learn more about [status callbacks][statuscallbacks].
</ParamField>

</Indent>

<Note>
By default, only digit input via keypad is enabled. When **at least one** speech input based parameter
is set (`speech_timeout`, `speech_end_timeout`, `speech_language` or `speech_hints`), speech input is
enabled and digit input is disabled.

To enable speech and digit based input collection at once, set at least one speech input parameter
and at least one digit input based parameter (`max_digits`, `terminators`, `digit_timeout`, and `initial_timeout`).
</Note>

## Variables

Read by the method:

- **say_voice:** (in) - optional voice to use for text to speech.
- **say_language:** (in) - optional language to use for text to speech.
- **say_gender:** (in) - optional gender to use for text to speech.

## Possible values for voice, language, and ringtone

<Markdown src="/snippets/swml/_voice-table.mdx" />

### Set by the method

- **prompt_result:** (out) - `failed`, `no_input`, `match_speech`, `match_digits`, or `no_match`.
- **prompt_value:** (out) - the digits or utterance collected.
- **prompt_digit_terminator:** (out) - digit terminator collected, if any.
- **prompt_speech_confidence:** (out) - speech confidence measured, if any.

## **StatusCallbacks**

A POST request will be sent to `status_url` with a JSON payload like the following:

<ParamField path="event_type" type="string">
  The type of event. Always `calling.call.collect` for this method.
</ParamField>


<ParamField path="event_channel" type="string">
  The channel for the event, includes the SWML session ID.
</ParamField>


<ParamField path="timestamp" type="number">
  Unix timestamp (float) when the event was generated.
</ParamField>


<ParamField path="project_id" type="string">
  The project ID associated with the call.
</ParamField>


<ParamField path="space_id" type="string">
  The Space ID associated with the call.
</ParamField>


<ParamField path="params" type="object">
  An object containing prompt-specific parameters.
</ParamField>

<Indent>

<ParamField path="params.call_id" type="string">
  The call ID.
</ParamField>


<ParamField path="params.node_id" type="string">
  The node handling the call.
</ParamField>


<ParamField path="params.control_id" type="string">
  The control ID for this prompt operation.
</ParamField>


<ParamField path="params.result" type="object">
  The collection result details.
</ParamField>

<Indent>

<ParamField path="result.type" type="string">
  The type of input collected. **Valid values:** `digit`, `speech`, `no_input`, `no_match`, `start_of_input`, `finished`, `error`.
</ParamField>


<ParamField path="result.params.digits" type="string">
  The DTMF digits collected (when `type` is `digit`).
</ParamField>


<ParamField path="result.params.terminator" type="string">
  The terminator digit that ended collection (when `type` is `digit`).
</ParamField>


<ParamField path="result.params.text" type="string">
  The recognized speech text (when `type` is `speech`).
</ParamField>


<ParamField path="result.params.confidence" type="number">
  The speech recognition confidence score (when `type` is `speech`).
</ParamField>

</Indent>

<ParamField path="params.final" type="boolean">
  Whether this is the final result in a continuous collect session. Only present when `partial_results` is enabled. `true` indicates collection has ended; `false` indicates more results may follow.
</ParamField>


<ParamField path="params.state" type="string">
  The current collection state. Only present when continuous collect is enabled. **Valid values:** `collecting`, `finished`, `error`.
</ParamField>

</Indent>

## **Examples**

The [`play` method](/swml/methods/play) also has examples related to playing sounds from URLs. The
interface for playing sounds for `play` and `prompt` is identical.

### Play prompt and wait for digit press

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - prompt:
        play: 'say:Input a number'
    - switch:
        variable: prompt_value
        default:
          - play:
              url: 'say:You didn''t press one'
          - transfer:
              dest: main
        case:
          '1':
            - play:
                url: 'say:You pressed one'
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "prompt": {
          "play": "say:Input a number"
        }
      },
      {
        "switch": {
          "variable": "prompt_value",
          "default": [
            {
              "play": {
                "url": "say:You didn't press one"
              }
            },
            {
              "transfer": {
                "dest": "main"
              }
            }
          ],
          "case": {
            "1": [
              {
                "play": {
                  "url": "say:You pressed one"
                }
              }
            ]
          }
        }
      }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>

### Using terminators

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - prompt:
        play: 'say:PIN number please'
        max_digits: 10
        terminators: '*#5'
    - play:
        url: 'say: ${prompt_value} was terminated by ${prompt_digit_terminator}'
```
</CodeBlock>
<CodeBlock title="JSON">
```json

  "sections": {
    "main": [
      {
        "prompt": {

        }
      },
      {
        "play": {
          "url": "say: ${prompt_value} was terminated by ${prompt_digit_terminator}"
```

### Play prompt and collect digits, then pass the data to an external action

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - prompt:
        play: 'https://example.com/menu.wav'
    - transfer:
        dest: 'https://example.com/post_next_menu'
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "prompt": {
          "play": "https://example.com/menu.wav"
        }
      },
      {
        "transfer": {
          "dest": "https://example.com/post_next_menu"
        }
      }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>

In this case, the URL listed in transfer will be sent an HTTP POST request
with all the [out variables](#set-by-the-method) (like `prompt_value`) already set. For
more details on this behavior, refer to [`transfer`](/swml/methods/transfer)
statement's documentation.
