Play file(s), ringtones, speech or silence.

## **Properties**

<ParamField path="play" type="object" required={true} toc={true}>
  An object that accepts the following properties. Accepts either a single URL or multiple URLs. See [audio source](#audio-source) below.
</ParamField>

<Indent>

<ParamField path="play.status_url" type="string" toc={true}>
  HTTP or HTTPS URL to deliver play status events. Learn more about [status callbacks][statuscallbacks].
</ParamField>

</Indent>

## **Audio source**

The `play` object accepts one of the following properties to specify the audio source:

<Tabs>
<Tab title="Single URL (url)">

<ParamField path="play.url" type="string" required={true} toc={true}>
  A single [playable sound](#playable-sounds). Authentication can also be set in the url in the format of `username:password@url`.
</ParamField>

</Tab>
<Tab title="Multiple URLs (urls)">

<ParamField path="play.urls" type="string[]" required={true} toc={true}>
  An array of [playable sounds](#playable-sounds). Authentication can also be set in the url in the format of `username:password@url`.
</ParamField>

</Tab>
</Tabs>

## Variables

Read by the method:

- **`say_voice:`** (in) - Optional voice to use for text to speech.
- **`say_language:`** (in) - Optional language to use for text to speech.
- **`say_gender:`** (in) - Optional gender to use for text to speech.

## **StatusCallbacks**

A POST request will be sent to `status_url` with a JSON payload like the following:

<ParamField path="event_type" type="string" toc={true}>
  The type of event. Always `calling.call.play` for this method.
</ParamField>


<ParamField path="event_channel" type="string" toc={true}>
  The channel for the event, includes the SWML session ID.
</ParamField>


<ParamField path="timestamp" type="number" toc={true}>
  Unix timestamp (float) when the event was generated.
</ParamField>


<ParamField path="project_id" type="string" toc={true}>
  The project ID associated with the call.
</ParamField>


<ParamField path="space_id" type="string" toc={true}>
  The Space ID associated with the call.
</ParamField>


<ParamField path="params" type="object" toc={true}>
  An object containing playback-specific parameters.
</ParamField>

<Indent>

<ParamField path="params.call_id" type="string" toc={true}>
  The call ID.
</ParamField>


<ParamField path="params.node_id" type="string" toc={true}>
  The node handling the call.
</ParamField>


<ParamField path="params.control_id" type="string" toc={true}>
  The control ID for this play operation.
</ParamField>


<ParamField path="params.state" type="string" toc={true}>
  The current playback state. **Valid values:** `playing`, `paused`, `finished`, `error`.
</ParamField>

</Indent>

### Raw JSON example

```json
  "event_type": "calling.call.play",

  }

```

## Examples

### Playing a single URL

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - play:
        url: 'https://cdn.signalwire.com/swml/audio.mp3'
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "play": {
          "url": "https://cdn.signalwire.com/swml/audio.mp3"
        }
      }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>

### Playing multiple URLs

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - play:
        urls:
          - 'https://cdn.signalwire.com/swml/audio.mp3'
          - 'say: this is something to say'
          - 'silence: 3.0'
          - 'ring:10.0:us'
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "play": {
          "urls": [
            "https://cdn.signalwire.com/swml/audio.mp3",
            "say: this is something to say",
            "silence: 3.0",
            "ring:10.0:us"
          ]
        }
      }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>

### Specifying a voice to use for speaking

**Globally**

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - set:
        say_voice: gcloud.en-US-Neural2-A
    - play:
        url: 'say:Hi, do I sound different?'
    - play:
        url: 'say:I don''t, do I?'
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "set": {
          "say_voice": "gcloud.en-US-Neural2-A"
        }
      },
      {
        "play": {
          "url": "say:Hi, do I sound different?"
        }
      },
      {
        "play": {
          "url": "say:I don't, do I?"
        }
      }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>
