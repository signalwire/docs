The `parameters` object is used to define the input data that will be passed to the function.

**functions[].parameters** (type: object)
  An object that contains the [`parameters`](#parameters)

## Parameters

The `parameters` object defines the function parameters that will be passed to the AI.

<APIField
  name="parameters.type"
  type="string"
  required={true}
>
  Defines the top-level type of the parameters. Must be set to `"object"`

<APIField
  name="parameters.properties"
  type="object"
  required={true}
>
  An object containing the [properties](#properties) definitions to be passed to the function

**parameters.required** (type: string[])
  Array of required property names from the `properties` object

## Properties

The `properties` object defines the input data that will be passed to the function. It supports different types of parameters, each with their own set of configuration options.
The property name is a key in the `properties` object that is user-defined.

<APIField
  name="properties.{property_name}"
  type="object"
  required={true}
>
  An object with dynamic property names, where:<ul><li>**Keys:** User-defined strings, that set the property name.</li><li>**Values:** Must be one of the valid [schema types](#schema-types). Learn more about valid schema types from the [JSON Schema documentation](https://json-schema.org/understanding-json-schema/reference)</li></ul>

<h3>String Properties</h3>

  The type of property the AI is passing to the function. Must be set to `"string"`

**{property_name}.description** (type: string)
  A description of the property

**{property_name}.enum** (type: string[])
  An array of strings that are the possible values

**{property_name}.default** (type: string)
  The default string value

**{property_name}.pattern** (type: string)
  Regular expression pattern for the string value to match

<APIField name="{property_name}.nullable" type="boolean" default="false">
  Whether the property can be null

#### Example

```yaml andJSON

  type: object
  properties:
    property_name:
      type: string
      description: A string value
      pattern: ^[a-z]+$
```

<Tab title="array">

<h3>Array Properties</h3>

<APIField name="{property_name}.type" type="string" required={true}>
  The type of parameter(s) the AI is passing to the function. Must be set to `"array"`

**{property_name}.description** (type: string)
  A description of the property

<APIField name="{property_name}.items" type="object" required={true}>
  An array of items. These items must be one of the valid [schema types](#schema-types)

**{property_name}.default** (type: array)
  The default array value

<APIField name="{property_name}.nullable" type="boolean" default="false">
  Whether the property can be null

#### Example

```yaml andJSON
parameters:
  type: object
  properties:
    property_name:
      type: array
      description: Array of strings and objects
      items:
        - type: string
          description: A single string value that must be one of the possible values
          default: "one"
          enum:
            - "one"
            - "two"
            - "three"
        - type: object
          description: An object with a required property, `desired_value`, that must be one of the possible values
          required:
            - desired_value
          properties:
            desired_value:
              type: string
              description: A single string value that must be one of the possible values
              enum:
                - "four"
                - "five"
                - "six"
```

<Tab title="object">

<h3>Object Properties</h3>

<APIField name="{property_name}.type" type="string" required={true}>
  The type of parameter(s) the AI is passing to the function. Must be set to `"object"`

**{property_name}.description** (type: string)
  A description of the property

**{property_name}.properties** (type: object)
  An object that contains the [properties](#properties) definitions to be passed to the function.

**{property_name}.required** (type: string[])
  The property names that are required for the `properties` object

**{property_name}.default** (type: object)
  The default object value

<APIField name="{property_name}.nullable" type="boolean" default="false">
  Whether the property can be null

#### Example

```yaml andJSON
parameters:
  type: object
  properties:
    name:
      type: string
      description: The user's name
    age:
      type: integer
      description: The user's age
    address:
      type: object
      description: The user's address
      properties:
        street:
          type: string
          description: The user's street address
        city:
          type: string
          description: The user's city
      required:
        - street
        - city
```

</Tab>
