The URL that the user defines to which to send status callbacks and reports.

<ParamField path="amazon_bedrock.post_prompt_url" type="string">
  The URL to which to send status callbacks and reports. Authentication can also be set in the url in the format of `username:password@url`.
</ParamField>


## **Request Parameters for `post_prompt_url`**

SignalWire will make a request to the `post_prompt_url` with the following parameters:

<ParamField path="action" type="string">
  Action that prompted this request. The value will be "post_conversation".
</ParamField>


<ParamField path="ai_end_date" type="integer">
  Timestamp indicating when the AI session ended.
</ParamField>


<ParamField path="ai_session_id" type="string">
  A unique identifier for the AI session.
</ParamField>


<ParamField path="ai_start_date" type="integer">
  Timestamp indicating when the AI session started.
</ParamField>


<ParamField path="app_name" type="string">
  Name of the application that originated the request.
</ParamField>


<ParamField path="call_answer_date" type="integer">
  Timestamp indicating when the call was answered.
</ParamField>


<ParamField path="call_end_date" type="integer">
  Timestamp indicating when the call ended.
</ParamField>


<ParamField path="call_id" type="string">
  ID of the call.
</ParamField>


<ParamField path="call_log" type="object">
  The complete log of the call, as a JSON object.
</ParamField>


<ParamField path="call_log.content" type="string">
  Content of the call log entry.
</ParamField>


<ParamField path="call_log.role" type="string">
  Role associated with the call log entry (e.g., "system", "assistant", "user").
</ParamField>


<ParamField path="call_start_date" type="integer">
  Timestamp indicating when the call started.
</ParamField>


<ParamField path="caller_id_name" type="string">
  Name associated with the caller ID.
</ParamField>


<ParamField path="caller_id_number" type="string">
  Number associated with the caller ID.
</ParamField>


<ParamField path="content_disposition" type="string">
  Disposition of the content.
</ParamField>


<ParamField path="content_type" type="string">
  Type of content. The value will be `text/swaig`.
</ParamField>


<ParamField path="post_prompt_data" type="object">
  The answer from the AI agent to the `post_prompt`. The object contains the three following fields.
</ParamField>


<ParamField path="post_prompt_data.parsed" type="object">
  If a JSON object is detected within the answer, it is parsed and provided here.
</ParamField>


<ParamField path="post_prompt_data.raw" type="string">
  The raw data answer from the AI agent.
</ParamField>


<ParamField path="post_prompt_data.substituted" type="string">
  The answer from the AI agent, excluding any JSON.
</ParamField>


<ParamField path="project_id" type="string">
  ID of the Project.
</ParamField>


<ParamField path="space_id" type="string">
  ID of the Space.
</ParamField>


<ParamField path="SWMLVars" type="object">
  A collection of variables related to SWML.
</ParamField>


<ParamField path="swaig_log" type="object">
  A log related to SWAIG functions.
</ParamField>


<ParamField path="total_input_tokens" type="integer">
  Represents the total number of input tokens.
</ParamField>


<ParamField path="total_output_tokens" type="integer">
  Represents the total number of output tokens.
</ParamField>


<ParamField path="version" type="string">
  Version number.
</ParamField>

### Responding to Post Prompt Requests

The response to the callback request should be a JSON object with the following parameters:

```json
{
  "response": "ok"
}
```
