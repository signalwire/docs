[statuscallbacks]: #statuscallbacks

Record call in the background. Unlike the [`record` method](./record.mdx), the `record_call` method
will start the recording and continue executing the SWML script while allowing the recording to happen in the background.
To stop call recordings started with `record_call`, use the [`stop_call_record`](./stop_record_call.mdx) method.

<ParamField path="record_call" type="object" required={true}>
  An object that accepts the following properties.
</ParamField>


## **Properties**

<ParamField path="record_call.control_id" type="string" default="Auto-generated, saved to record_control_id variable">
  Identifier for this recording, to use with [`stop_record_call`](./stop_record_call.mdx)
</ParamField>


<ParamField path="record_call.stereo" type="boolean" default="false">
  Whether to record in stereo mode
</ParamField>


<ParamField path="record_call.format" type="string" default="wav">
  Format (`"wav"`, `"mp3"`, or `"mp4"`)
</ParamField>


<ParamField path="record_call.direction" type="string" default="both">
  Direction of the audio to record: `"speak"` for what party says, `"listen"` for what party hears, `"both"` for what the party hears and says
</ParamField>


<ParamField path="record_call.terminators" type="string">
  String of digits that will stop the recording when pressed. Default is empty (no terminators).
</ParamField>


<ParamField path="record_call.beep" type="boolean" default="false">
  Whether to play a beep before recording
</ParamField>


<ParamField path="record_call.input_sensitivity" type="number" default="44.0">
  How sensitive the recording voice activity detector is to background noise? A larger value is more sensitive. Allowed values from `0.0` to `100.0`.
</ParamField>


<ParamField path="record_call.initial_timeout" type="number" default="0">
  How long, in seconds, to wait for speech to start?
</ParamField>


<ParamField path="record_call.end_silence_timeout" type="number" default="0">
  How much silence, in seconds, will end the recording?
</ParamField>


<ParamField path="record_call.max_length" type="number">
  Maximum length of the recording in seconds.
</ParamField>


<ParamField path="record_call.status_url" type="string">
  HTTP or HTTPS URL to deliver record status events. Learn more about [status callbacks][statuscallbacks].
</ParamField>


## **Variables**

Set by the method:

- **record_call_url:** (out) the URL of the newly started recording.
- **record_call_result:** (out) `success` | `failed`.
- **record_control_id:** (out) control ID of this recording.

## **StatusCallbacks**

A POST request will be sent to `status_url` with a JSON payload like the following:

<ParamField path="event_type" type="string">
  The type of event. Always `calling.call.record` for this method.
</ParamField>


<ParamField path="event_channel" type="string">
  The channel for the event, includes the SWML session ID.
</ParamField>


<ParamField path="timestamp" type="number">
  Unix timestamp (float) when the event was generated.
</ParamField>


<ParamField path="project_id" type="string">
  The project ID associated with the call.
</ParamField>


<ParamField path="space_id" type="string">
  The Space ID associated with the call.
</ParamField>


<ParamField path="params" type="object">
  An object containing recording-specific parameters.
</ParamField>


<ParamField path="params.call_id" type="string">
  The call ID.
</ParamField>


<ParamField path="params.node_id" type="string">
  The node handling the call.
</ParamField>


<ParamField path="params.control_id" type="string">
  The control ID for this record operation.
</ParamField>


<ParamField path="params.state" type="string">
  The current recording state. **Valid values:** `recording`, `paused`, `finished`, `no_input`, `error`.
</ParamField>


<ParamField path="params.record" type="object">
  Recording result details (present when state is `finished`).
</ParamField>


<ParamField path="params.record.url" type="string">
  URL to download the recording.
</ParamField>


<ParamField path="params.record.duration" type="number">
  Recording duration in seconds.
</ParamField>


<ParamField path="params.record.size" type="number">
  Recording file size in bytes.
</ParamField>


<ParamField path="params.record.format" type="string">
  Recording format. **Valid values:** `wav`, `mp3`.
</ParamField>

{
  "event_type": "calling.call.record",

## **Examples**

### Start an MP3 recording of the call

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - record_call:
        format: mp3
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "record_call": {
          "format": "mp3"
        }
      }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>

### Record and play back

#### Record both sides of the conversation:

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - record_call:
        beep: true
        terminators: '#'
    - play:
        urls:
          - 'say:Leave your message now'
          - 'silence:10'
    - stop_record_call: {}
    - play:
        urls:
          - 'say:Playing back'
          - '${record_call_url}'
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "record_call": {
          "beep": true,
          "terminators": "#"
        }
      },
      {
        "play": {
          "urls": [
            "say:Leave your message now",
            "silence:10"
          ]
        }
      },
      {
        "stop_record_call": {}
      },
      {
        "play": {
          "urls": [
            "say:Playing back",
            "${record_call_url}"
          ]
        }
      }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>
