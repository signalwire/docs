Join an ad-hoc audio conference started on either the SignalWire or Compatibility API.
This method allows you to connect the current call to a named conference where multiple participants can communicate simultaneously.

<ParamField toc={true} path="join_conference" type="object" required={true}>
  An object that accepts the following properties.
</ParamField>


## **Properties**

<ParamField toc={true} path="join_conference.name" type="string" required={true}>
  Name of conference.
</ParamField>


<ParamField toc={true} path="join_conference.muted" type="boolean" default="false">
  Whether to join the conference in a muted state. If set to `true`, the participant will be muted upon joining.
</ParamField>


<ParamField toc={true} path="join_conference.beep" type="string" default="true">
  Sets the behavior of the beep sound when joining or leaving the conference.
</ParamField>

**Possible Values**: `true`, `false`, `onEnter`, `onExit`

<ParamField toc={true} path="join_conference.start_on_enter" type="boolean" default="true">
  Starts the conference when the main participant joins. This means the start action will not wait on more participants to join before starting.
</ParamField>


<ParamField toc={true} path="join_conference.end_on_exit" type="boolean" default="false">
  Ends the conference when the main participant leaves. This means the end action will not wait on more participants to leave before ending.
</ParamField>


<ParamField toc={true} path="join_conference.wait_url" type="string">
  A URL to fetch SWML while waiting for the conference to start (before a `start_on_enter` participant joins). SignalWire sends a POST request with the standard [document-fetching webhook](/swml#document-fetching-webhook) body. The response should be a SWML document containing audio to play (e.g., hold music). Default hold music will be played if not set.
</ParamField>


<ParamField toc={true} path="join_conference.max_participants" type="integer" default="100000">
  The maximum number of participants allowed in the conference. If the limit is reached, new participants will not be able to join.
</ParamField>


<ParamField toc={true} path="join_conference.record" type="string" default="do-not-record">
  Enables or disables recording of the conference.
</ParamField>

**Possible Values**: `do-not-record`, `record-from-start`

<ParamField toc={true} path="join_conference.region" type="string">
  Specifies the geographical region where the conference will be hosted.
</ParamField>


<ParamField toc={true} path="join_conference.trim" type="string" default="trim-silence">
  If set to `trim-silence`, it will remove silence from the start of the recording. If set to `do-not-trim`, it will keep the silence.
</ParamField>

**Possible Values**: `trim-silence`, `do-not-trim`

<ParamField toc={true} path="join_conference.coach" type="string">
  Coach accepts a [call SID](/platform/dashboard/guides/what-is-a-sid) of a call that is currently connected to an in-progress conference. Specifying a call SID that does not exist or is no longer connected will result in a failure.
</ParamField>


<ParamField toc={true} path="join_conference.status_callback_event" type="string">
  The events to listen for and send to the status callback URL.
</ParamField>

**Possible Values**: `start`, `end`, `join`, `leave`, `mute`, `hold`, `modify`, `speaker`, `announcement`

<ParamField toc={true} path="join_conference.status_callback" type="string">
  The URL to which status events will be sent. This URL must be publicly accessible and able to handle HTTP requests. Learn more about [status callbacks][statuscallbacks].
</ParamField>


<ParamField toc={true} path="join_conference.status_callback_method" type="string" default="POST">
  The HTTP method to use when sending status events to the status callback URL.
</ParamField>

**Possible Values**: `GET`, `POST`

<ParamField toc={true} path="join_conference.recording_status_callback" type="string">
  The URL to which recording status events will be sent. This URL must be publicly accessible and able to handle HTTP requests. Learn more about [status callbacks][statuscallbacks].
</ParamField>


<ParamField toc={true} path="join_conference.recording_status_callback_method" type="string" default="POST">
  The HTTP method to use when sending recording status events to the recording status callback URL.
</ParamField>

**Possible Values**: `GET`, `POST`

<ParamField toc={true} path="join_conference.recording_status_callback_event" type="string">
  The events to listen for and send to the recording status callback URL.
</ParamField>

**Possible Values**: `in-progress`, `completed`, `absent`

<ParamField toc={true} path="join_conference.result" type="object">
  Allows the user to specify a custom action to be executed when the conference result is returned (typically when it has ended). The actions can a `switch` object or a `cond` array. The `switch` object allows for conditional execution based on the result of the conference, while the `cond` array allows for multiple conditions to be checked in sequence. If neither is provided, the default action will be to end the conference.
</ParamField>


## **Variables**

<ParamField toc={true} path="join_conference_result" type="string">
  The result of the conference join attempt. Possible values: `completed` (successfully joined and left the conference), `answered` (successfully joined the conference), `no-answer` (failed to join due to no answer), `failed` (failed to join due to an error), `canceled` (join attempt was canceled).
</ParamField>


<ParamField toc={true} path="return_value" type="string">
  Contains the same value as `join_conference_result` for use in conditional logic.
</ParamField>


## **StatusCallbacks**

### Conference status events (`status_callback`)

A POST request will be sent to `status_callback` with a JSON payload for events specified in `status_callback_event`.

<ParamField path="event_type" type="string">
  The type of event. Always `calling.call.conference` for conference status events.
</ParamField>


<ParamField path="event_channel" type="string">
  The channel for the event, includes the SWML session ID.
</ParamField>


<ParamField path="timestamp" type="number">
  Unix timestamp (float) when the event was generated.
</ParamField>


<ParamField path="project_id" type="string">
  The project ID associated with the call.
</ParamField>


<ParamField path="space_id" type="string">
  The Space ID associated with the call.
</ParamField>


<ParamField path="params" type="object">
  An object containing conference-specific parameters.
</ParamField>


<ParamField path="params.call_id" type="string">
  The call ID of the participant.
</ParamField>


<ParamField path="params.conference_name" type="string">
  The name of the conference.
</ParamField>


<ParamField path="params.conference_id" type="string">
  The unique ID of the conference.
</ParamField>


<ParamField path="params.conference_event" type="string">
  The conference event. **Valid values:** `start`, `end`, `join`, `leave`, `mute`, `hold`, `speaker`.
</ParamField>


<ParamField path="params.participant_count" type="number">
  The number of participants currently in the conference.
</ParamField>


#### Raw JSON example

```json
{
  "event_type": "calling.call.conference",
  "event_channel": "swml:xxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "timestamp": 1640000000.123,
  "project_id": "xxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "space_id": "xxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "params": {
    "call_id": "xxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "conference_name": "my_conference",
    "conference_id": "xxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "conference_event": "join",
    "participant_count": 3
  }
}
```

### Recording status events (`recording_status_callback`)

A POST request will be sent to `recording_status_callback` with a JSON payload for events specified in `recording_status_callback_event`.

<ParamField path="event_type" type="string">
  The type of event. Always `calling.call.conference.recording` for recording status events.
</ParamField>


<ParamField path="event_channel" type="string">
  The channel for the event, includes the SWML session ID.
</ParamField>


<ParamField path="timestamp" type="number">
  Unix timestamp (float) when the event was generated.
</ParamField>


<ParamField path="project_id" type="string">
  The project ID associated with the call.
</ParamField>


<ParamField path="space_id" type="string">
  The Space ID associated with the call.
</ParamField>


<ParamField path="params" type="object">
  An object containing recording-specific parameters.
</ParamField>


<ParamField path="params.conference_name" type="string">
  The name of the conference.
</ParamField>


<ParamField path="params.conference_id" type="string">
  The unique ID of the conference.
</ParamField>


<ParamField path="params.recording_state" type="string">
  The current recording state. **Valid values:** `in-progress`, `completed`, `absent`.
</ParamField>


<ParamField path="params.recording_url" type="string">
  URL to download the conference recording (present when state is `completed`).
</ParamField>


<ParamField path="params.duration" type="number">
  Recording duration in seconds.
</ParamField>


<ParamField path="params.size" type="number">
  Recording file size in bytes.
</ParamField>


#### Raw JSON example

```json
{
  "event_type": "calling.call.conference.recording",
  "event_channel": "swml:xxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "timestamp": 1640000000.123,
  "project_id": "xxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "space_id": "xxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "params": {
    "conference_name": "my_conference",
    "conference_id": "xxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "recording_state": "completed",
    "recording_url": "https://your-space.signalwire.com/api/v1/recordings/rec-uuid/download",
    "duration": 300,
    "size": 4800000
  }
}
```

## **Examples**

### Basic Conference Join

<CodeBlocks>
<CodeBlock title="YAML">
```yaml
version: 1.0.0
sections:
  main:
    - join_conference:
        name: "team_meeting"
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "join_conference": {
          "name": "team_meeting"
        }
      }
    ]
  }
}
```
</CodeBlock>
</CodeBlocks>

### Conference with Custom Settings

<CodeBlocks>
<CodeBlock title="YAML">
```yaml

sections:
  main:
    - join_conference:
        name: "team_meeting"
        muted: false
        beep: "onEnter"
        start_on_enter: true
        max_participants: 10
        record: "record-from-start"
```
</CodeBlock>
<CodeBlock title="JSON">
```json
{
  "version": "1.0.0",
  "sections": {
    "main": [
      {
        "join_conference": {
          "name": "team_meeting",
          "muted": false,
          "beep": "onEnter",
          "start_on_enter": true,
          "max_participants": 10,
          "record": "record-from-start"
```

</CodeBlock>
