SkillBase is the abstract base class for all agent skills. Skills are modular,
reusable capabilities -- such as weather lookup, web search, or calendar access --
that can be added to any [`AgentBase`][agentbase]
agent with a single call to `agent.add_skill()`.

Extend SkillBase to create custom skills. The SDK discovers skills automatically
from the `signalwire/skills/` directory and validates dependencies on load.

<Info>
For the catalog of 17+ built-in skills and their configuration parameters, see
the [Skills][skills] page.
</Info>

## **Class Attributes**

Define these on your subclass to configure the skill:

<ParamField path="SKILL_NAME" type="String" required={true} toc={true}>
  Unique identifier for the skill (e.g., `"weather"`, `"web_search"`). Used as
  the key when calling `agent.add_skill()`.
</ParamField>

<ParamField path="SKILL_DESCRIPTION" type="String" required={true} toc={true}>
  Human-readable description of the skill.
</ParamField>

<ParamField path="SKILL_VERSION" type="String" default="1.0.0" toc={true}>
  Semantic version string.
</ParamField>

<ParamField path="REQUIRED_PACKAGES" type="Array\<String\>" default="[]" toc={true}>
  Gems the skill needs. Checked on setup with `validate_packages()`.
</ParamField>

<ParamField path="REQUIRED_ENV_VARS" type="Array\<String\>" default="[]" toc={true}>
  Environment variables the skill needs. Checked on setup with `validate_env_vars()`.
</ParamField>

<ParamField path="SUPPORTS_MULTIPLE_INSTANCES" type="Boolean" default="false" toc={true}>
  When `true`, the same skill can be added to an agent multiple times with
  different configurations (distinguished by a `tool_name` parameter).
</ParamField>

## **Instance Properties**

<ParamField path="agent" type="AgentBase" toc={true}>
  Reference to the parent agent.
</ParamField>

<ParamField path="params" type="Hash" toc={true}>
  Configuration parameters (with `swaig_fields` removed).
</ParamField>

<ParamField path="logger" type="Logger" toc={true}>
  Skill-specific logger namespaced as `signalwire.skills.<SKILL_NAME>`.
</ParamField>

<ParamField path="swaig_fields" type="Hash" toc={true}>
  SWAIG metadata extracted from params, automatically merged into tool
  definitions when using `define_tool()`.
</ParamField>

## **Examples**

### Custom skill

```ruby

class WeatherSkill < SignalWire::Skills::SkillBase
    SKILL_NAME = "weather"
    SKILL_DESCRIPTION = "Provides weather information"
    SKILL_VERSION = "1.0.0"
    REQUIRED_PACKAGES = []
    REQUIRED_ENV_VARS = ["WEATHER_API_KEY"]

    def setup
        unless validate_packages
            return false
        unless validate_env_vars
            return false
        @api_key = ENV['WEATHER_API_KEY']
        return true

    def register_tools
        define_tool(
            name="get_weather",
            description="Get current weather for a location",
            handler=method(:_get_weather),
            parameters={
                "type": "object",
                "properties": {
                    "location": {
                        "type": "string",
                        "description": "City name or zip code"
                    }
                },
                "required": ["location"]
            }
        )

    def _get_weather(args, raw_data)
        # import requests
        location = args["location"]
        resp = Net::HTTP.get_response(URI(
            "https://api.weather.com/v1/current?q=#{location}&key=#{@api_key}"
        )
        data = JSON.parse(resp.body)
        return SignalWire::Swaig::FunctionResult.new(
            "Weather in #{location}: #{data['condition']}, #{data['temp']}F"
        )

    def get_hints
        return ["weather", "temperature", "forecast"]

    def _get_prompt_sections
        return [{
            "title": "Weather Information",
            "body": "You can check weather for any location."
        }]

    def @get_parameter_schema
        schema = super
        schema.merge!({
            "units": {
                "type": "string",
                "description": "Temperature units",
                "default": "fahrenheit",
                "enum": ["fahrenheit", "celsius"]
            }
        })
        return schema
```

### Using a skill

```ruby
require 'signalwire'

agent = SignalWire::AgentBase.new(name: "weather-agent")
agent.set_prompt_text("You are a helpful assistant.")
agent.add_skill("weather")

# Or with custom configuration
agent.add_skill("weather", {"units": "celsius"})

agent.run()
```
