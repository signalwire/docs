Programmatically define a SWAIG function (tool) that the AI can invoke during a
conversation. This is the imperative alternative to the
[`define_tool` method](#define-tool).

<Info>
Tool definitions map to SWML [SWAIG function][swaig-function]
entries. See the [SWML SWAIG functions reference][swml-swaig-functions-reference]
for the full specification.
</Info>

<Tip>
For most cases, the [`define_tool` method](#define-tool)
is simpler and supports automatic parameter inference from parameter types. Use
`define_tool()` when you need dynamic tool registration or when the tool definition
comes from external configuration.
</Tip>

## **Parameters**

<ParamField path="name" type="String" required={true} toc={true}>
  Function name. Must be unique within the agent. The AI uses this name to invoke the
  function.
</ParamField>

<ParamField path="description" type="String" required={true} toc={true}>
  Human-readable description of what the function does. The AI reads this to decide
  when to call the function.
</ParamField>

<ParamField path="parameters" type="Hash" required={true} toc={true}>
  JSON Schema describing the function's parameters. The AI generates arguments
  conforming to this schema.
</ParamField>

<ParamField path="handler" type="Proc" required={true} toc={true}>
  Ruby method or Proc to call when the AI invokes this tool. Receives `(args, raw_data)`
  and should return a [`FunctionResult`][functionresult].
</ParamField>

<ParamField path="secure" type="Boolean" default="true" toc={true}>
  Whether to require token validation on tool calls. Recommended for production.
</ParamField>

<ParamField path="fillers" type="Hash, nil" toc={true}>
  Language-specific filler phrases spoken while the function executes.
  Format: `{"en-US": ["Looking that up...", "One moment..."]}`.
</ParamField>

<ParamField path="webhook_url" type="String, nil" toc={true}>
  External URL to forward the tool call to instead of executing locally.
</ParamField>

<ParamField path="required" type="Array\<String\>, nil" toc={true}>
  List of required parameter names from the JSON Schema.
</ParamField>


<ParamField path="**swaig_fields" type="Object" toc={true}>
  Additional SWAIG fields to include in the function definition
  (e.g., `wait_file`, `meta_data`).
</ParamField>

## **Returns**

[`AgentBase`][ref-agentbase] -- Returns self for method chaining.

## **Example**

```ruby {11}
require 'signalwire'

agent = SignalWire::AgentBase.new(name: "weather-agent", route: "/weather")
agent.set_prompt_text("You are a helpful assistant.")

def get_weather(args, raw_data: nil)
    city = args["city"] || "unknown"
    return SignalWire::Swaig::FunctionResult.new("The weather in #{city} is 72F and sunny.")

agent.define_tool(
    name="get_weather",
    description="Get the current weather for a city",
    parameters={
        "type": "object",
        "properties": {
            "city": {
                "type": "string",
                "description": "City name"
            }
        }
    },
    handler=get_weather,
    required=["city"],
    fillers={"en-US": ["Checking the weather...", "Let me look that up..."]}
)
agent.serve()
```
