Register a callback for debug events received at the `/debug_events` endpoint.
Register a debug event handler. Both sync and async handlers are supported.

<Note>
[`enable_debug_events()`][enable-debug-events] must be called before events will be delivered.
</Note>

## **Parameters**

<ParamField path="handler" type="Proc" required={true} toc={true}>
  Callback function with signature `(event_type: str, data: dict)`.

  - `event_type` -- Event label string (e.g., `"barge"`, `"llm_error"`, `"session_start"`, `"step_change"`)
  - `data` -- Full event payload including `call_id`, `label`, and event-specific fields
</ParamField>

## **Returns**

`Proc` -- The handler function, unchanged. This allows `on_debug_event` to be used as a
method.

## **Example**

```ruby {7}
require 'signalwire'

agent = SignalWire::AgentBase.new(name: "debug-agent", route: "/debug")
agent.set_prompt_text("You are a helpful assistant.")
agent.enable_debug_events(level: 1)

@agent.on_debug_event
def handle_debug(event_type, data)
    if event_type == "llm_error"
        puts("LLM error: #{data}")
    elsif event_type == "barge"
        puts("Barge detected: #{data["barge_elapsed_ms"]}ms")
    elsif event_type == "step_change"
        puts("Step changed to: #{data["step"]}")

agent.serve()
```
