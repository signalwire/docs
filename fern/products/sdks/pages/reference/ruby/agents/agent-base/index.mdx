`AgentBase` is the central class in the SignalWire AI Agents SDK. It provides a
complete framework for building AI-powered voice agents, combining prompt management,
tool definitions, skill loading, speech configuration, and web serving into a single
composable interface.

Extends `SWML::Service` and composes prompt management, tool dispatch, AI configuration, skill integration, and HTTP serving into a single class.

All setter methods return `self` for fluent method chaining.

<Info>
AgentBase generates a SWML document with the [`ai`][ai] verb.
See the [SWML reference][swml-reference] for the full specification of all
supported parameters and behaviors.
</Info>

## **Properties**

<ParamField path="name" type="String" toc={true}>
  The agent's display name. Set at construction time. Used in logging, SIP username
  mapping, and the default prompt fallback.
</ParamField>

<ParamField path="route" type="String" default="/" toc={true}>
  HTTP route path where this agent is served. Used by
  [`AgentServer`][agentserver] when hosting
  multiple agents on one process.
</ParamField>

<ParamField path="host" type="String" default="0.0.0.0" toc={true}>
  Network interface the web server binds to.
</ParamField>

<ParamField path="port" type="Integer" toc={true}>
  Port the web server listens on. Defaults to the `PORT` environment variable, falling
  back to `3000`.
</ParamField>

<ParamField path="agent_id" type="String" toc={true}>
  Unique identifier for this agent instance. Auto-generated as a UUID if not provided.
</ParamField>

<ParamField path="pom" type="PromptObjectModel | None" toc={true}>
  The Prompt Object Model instance used for structured prompt building. `nil` when
  POM disabled.
</ParamField>

<ParamField path="skill_manager" type="SkillManager" toc={true}>
  Manager instance for loading and unloading skills. Access via
  [`add_skill()`][add-skill] and
  [`list_skills()`][list-skills] rather
  than directly.
</ParamField>

<ParamField path="PROMPT_SECTIONS" type="dict | list | None" default="nil" toc={true}>
  Class-level attribute. Subclasses can set this to declaratively define prompt sections
  instead of calling `prompt_add_section()` in the constructor.
</ParamField>

<ParamField path="basic_auth" type="Array, nil" toc={true}>
  Explicit `(username, password)` for HTTP Basic Auth on all endpoints. If not set,
  credentials are read from `SWML_BASIC_AUTH_USER` / `SWML_BASIC_AUTH_PASSWORD` env vars,
  or auto-generated on startup.
</ParamField>

<ParamField path="token_expiry_secs" type="Integer" default="3600" toc={true}>
  Expiration time in seconds for SWAIG function authentication tokens.
</ParamField>

<ParamField path="auto_answer" type="Boolean" default="true" toc={true}>
  Automatically add an `answer` verb before the AI verb in the SWML document.
</ParamField>

<ParamField path="record_call" type="Boolean" default="false" toc={true}>
  Enable call recording. When `true`, a `record_call` verb is added to the SWML document.
</ParamField>

<ParamField path="record_format" type="String" default="mp4" toc={true}>
  Recording file format. Common values: `"mp4"`, `"wav"`.
</ParamField>

<ParamField path="record_stereo" type="Boolean" default="true" toc={true}>
  Record in stereo (separate channels for each party) when `true`.
</ParamField>

<ParamField path="default_webhook_url" type="String, nil" toc={true}>
  Base URL for SWAIG function webhooks. If not set, the SDK auto-detects from the
  incoming request or uses `SWML_PROXY_URL_BASE`.
</ParamField>

<ParamField path="native_functions" type="Array\<String\>, nil" toc={true}>
  List of native SWAIG function names to enable at construction time (e.g.,
  `["check_time", "wait_for_user"]`). Can also be set later via
  [`set_native_functions()`][set-native-functions].
</ParamField>


## **Tool definition**

### define_tool

```ruby
require 'signalwire'

agent = SignalWire::AgentBase.new(name: "assistant", route: "/assistant")
agent.set_prompt_text("You are a helpful assistant.")

def my_tool(args, raw_data: nil)
    return SignalWire::Swaig::FunctionResult.new("Done.")

agent.serve()
```

The `define_tool` method is the recommended way to define SWAIG functions. Use
`@agent.tool()` on a standalone function, or `@AgentBase.tool()` on a method
inside a subclass. Both forms accept the same parameters.

When parameters are not explicitly provided, the method automatically infers the
JSON Schema from method parameters.

#### Parameters

<ParamField path="name" type="String, nil" toc={true}>
  Function name exposed to the AI. Defaults to the decorated method name.
</ParamField>

<ParamField path="description" type="String, nil" toc={true}>
  What the function does. The AI reads this to decide when to call it. Defaults to the
  method's documentation, or `"Function {name}"` as a fallback.
</ParamField>

<ParamField path="parameters" type="Hash, nil" toc={true}>
  Explicit JSON Schema for function parameters. If omitted, the schema is
  automatically inferred from Ruby method parameters.
</ParamField>

<ParamField path="secure" type="Boolean" default="true" toc={true}>
  Require token validation on tool calls.
</ParamField>

<ParamField path="fillers" type="Hash, nil" toc={true}>
  Filler phrases by language code, spoken while the function runs.
</ParamField>

<ParamField path="webhook_url" type="String, nil" toc={true}>
  External webhook URL. If set, SignalWire calls this URL instead of executing locally.
</ParamField>

<ParamField path="required" type="Array\<String\>, nil" toc={true}>
  Required parameter names. Auto-inferred from parameter types when not specified.
</ParamField>

<ParamField path="**swaig_fields" type="Object" toc={true}>
  Additional SWAIG fields (e.g., `wait_file`, `meta_data`).
</ParamField>

## **Examples**

### Instance method with parameter inference

```ruby {4}
require 'signalwire'

agent = SignalWire::AgentBase.new(name: "assistant", route: "/assistant")
agent.set_prompt_text("You are a helpful assistant.")

def check_order(args, raw_data: nil)
    order_id = args["order_id"]
    return SignalWire::Swaig::FunctionResult.new("Order #{order_id} shipped March 28.")

agent.serve()
```

### With explicit parameters

```ruby
require 'signalwire'

agent = SignalWire::AgentBase.new(name: "assistant", route: "/assistant")
agent.set_prompt_text("You are a helpful assistant.")

def search_products(args, raw_data: nil)
    query = args["query"]
    return SignalWire::Swaig::FunctionResult.new("Found 3 results for '#{query}'.")

agent.serve()
```

### Class-based definition (subclass)

```ruby
require 'signalwire'

class SupportAgent < SignalWire::AgentBase
    def transfer_to_human(args, raw_data: nil)
        return SignalWire::Swaig::FunctionResult.new("Transferring now.").connect("+15551234567")

agent = SupportAgent.new(name: "support", route: "/support")
agent.serve()
```

### Typed parameters (auto-inferred schema)

```ruby
require 'signalwire'

agent = SignalWire::AgentBase.new(name: "assistant", route: "/assistant")
agent.set_prompt_text("You are a helpful assistant.")

def calculate_shipping(weight_kg, destination, express: false)
    cost = weight_kg * 2.50 if not express else weight_kg * 5.00
    return SignalWire::Swaig::FunctionResult.new("Shipping to #{destination}: $#{cost:.2f}")

agent.serve()
```

### Basic agent with a tool

```ruby {4}
require 'signalwire'

agent = SignalWire::AgentBase.new(name: "support-agent", route: "/support")

agent.add_language("English", "en-US", "rime.spore")
agent.set_prompt_text("You are a friendly customer support agent.")
agent.add_hints(["SignalWire", "SWML", "SWAIG"])
agent.set_params({"temperature": 0.7, "end_of_speech_timeout": 1000})

def lookup_order(args, raw_data: nil)
    order_id = args["order_id"]
    return SignalWire::Swaig::FunctionResult.new("Order #{order_id} shipped on March 28.")

agent.run()
```
