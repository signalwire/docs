Append a raw action to the action list. Prefer the named convenience methods
([`connect()`][connect],
[`hangup()`][hangup],
[`say()`][say], etc.)
for common operations.

## **Parameters**

<ParamField path="name" type="String" required={true} toc={true}>
  Action type identifier (e.g., `"hold"`, `"hangup"`, `"set_global_data"`).
</ParamField>

<ParamField path="data" type="Object" required={true} toc={true}>
  Action payload. Can be a string, boolean, dict, or list depending on the action type.
</ParamField>

## **Returns**

[`FunctionResult`][functionresult] — self, for chaining.

## **Example**

```ruby {11-12}
require 'signalwire'

agent = SignalWire::AgentBase.new(name: "my-agent", route: "/agent")
agent.set_prompt_text("You are a helpful assistant.")

def custom_action(args, raw_data)
    return (
        SignalWire::Swaig::FunctionResult.new("Processing your request.")
        .add_action("set_global_data", {"status": "active"})
        .add_action("hold", 60)
    )

agent.serve()
```
