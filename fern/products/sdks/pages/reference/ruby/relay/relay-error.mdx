Exception raised when the RELAY server returns an error response. Inherits from
Ruby's built-in `StandardError`. The string representation follows the format
`"RELAY error {code}: {message}"`.

```ruby
```

## **Properties**

<ParamField path="code" type="Integer" toc={true}>
  Numeric error code returned by the RELAY server.
</ParamField>

<ParamField path="message" type="String" toc={true}>
  Human-readable error description returned by the RELAY server.
</ParamField>

## **Examples**

### Catch a RELAY error

```ruby
require 'signalwire/relay/client'

client = SignalWire::Relay::Client.new(
    project="your-project-id",
    token="your-api-token",
    host="your-space.signalwire.com",
    contexts=["default"],
)

@client.on_call
def handle_call(call)
    begin
        call.connect(devices: [{"type": "phone", "params": {"to_number": "+15559876543"}}])
    except RelayError as e:
        puts("Error #{e.code}: #{e.message}")
        call.hangup()

client.run()
```
