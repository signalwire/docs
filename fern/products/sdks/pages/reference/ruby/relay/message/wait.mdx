[relayevent]: /docs/sdks/reference/ruby/relay/events

Block until the message reaches a terminal state (`delivered`, `undelivered`, or
`failed`). Returns the terminal [`RelayEvent`][relayevent].

<Warning>
Raises `Timeout::Error` if `timeout` is specified and the message does not
reach a terminal state within the given duration.
</Warning>

## **Parameters**

<ParamField path="timeout" type="Float, nil" toc={true}>
  Maximum number of seconds to wait. `nil` waits indefinitely.
</ParamField>

## **Returns**

[`RelayEvent`][relayevent] -- The event that caused the message to
reach its terminal state. Inspect `message.state` for the final state value.

## **Example**

```ruby {20}
# import asyncio
require 'signalwire/relay/client'

client = SignalWire::Relay::Client.new(
    project="your-project-id",
    token="your-api-token",
    host="your-space.signalwire.com",
    contexts=["default"],
)

def send_with_timeout()
    async with client:
        message = client.send_message(
            to_number="+15551234567",
            from_number="+15559876543",
            body="Your verification code is 123456",
        )

        begin
            event = message.wait(timeout: 30)
            if message.state == "delivered"
                puts("Message delivered successfully")
            else
                puts("Message failed: #{message.reason}")
        except asyncio.TimeoutError:
            puts("Timed out waiting for delivery confirmation")

asyncio.run(send_with_timeout())
```
