Static factory that creates a SWAIG function definition using pattern-matching
expressions instead of webhooks.

## **Parameters**

<ParamField path="name" type="string" required={true} toc={true}>
  Function name.
</ParamField>

<ParamField path="purpose" type="string" required={true} toc={true}>
  Description of the function.
</ParamField>

<ParamField path="parameters" type="List&lt;Dictionary&lt;string, object&gt;&gt;" required={true} toc={true}>
  Parameter definitions.
</ParamField>

<ParamField path="expressions" type="List&lt;Dictionary&lt;string, object&gt;&gt;" required={true} toc={true}>
  Expressions with `string`, `pattern`, `output`, and optionally `nomatch_output`.
</ParamField>

## **Returns**

`Dictionary<string, object>` -- A complete SWAIG function definition.

## **Example**

```csharp
using SignalWire.DataMap;

var tool = DataMap.CreateExpressionTool(
    name: "route_department",
    purpose: "Route to the correct garden department",
    parameters: new List<Dictionary<string, object>>
    {
        new() { ["name"] = "department", ["type"] = "string", ["description"] = "Department name" }
    },
    expressions: new List<Dictionary<string, object>>
    {
        new()
        {
            ["string"] = "%{args.department}",
            ["pattern"] = "tropical|indoor",
            ["output"] = new Dictionary<string, object>
            {
                ["response"] = "Transferring to %{args.department} plants department"
            }
        }
    }
);
```
