`DocumentProcessor` handles document text extraction and chunking for search index
construction. It supports multiple file formats (PDF, DOCX, HTML, Markdown, Excel,
PowerPoint, RTF) and provides several chunking strategies optimized for different
content types and search use cases.

```python
from signalwire.search import DocumentProcessor
```

<Warning>
Full document processing requires additional dependencies. Install with
`pip install signalwire[search-full]` for PDF, DOCX, and other format support.
</Warning>

## **Properties**

</ParamField>

## **Constructor**

```python {1}
processor = DocumentProcessor(
    chunking_strategy="sentence",
    max_sentences_per_chunk=5,
)
```

<ParamField path="chunking_strategy" type="str" default="sentence" toc={true}>
  Strategy for splitting documents into chunks. Valid values:
  - `"sentence"` -- sentence-based chunking with overlap (default)
  - `"sliding"` -- sliding window with word-based chunks
  - `"paragraph"` -- split on natural paragraph boundaries
  - `"page"` -- page-based chunking (best for PDFs)
  - `"semantic"` -- split based on semantic similarity between sentences
  - `"topic"` -- split based on topic/keyword changes
  - `"qa"` -- optimized for question-answer content
  - `"json"` -- JSON structure-aware chunking for pre-processed content
  - `"markdown"` -- Markdown structure-aware chunking with code block detection
</ParamField>

<ParamField path="max_sentences_per_chunk" type="int" default="5" toc={true}>
  Maximum number of sentences per chunk for the `sentence` strategy.
</ParamField>

<ParamField path="chunk_size" type="int" default="50" toc={true}>
  Number of words per chunk for the `sliding` strategy.
</ParamField>

<ParamField path="chunk_overlap" type="int" default="10" toc={true}>
  Number of overlapping words between consecutive chunks for the `sliding` strategy.
</ParamField>

<ParamField path="split_newlines" type="Optional[int]" toc={true}>
  For the `sentence` strategy, split on this many consecutive newlines before
  sentence tokenization. Defaults to `2` when not set.
</ParamField>

<ParamField path="index_nlp_backend" type="str" default="nltk" toc={true}>
  NLP backend for document processing. Valid values:
  - `"nltk"` -- fast, lightweight (default)
  - `"spacy"` -- better quality, requires spaCy models
</ParamField>

<ParamField path="verbose" type="bool" default="false" toc={true}>
  Enable verbose logging output.
</ParamField>

<ParamField path="semantic_threshold" type="float" default="0.5" toc={true}>
  Similarity threshold for the `semantic` chunking strategy. Sentences with
  similarity below this threshold trigger a chunk boundary.
</ParamField>

<ParamField path="topic_threshold" type="float" default="0.3" toc={true}>
  Keyword overlap threshold for the `topic` chunking strategy.
</ParamField>

#### Example

```python {3}
from signalwire.search import DocumentProcessor

processor = DocumentProcessor(
    chunking_strategy="markdown",
    max_sentences_per_chunk=8,
)
```
