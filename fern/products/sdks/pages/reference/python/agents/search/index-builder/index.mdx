`IndexBuilder` processes document directories into searchable indexes. It handles
file discovery, text extraction, chunking, embedding generation, and storage into
either SQLite (`.swsearch` files) or pgvector backends. This is the programmatic
equivalent of the [`sw-search`][sw-search] CLI tool.

```python
from signalwire.search import IndexBuilder
```

<Warning>
Requires search dependencies. Install with `pip install signalwire[search-full]`
for full document processing support.
</Warning>

## **Properties**

<ParamField path="model_name" type="str" toc={true}>
  Name of the sentence transformer model used for embeddings.
</ParamField>

<ParamField path="chunking_strategy" type="str" toc={true}>
  Active chunking strategy passed to the internal `DocumentProcessor`.
</ParamField>

<ParamField path="backend" type="str" toc={true}>
  Storage backend. Either `"sqlite"` or `"pgvector"`.
</ParamField>

<ParamField path="doc_processor" type="DocumentProcessor" toc={true}>
  The [`DocumentProcessor`][documentprocessor]
  instance used for chunking.
</ParamField>

<ParamField path="model" type="SentenceTransformer | None" toc={true}>
  The loaded sentence transformer model. `None` until the first call to
  `build_index()` or `build_index_from_sources()`.
</ParamField>

## **Constructor**

```python {1}
builder = IndexBuilder(
    model_name="sentence-transformers/all-MiniLM-L6-v2",
    chunking_strategy="sentence",
)
```

<ParamField path="model_name" type="str" default="sentence-transformers/all-mpnet-base-v2" toc={true}>
  Name of the sentence transformer model for generating embeddings.
  Use [`resolve_model_alias()`][resolvemodelalias]
  to convert short aliases like `"mini"` or `"base"` to full model names.
</ParamField>

<ParamField path="chunking_strategy" type="str" default="sentence" toc={true}>
  Document chunking strategy. Valid values: `"sentence"`, `"sliding"`, `"paragraph"`,
  `"page"`, `"semantic"`, `"topic"`, `"qa"`, `"json"`, `"markdown"`.
  See [`DocumentProcessor`][documentprocessor]
  for details on each strategy.
</ParamField>

<ParamField path="max_sentences_per_chunk" type="int" default="5" toc={true}>
  Maximum sentences per chunk for the `sentence` strategy.
</ParamField>

<ParamField path="chunk_size" type="int" default="50" toc={true}>
  Words per chunk for the `sliding` strategy.
</ParamField>

<ParamField path="chunk_overlap" type="int" default="10" toc={true}>
  Word overlap between chunks for the `sliding` strategy.
</ParamField>

<ParamField path="split_newlines" type="Optional[int]" toc={true}>
  Split on consecutive newlines before sentence tokenization.
</ParamField>

<ParamField path="index_nlp_backend" type="str" default="nltk" toc={true}>
  NLP backend for document processing. `"nltk"` or `"spacy"`.
</ParamField>

<ParamField path="verbose" type="bool" default="false" toc={true}>
  Enable verbose progress output.
</ParamField>

<ParamField path="semantic_threshold" type="float" default="0.5" toc={true}>
  Similarity threshold for semantic chunking.
</ParamField>

<ParamField path="topic_threshold" type="float" default="0.3" toc={true}>
  Keyword overlap threshold for topic chunking.
</ParamField>

<ParamField path="backend" type="str" default="sqlite" toc={true}>
  Storage backend. `"sqlite"` or `"pgvector"`.
</ParamField>

<ParamField path="connection_string" type="Optional[str]" toc={true}>
  PostgreSQL connection string. Required when `backend` is `"pgvector"`.
</ParamField>

#### Example

```python {3}
from signalwire.search import IndexBuilder

builder = IndexBuilder(
    model_name="sentence-transformers/all-MiniLM-L6-v2",
    chunking_strategy="markdown",
    verbose=True,
)
```
