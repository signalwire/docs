The `Agent` class mirrors a LiveKit agent. It holds the system prompt (instructions),
a list of tools, and optional lifecycle hooks. When bound to an
[`AgentSession`][agentsession], the session
translates this into a SignalWire
[`AgentBase`][agentbase] under the hood.

```python {8-11}
from signalwire.livewire import Agent, function_tool

@function_tool
def get_weather(city: str) -> str:
    """Get the current weather for a city."""
    return f"It is sunny in {city}."

agent = Agent(
    instructions="You are a helpful weather assistant.",
    tools=[get_weather],
)
```

## **Properties**

<ParamField path="instructions" type="str" toc={true}>
  The system prompt text for the agent.
</ParamField>

<ParamField path="session" type="Optional[AgentSession]" toc={true}>
  The [`AgentSession`](/docs/sdks/reference/python/agents/livewire/agent-session) this
  agent is bound to. Set automatically when `session.start(agent)` is called.
</ParamField>

## **Constructor**

```python {1-5}
agent = Agent(
    instructions="You are a helpful assistant.",
    tools=[my_tool],
    llm="gpt-4o",
)
```

<ParamField path="instructions" type="str" default='""' toc={true}>
  The system prompt that defines the agent's behavior and personality.
</ParamField>

<ParamField path="tools" type="Optional[list[Any]]" default="None" toc={true}>
  A list of functions decorated with
  [`@function_tool`](/docs/sdks/reference/python/agents/livewire/function-tool).
  Tools registered here are merged with any tools on the `AgentSession`.
</ParamField>

<ParamField path="llm" type="Any" default="NOT_GIVEN" toc={true}>
  LLM model identifier (e.g., `"gpt-4o"` or `"openai/gpt-4o"`). When a provider
  prefix is included, it is stripped automatically. The model string is mapped to the
  SignalWire AI `model` parameter.
</ParamField>

<ParamField path="allow_interruptions" type="Any" default="NOT_GIVEN" toc={true}>
  Whether the caller can interrupt (barge in on) the agent while it speaks. When set
  to `False`, the SignalWire `barge_confidence` parameter is set to `1.0`, effectively
  disabling barge-in.
</ParamField>

<ParamField path="min_endpointing_delay" type="Any" default="NOT_GIVEN" toc={true}>
  Minimum silence duration (in seconds) before the platform considers the user's
  turn complete. Mapped to the SignalWire `end_of_speech_timeout` parameter (converted
  to milliseconds).
</ParamField>

<ParamField path="max_endpointing_delay" type="Any" default="NOT_GIVEN" toc={true}>
  Maximum silence duration (in seconds) before the platform stops waiting for more
  speech. Mapped to the SignalWire `attention_timeout` parameter (converted to
  milliseconds).
</ParamField>

<ParamField path="chat_ctx" type="Any" default="NOT_GIVEN" toc={true}>
  Initial chat context. Accepted for API compatibility.
</ParamField>

<ParamField path="stt" type="Any" default="NOT_GIVEN" toc={true}>
  Speech-to-text plugin instance.
  <Note>No-op. SignalWire's control plane handles speech recognition automatically.</Note>
</ParamField>

<ParamField path="tts" type="Any" default="NOT_GIVEN" toc={true}>
  Text-to-speech plugin instance.
  <Note>No-op. SignalWire's control plane handles text-to-speech automatically.</Note>
</ParamField>

<ParamField path="vad" type="Any" default="NOT_GIVEN" toc={true}>
  Voice activity detection plugin instance.
  <Note>No-op. SignalWire's control plane handles VAD automatically.</Note>
</ParamField>

<ParamField path="turn_detection" type="Any" default="NOT_GIVEN" toc={true}>
  Turn detection configuration.
  <Note>No-op. SignalWire's control plane handles turn detection automatically.</Note>
</ParamField>

<ParamField path="mcp_servers" type="Any" default="NOT_GIVEN" toc={true}>
  MCP server list.
  <Note>Not yet supported in LiveWire. Register tools via [`@function_tool`](/docs/sdks/reference/python/agents/livewire/function-tool) instead.</Note>
</ParamField>
