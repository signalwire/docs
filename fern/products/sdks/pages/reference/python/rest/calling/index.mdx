The `CallingNamespace` provides REST-based call control through the
[`RestClient`](/docs/sdks/reference/python/rest/client). All 37 commands are dispatched
as POST requests to a single endpoint (`/api/calling/calls`) with a `command` field
identifying the operation.

Access via `client.calling` on a [`RestClient`](/docs/sdks/reference/python/rest/client) instance.

<Note>
Unlike the RELAY client which uses persistent WebSocket connections, the Calling namespace
sends each command as an independent HTTP request. This is suitable for server-side
orchestration where you do not need real-time event streams.
</Note>

## Methods

### Call Lifecycle

| Method | Description |
|---|---|
| [`dial`](/docs/sdks/reference/python/rest/calling/dial) | Initiate a new outbound call. |
| [`update`](/docs/sdks/reference/python/rest/calling/update) | Update parameters on an active call. |
| [`end`](/docs/sdks/reference/python/rest/calling/end) | End an active call. |

### Call Control

| Method | Description |
|---|---|
| [`transfer`](/docs/sdks/reference/python/rest/calling/transfer) | Transfer a call to a new destination. |
| [`disconnect`](/docs/sdks/reference/python/rest/calling/disconnect) | Disconnect (unbridge) a connected call. |

### Media and Playback

| Method | Description |
|---|---|
| [`play`](/docs/sdks/reference/python/rest/calling/play) | Play audio or TTS on a call. |
| [`play_pause`](/docs/sdks/reference/python/rest/calling/play-pause) | Pause an active playback. |
| [`play_resume`](/docs/sdks/reference/python/rest/calling/play-resume) | Resume a paused playback. |
| [`play_stop`](/docs/sdks/reference/python/rest/calling/play-stop) | Stop an active playback. |
| [`play_volume`](/docs/sdks/reference/python/rest/calling/play-volume) | Adjust playback volume. |
