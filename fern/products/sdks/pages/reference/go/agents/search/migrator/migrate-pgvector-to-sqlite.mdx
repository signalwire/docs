Migrate a pgvector collection to a SQLite `.swsearch` file. This is useful
for creating local backups or moving from a PostgreSQL deployment to a
portable file-based index.

<Note>
This method creates the SQLite schema and copies configuration, but full chunk
migration from pgvector to SQLite is planned for a future release.
</Note>

## **Parameters**

<ParamField path="connection_string" type="string" required={true} toc={true}>
  PostgreSQL connection string.
</ParamField>

<ParamField path="collection_name" type="string" required={true} toc={true}>
  Name of the source pgvector collection.
</ParamField>

<ParamField path="output_path" type="string" required={true} toc={true}>
  Output path for the `.swsearch` file. The `.swsearch` extension is appended
  automatically if not present.
</ParamField>

<ParamField path="batch_size" type="int" default="100" toc={true}>
  Number of chunks to fetch per batch.
</ParamField>

## **Returns**

`dict[str, Any]` -- Migration statistics with `source`, `target`, `chunks_migrated`,
`errors`, and `config` fields.

## **Example**

```go {4}
package main

import "fmt"

func main() {
    migrator := SearchIndexMigrator(verbose=true)
    stats := migrator.migrate_pgvector_to_sqlite(
        connection_string := "postgresql://user:pass@localhost/mydb",
        collection_name := "knowledge",
        output_path := "./knowledge-backup.swsearch",
    )

    fmt.Println(fmt.Sprintf("Migrated %v chunks to %v", stats['chunks_migrated'], stats['target']))
}
```
