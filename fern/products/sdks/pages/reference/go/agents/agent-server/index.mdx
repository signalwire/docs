AgentServer hosts multiple [`AgentBase`][agentbase] instances
on a single HTTP server. Each agent registers at its own URL route, and the server provides
unified health monitoring, SIP-based routing, and static file serving. Use AgentServer when you
have several related agents (sales, support, billing) that share the same deployment environment.

For a single agent, use [`AgentBase.serve()`][agentbase-serve]
or [`AgentBase.run()`][agentbase-run] instead.

## **Example**

```go {17}
package main

import (
    "github.com/signalwire/signalwire-go/pkg/agent"
    "github.com/signalwire/signalwire-go/pkg/server"
)

func main() {
    // SalesAgent embeds AgentBase
        func __init__() *swaig.FunctionResult {
            // Base config: name="sales-agent", route="/sales"
            AddLanguage("English", "en-US", "rime.spore")
            PromptAddSection("Role", "You are a sales representative.")

    // SupportAgent embeds AgentBase
        func __init__() *swaig.FunctionResult {
            // Base config: name="support-agent", route="/support"
            AddLanguage("English", "en-US", "rime.spore")
            PromptAddSection("Role", "You are a support specialist.")

        server := server.NewAgentServer(server.WithServerHost("0.0.0.0"), server.WithServerPort(3000))
        server.register(SalesAgent(), "/sales")
        server.register(SupportAgent(), "/support")
        server.Run()
}
```

After starting, agents are available at:

| Endpoint | Description |
|----------|-------------|
| `http://localhost:3000/sales` | Sales agent |
| `http://localhost:3000/support` | Support agent |
| `http://localhost:3000/health` | Health check |
| `http://localhost:3000/ready` | Readiness check |
