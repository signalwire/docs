[enable-debug-events]: /docs/sdks/reference/php/agents/agent-base/enable-debug-events

Register a callback for debug events received at the `/debug_events` endpoint.
Use as a decorator. Both sync and async handlers are supported.

<Note>
[`enableDebugEvents()`][enable-debug-events] must be called before events will be delivered.
</Note>

## **Parameters**

<ParamField path="handler" type="callable" required={true} toc={true}>
  Callback function with signature `(event_type: string, data: dict)`.

  - `event_type` -- Event label string (e.g., `"barge"`, `"llm_error"`, `"session_start"`, `"step_change"`)
  - `data` -- Full event payload including `call_id`, `label`, and event-specific fields
</ParamField>

## **Returns**

`callable` -- The handler function, unchanged. This allows `onDebugEvent` to be used as a
decorator.

## **Example**

```php {7}
<?php
use SignalWire\Agent\AgentBase;

$agent = new AgentBase(name: "debug-agent", route: "/debug");
$agent->setPromptText("You are a helpful assistant.");
$agent->enableDebugEvents(level: 1);

// Handler: $agent->onDebugEvent()
function handleDebug($event_type, $data)
{
    if ($event_type == "llm_error") {
        echo "LLM error: {$data}";
    }
    } elseif ($event_type == "barge") {
        echo "Barge detected: {$data['barge_elapsed_ms'] ?? null}ms";
    }
    } elseif ($event_type == "step_change") {
        echo "Step changed to: {$data['step'] ?? null}";
    }

$agent->serve();


```
