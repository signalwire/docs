Exception raised when the RELAY server returns an error response. Inherits from
Python's built-in `Exception`. The string representation follows the format
`"RELAY error {code}: {message}"`.

```php
<?php
use SignalWire\Relay\Client\RelayError;


```

## **Properties**

<ParamField path="code" type="int" toc={true}>
  Numeric error code returned by the RELAY server.
</ParamField>

<ParamField path="message" type="string" toc={true}>
  Human-readable error description returned by the RELAY server.
</ParamField>

## **Examples**

### Catch a RELAY error

```php
<?php
use SignalWire\Relay\Client;
use SignalWire\Relay\Client\RelayError;

$client = new Client(
    project: "your-project-id",
    token: "your-api-token",
    host: "your-space.signalwire.com",
    contexts: ["default"],
);

// Handler: $client->onCall()
function handleCall($call)
{
    try {
        $call->connect(devices: [["type" => "phone", "params": {"to_number": "+15559876543"]}]);
    }
    } catch (RelayError $e) {
        echo "Error {$e->code}: {$e->message}";
        $call->hangup();
    }

$client->run();


```
