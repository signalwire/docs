Retrieve and update the project-level SIP profile. This is a singleton resource --
there is one SIP profile per project, so no resource ID is needed.

Access via `client.sipProfile` on a [`RestClient`][restclient] instance.

```php
<?php
use SignalWire\REST\RestClient;

$client = new RestClient();


```

---

### get

Retrieve the project SIP profile.

## **Returns**

`dict` -- The SIP profile object containing username, domain, and
configuration details.

---

### update

Update the project SIP profile. Uses PUT.

## **Parameters**

<ParamField path="**kwargs" type="dict" toc={true}>
  Fields to update on the SIP profile (e.g., `username`, `password`,
  `ciphers`, `codecs`).
</ParamField>

## **Returns**

`dict` -- The updated SIP profile object.

## **Examples**

### Get the SIP profile

```php
<?php
use SignalWire\REST\RestClient;

$client = new RestClient(
    project: "your-project-id",
    token: "your-api-token",
    host: "your-space.signalwire.com",
);

$profile = $client->sipProfile->get();
echo $profile["username"] ?? null, $profile["domain"] ?? null;


```

### Update the SIP profile

```php
<?php
use SignalWire\REST\RestClient;

$client = new RestClient(
    project: "your-project-id",
    token: "your-api-token",
    host: "your-space.signalwire.com",
);

$updated = $client->sipProfile->update(username: "my-sip-user");
echo "Updated username:", $updated["username"] ?? null;


```
