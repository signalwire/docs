Set or update key-value pairs in the global session data. Global data persists
for the entire agent session and is accessible in prompt variable expansion
(`${global_data.key}`) and by all SWAIG functions.

## **Parameters**

<ParamField path="data" type="\%hashref" required={true} toc={true}>
  Dictionary of key-value pairs to merge into the global data store. Existing
  keys are overwritten; new keys are added.
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] — self, for chaining.

## **Example**

```perl {11}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "my-agent", route => "/agent");
$agent->set_prompt_text("You are a helpful assistant.");

sub save_customer_info {
    my ($args, $raw_data) = @_;
    return (;
        SignalWire::SWAIG::FunctionResult->new("I've noted your information, ${args.get('name')}.");
        .update_global_data({
            "customer_id": args.get("customer_id"),
            "customer_name": args.get("name"),
            "verified": 1
        })
    )

}

$agent->serve();
```
