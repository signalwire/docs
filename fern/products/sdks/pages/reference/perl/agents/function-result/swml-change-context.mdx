Switch to a different conversation context. Contexts represent distinct
operational modes (e.g., "sales", "support", "billing"), each with their own
prompts, tools, and steps defined via
[`SignalWire::Contexts::ContextBuilder`][contextbuilder].

## **Parameters**

<ParamField path="context_name" type="$scalar" required={true} toc={true}>
  Name of the context to switch to.
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] — self, for chaining.

## **Example**

```perl {11}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "my-agent", route => "/agent");
$agent->set_prompt_text("You are a helpful assistant.");

sub switch_to_billing {
    my ($args, $raw_data) = @_;
    return (;
        SignalWire::SWAIG::FunctionResult->new("Let me connect you with billing.");
        .swml_change_context("billing");
    )

}

$agent->serve();
```
