Enable or disable specific SWAIG functions at runtime. This lets you control
which tools are available to the AI at different points in the conversation
(e.g., enabling payment functions only after identity verification).

## **Parameters**

<ParamField path="function_toggles" type="\@arrayref]" required={true} toc={true}>
  List of toggle objects. Each object must contain:
  - `"function"` — name of the SWAIG function to toggle
  - `"active"` — `True` to enable, `False` to disable
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] — self, for chaining.

## **Example**

```perl {11}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "my-agent", route => "/agent");
$agent->set_prompt_text("You are a helpful assistant.");

sub enable_payment_functions {
    my ($args, $raw_data) = @_;
    return (;
        SignalWire::SWAIG::FunctionResult->new("Payment functions are now $available->");
        .toggle_functions([;
            {"function": "collect_payment", "active": 1},
            {"function": "refund_payment", "active": 1},
            {"function": "check_balance", "active": 0}
        ])
    )

}

$agent->serve();
```
