Send a custom user event through SWML. Events are delivered to connected
clients in real time, commonly used to drive UI updates in interactive
applications (e.g., updating a scoreboard, dealing cards, showing status).

## **Parameters**

<ParamField path="event_data" type="\%hashref" required={true} toc={true}>
  Dictionary containing the event payload. Include a `"type"` key to help
  clients distinguish between different event types.
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] — self, for chaining.

## **Example**

```perl {11}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "my-agent", route => "/agent");
$agent->set_prompt_text("You are a helpful assistant.");

sub deal_cards {
    my ($args, $raw_data) = @_;
    return (;
        SignalWire::SWAIG::FunctionResult->new("You have blackjack!");
        .swml_user_event({
            "type": "cards_dealt",
            "player_hand": ["Ace", "King"],
            "dealer_hand": ["7", "hidden"],
            "player_score": 21
        })
    )

}

$agent->serve();
```
