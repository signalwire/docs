Execute a raw SWML document as an action. This is the escape hatch for advanced
use cases that are not covered by the named convenience methods.

<Note>
Most use cases are covered by the specific action methods
([`connect()`][connect],
[`record_call()`][record-call],
[`send_sms()`][send-sms], etc.).
Use `execute_swml()` only when you need SWML features not available through
convenience methods.
</Note>

## **Parameters**

<ParamField path="swml_content" type="$scalar | \%hashref | SWML object" required={true} toc={true}>
  SWML content in one of three formats:
  - `str` — raw SWML JSON text (parsed internally)
  - `dict` — SWML data structure
  - SWML object — any object with a `.to_dict()` method (e.g., a [`SignalWire::SWML::Document`][ref-swmlbuilder] instance)
</ParamField>

<ParamField path="transfer" type="$boolean" default="False" toc={true}>
  When `True`, the call exits the agent after the SWML executes. When `False`,
  the SWML executes inline and the agent continues.
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] — self, for chaining.

## **Example**

```perl {20}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "my-agent", route => "/agent");
$agent->set_prompt_text("You are a helpful assistant.");

sub play_announcement {
    my ($args, $raw_data) = @_;
    my $swml_doc = {
        "version": "1.0.0",
        "sections": {
            "main": [
                {"play": {"url": "https://example.com/announcement.mp3"}},
                {"hangup": {}}
            ]
        }
    }
    return (;
        SignalWire::SWAIG::FunctionResult->new();
        .execute_swml(swml_doc, transfer => 0);
    )

}

$agent->serve();
```
