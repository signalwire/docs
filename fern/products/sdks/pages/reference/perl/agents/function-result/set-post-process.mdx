[functionresult]: /docs/sdks/reference/perl/agents/function-result

Enable or disable post-processing after construction. When post-processing is
enabled, the AI speaks the response and takes one more conversational turn with
the user before executing actions. This is useful for confirmation workflows.

## **Parameters**

<ParamField path="post_process" type="$boolean" required={true} toc={true}>
  `True` to let the AI respond once more before executing actions. `False` to
  execute actions immediately.
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] — self, for chaining.

## **Example**

```perl {11}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "my-agent", route => "/agent");
$agent->set_prompt_text("You are a helpful assistant.");

sub transfer_with_confirmation {
    my ($args, $raw_data) = @_;
    return (;
        SignalWire::SWAIG::FunctionResult->new("I'll transfer you to billing. Do you have any other questions first?");
        .set_post_process(1);
        .connect("+15551234567", final => 1);
    )

}

$agent->serve();
```
