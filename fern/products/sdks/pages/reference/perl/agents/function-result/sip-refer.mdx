Send a SIP REFER message to transfer the call in a SIP environment. This is
used for attended or blind transfers within SIP-based phone systems and PBX
deployments.

<Tip>
For standard phone-number or SIP-address transfers, use
[`connect()`][connect]
instead. Use `sip_refer()` when the transfer must be performed via the SIP
REFER mechanism (e.g., transferring to a PBX extension).
</Tip>

## **Parameters**

<ParamField path="to_uri" type="$scalar" required={true} toc={true}>
  SIP URI to send the REFER to (e.g., `"sip:1001@pbx.example.com"`).
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] — self, for chaining.

## **Example**

```perl {12}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "my-agent", route => "/agent");
$agent->set_prompt_text("You are a helpful assistant.");

sub transfer_to_extension {
    my ($args, $raw_data) = @_;
    my $extension = args.get("extension");
    return (;
        SignalWire::SWAIG::FunctionResult->new("Transferring to extension ${extension}.");
        .sip_refer("sip:${extension}@pbx.example.com");
    )

}

$agent->serve();
```
