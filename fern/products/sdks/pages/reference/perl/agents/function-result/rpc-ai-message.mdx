Inject a message into the AI agent running on another call. Useful for
cross-call coordination, such as notifying a held caller's agent about a
status change or instructing it to relay information.

## **Parameters**

<ParamField path="call_id" type="$scalar" required={true} toc={true}>
  Call ID of the target call whose AI agent should receive the message.
</ParamField>

<ParamField path="message_text" type="$scalar" required={true} toc={true}>
  The message text to inject into the target AI's conversation.
</ParamField>

<ParamField path="role" type="$scalar" default="system" toc={true}>
  Role for the injected message. Typically `"system"` for instructions.
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] — self, for chaining.

## **Example**

```perl {12}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "my-agent", route => "/agent");
$agent->set_prompt_text("You are a helpful assistant.");

sub notify_caller {
    my ($args, $raw_data) = @_;
    my $caller_call_id = args.get("original_call_id");
    return (;
        SignalWire::SWAIG::FunctionResult->new("I'll let them know.");
        .rpc_ai_message(
            my $call_id = caller_call_id,
            my $message_text = "The person you're trying to reach is unavailable. Please leave a $message->";
        )
    )

}

$agent->serve();
```
