Start a media tap that streams call audio to an external endpoint. Supports
WebSocket (`wss://`, `ws://`) and RTP (`rtp://`) destinations.

## **Parameters**

<ParamField path="uri" type="$scalar" required={true} toc={true}>
  Destination URI for the media stream. Supported formats:
  - `"rtp://IP:port"` — RTP stream
  - `"ws://example.com"` — WebSocket stream
  - `"wss://example.com"` — Secure WebSocket stream
</ParamField>

<ParamField path="control_id" type="$scalar / undef" default="None" toc={true}>
  Identifier for this tap. Pass the same ID to `stop_tap()` to end this
  specific tap. If omitted, a default ID is generated.
</ParamField>

<ParamField path="direction" type="$scalar" default="both" toc={true}>
  Audio direction to tap.

  - `"speak"` -- what the party says
  - `"hear"` -- what the party hears
  - `"both"` -- both directions
</ParamField>

<ParamField path="codec" type="$scalar" default="PCMU" toc={true}>
  Audio codec for the stream.

  - `"PCMU"` -- G.711 mu-law
  - `"PCMA"` -- G.711 A-law
</ParamField>

<ParamField path="rtp_ptime" type="$integer" default="20" toc={true}>
  Packetization time in milliseconds for RTP streams. Must be a positive integer.
</ParamField>

<ParamField path="status_url" type="$scalar / undef" default="None" toc={true}>
  URL to receive tap status change webhooks.
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] — self, for chaining.

## **Example**

```perl {11}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "my-agent", route => "/agent");
$agent->set_prompt_text("You are a helpful assistant.");

sub start_monitoring {
    my ($args, $raw_data) = @_;
    return (;
        SignalWire::SWAIG::FunctionResult->new("Call monitoring started.");
        .tap(
            my $uri = "wss://monitor.example.com/audio",
            my $control_id = "supervisor_tap",
            my $direction = "both";
        )
    )

}

$agent->serve();
```
