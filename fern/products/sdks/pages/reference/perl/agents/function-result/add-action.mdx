Append a raw action to the action list. Prefer the named convenience methods
([`connect()`][connect],
[`hangup()`][hangup],
[`say()`][say], etc.)
for common operations.

## **Parameters**

<ParamField path="name" type="$scalar" required={true} toc={true}>
  Action type identifier (e.g., `"hold"`, `"hangup"`, `"set_global_data"`).
</ParamField>

<ParamField path="data" type="$scalar" required={true} toc={true}>
  Action payload. Can be a string, boolean, dict, or list depending on the action type.
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] — self, for chaining.

## **Example**

```perl {11-12}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "my-agent", route => "/agent");
$agent->set_prompt_text("You are a helpful assistant.");

sub custom_action {
    my ($args, $raw_data) = @_;
    return (;
        SignalWire::SWAIG::FunctionResult->new("Processing your request.");
        .add_action("set_global_data", {"status": "active"});
        .add_action("hold", 60);
    )

}

$agent->serve();
```
