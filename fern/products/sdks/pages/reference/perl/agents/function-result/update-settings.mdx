Update AI runtime settings dynamically during a call. Changes take effect
for subsequent LLM turns.

## **Parameters**

<ParamField path="settings" type="\%hashref" required={true} toc={true}>
  Dictionary of settings to update. Supported keys:

  | Key | Type | Range |
  |-----|------|-------|
  | `frequency-penalty` | `float` | -2.0 to 2.0 |
  | `presence-penalty` | `float` | -2.0 to 2.0 |
  | `max-tokens` | `int` | 0 to 4096 |
  | `top-p` | `float` | 0.0 to 1.0 |
  | `confidence` | `float` | 0.0 to 1.0 |
  | `barge-confidence` | `float` | 0.0 to 1.0 |
  | `temperature` | `float` | 0.0 to 2.0 (clamped to 1.5) |
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] — self, for chaining.

## **Example**

```perl {11}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "my-agent", route => "/agent");
$agent->set_prompt_text("You are a helpful assistant.");

sub adjust_for_technical_discussion {
    my ($args, $raw_data) = @_;
    return (;
        SignalWire::SWAIG::FunctionResult->new("Adjusting response parameters.");
        .update_settings({
            "temperature": 0.3,
            "confidence": 0.9,
            "barge-confidence": 0.8
        })
    )

}

$agent->serve();
```
