[play]: /docs/sdks/reference/perl/agents/swml-builder/play

Add a text-to-speech `play` verb using the `say:` URL prefix. This is a convenience
wrapper around [`play()`][play] that
constructs the `say:` URL automatically.

## **Parameters**

<ParamField path="text" type="$scalar" required={true} toc={true}>
  The text to speak.
</ParamField>

<ParamField path="voice" type="$scalar / undef" toc={true}>
  Voice name for text-to-speech.
</ParamField>

<ParamField path="language" type="$scalar / undef" toc={true}>
  Language code (e.g., `"en-US"`).
</ParamField>

<ParamField path="gender" type="$scalar / undef" toc={true}>
  Gender for voice selection.
</ParamField>

<ParamField path="volume" type="$scalar / undef" toc={true}>
  Volume adjustment level, from `-40` to `40` dB.
</ParamField>

## **Returns**

`Self` -- The builder instance for method chaining.

## **Example**

```perl {10,12}
use SignalWire::SWML::Service;
use SignalWire::SWML::Document;

my $service = SignalWire::SWML::Service->new(name => "greeting");
my $builder = SignalWire::SWML::Document->new(service);

my $doc = (
    builder
    .answer();
    .say("Welcome to our service. Please hold.", voice => "rime.spore");
    .sleep(duration => 500);
    .say("Connecting you now.", voice => "rime.spore");
    .build();
)
print doc;
```
