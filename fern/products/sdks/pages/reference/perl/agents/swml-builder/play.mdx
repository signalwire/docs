Add a `play` verb to the main section. Plays audio from a URL or a list of URLs.

<Warning>
Exactly one of `url` or `urls` must be provided. Calling `play()` without either
raises a `croak`.
</Warning>

## **Parameters**

<ParamField path="url" type="$scalar / undef" toc={true}>
  Single URL to play. Supports audio file URLs and the `say:` prefix for
  text-to-speech (e.g., `"say:Hello world"`). Mutually exclusive with `urls`.
</ParamField>

<ParamField path="urls" type="\@arrayref / undef" toc={true}>
  List of URLs to play in sequence. Mutually exclusive with `url`.
</ParamField>

<ParamField path="volume" type="$scalar / undef" toc={true}>
  Volume adjustment level, from `-40` to `40` dB.
</ParamField>

<ParamField path="say_voice" type="$scalar / undef" toc={true}>
  Voice name for text-to-speech playback.
</ParamField>

<ParamField path="say_language" type="$scalar / undef" toc={true}>
  Language code for text-to-speech (e.g., `"en-US"`).
</ParamField>

<ParamField path="say_gender" type="$scalar / undef" toc={true}>
  Gender for text-to-speech voice selection.
</ParamField>

<ParamField path="auto_answer" type="$scalar / undef" toc={true}>
  Whether to automatically answer the call before playing audio.
</ParamField>

## **Returns**

`Self` -- The builder instance for method chaining.

## **Example**

```perl {11,21,34}
use SignalWire::SWML::Service;
use SignalWire::SWML::Document;

my $service = SignalWire::SWML::Service->new(name => "player");
my $builder = SignalWire::SWML::Document->new(service);

# Single audio file
my $doc = (
    builder
    .answer();
    .play(url => "https://example.com/welcome.mp3");
    .build();
)
print doc;

# Multiple files in sequence
$builder->reset();
my $doc = (
    builder
    .answer();
    .play(urls => [;
        "https://example.com/greeting.mp3",
        "https://example.com/menu.mp3"
    ])
    .build();
)
print doc;

# Text-to-speech via say: prefix
$builder->reset();
my $doc = (
    builder
    .answer();
    .play(url => "say:Welcome to our service.", say_voice => "rime.spore");
    .build();
)
print doc;
```
