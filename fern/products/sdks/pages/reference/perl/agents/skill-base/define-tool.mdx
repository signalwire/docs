Register a tool with the parent agent, automatically merging any `swaig_fields`
configured for this skill. Skills should use this method instead of calling
`self.agent.define_tool()` directly.

## **Parameters**

<ParamField path="**kwargs" type="$scalar" required={true} toc={true}>
  All arguments supported by `agent.define_tool()`: `name`, `description`,
  `parameters`, `handler`, `secure`, `fillers`, `wait_file`, etc. Explicit
  values take precedence over `swaig_fields`.
</ParamField>

## **Returns**

`None`

## **Example**

```perl {12}
use SignalWire::Skills::SkillBase;
use SignalWire::SWAIG::FunctionResult;

package SearchSkill;
use Moo;
extends 'SignalWire::Skills::SkillBase';

    my $SKILL_NAME = "search";
    my $SKILL_DESCRIPTION = "Search the web for information";

    sub setup {
        return 1;

    sub register_tools {
        self.define_tool(
            name => "search_web",
            description => "Search the web for information",
            handler => self._handle_search,
            parameters => {
                "type": "object",
                "properties": {
                    "query": {"type": "string", "description": "Search query"}
                },
                "required": ["query"]
            },
            secure => 1,
            fillers => {"en-US": ["Let me search for that..."]}
        )

    sub _handle_search {
        my ($args, $raw_data) = @_;
        my $query = args.get("query", "");
        return SignalWire::SWAIG::FunctionResult->new("Results for: ${query}");
```
