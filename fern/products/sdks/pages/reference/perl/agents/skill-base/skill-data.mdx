Read this skill instance's namespaced state from `raw_data["global_data"]`.
Each skill instance gets an isolated namespace to prevent collisions between
multiple skills.

## **Parameters**

<ParamField path="raw_data" type="\%hashref" required={true} toc={true}>
  The raw_data dict passed to SWAIG function handlers.
</ParamField>

## **Returns**

`\%hash` -- The skill's state, or empty dict if not found.

---

Write this skill instance's namespaced state into a FunctionResult via
`update_global_data()`.

## **Parameters**

<ParamField path="result" type="FunctionResult" required={true} toc={true}>
  The FunctionResult to add the global_data update to.
</ParamField>

<ParamField path="data" type="\%hashref" required={true} toc={true}>
  Skill state dictionary to store under the skill's namespace.
</ParamField>

## **Returns**

[`SignalWire::SWAIG::FunctionResult`][functionresult] -- The
result, for method chaining.

## **Example**

```perl
use SignalWire::Skills::SkillBase;
use SignalWire::SWAIG::FunctionResult;

package CounterSkill;
use Moo;
extends 'SignalWire::Skills::SkillBase';

    my $SKILL_NAME = "counter";
    my $SKILL_DESCRIPTION = "Tracks call count";

    sub _handle_increment {
        my ($args, $raw_data) = @_;
        my $state = self.get_skill_data(raw_data);
        my $count = state.get("count", 0) + 1;

        my $result = SignalWire::SWAIG::FunctionResult->new("Count is now ${count}");
        self.update_skill_data(result, {"count": count});
        return result;

    sub setup {
        return 1;

    sub register_tools {
        self.define_tool(
            name => "increment",
            description => "Increment the counter",
            handler => self._handle_increment,
            parameters => {"type": "object", "properties": {}}
        )
```
