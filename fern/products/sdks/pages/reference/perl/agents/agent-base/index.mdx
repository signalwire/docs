`SignalWire::Agent::AgentBase` is the central class in the SignalWire AI Agents SDK. It provides a
complete framework for building AI-powered voice agents, combining prompt management,
tool definitions, skill loading, speech configuration, and web serving into a single
composable interface.

Extends [`SignalWire::SWML::Service`][swmlservice] and composes
functionality from nine mixins: `PromptMixin`, `ToolMixin`, `SkillMixin`, `AIConfigMixin`,
`WebMixin`, `AuthMixin`, `ServerlessMixin`, `StateMixin`, and `MCPServerMixin`.

All setter methods return `self` for fluent method chaining.

<Info>
AgentBase generates a SWML document with the [`ai`][ai] verb.
See the [SWML reference][swml-reference] for the full specification of all
supported parameters and behaviors.
</Info>

## **Properties**

<ParamField path="name" type="$scalar" toc={true}>
  The agent's display name. Set at construction time. Used in logging, SIP username
  mapping, and the default prompt fallback.
</ParamField>

<ParamField path="route" type="$scalar" default="/" toc={true}>
  HTTP route path where this agent is served. Used by
  [`SignalWire::Server::AgentServer`][agentserver] when hosting
  multiple agents on one process.
</ParamField>

<ParamField path="host" type="$scalar" default="0.0.0.0" toc={true}>
  Network interface the web server binds to.
</ParamField>

<ParamField path="port" type="$integer" toc={true}>
  Port the web server listens on. Defaults to the `PORT` environment variable, falling
  back to `3000`.
</ParamField>

<ParamField path="agent_id" type="$scalar" toc={true}>
  Unique identifier for this agent instance. Auto-generated as a UUID if not provided.
</ParamField>

<ParamField path="pom" type="$scalar / undef" toc={true}>
  The Prompt Object Model instance used for structured prompt building. `None` when
  `use_pom=False`.
</ParamField>

<ParamField path="skill_manager" type="SignalWire::Skills::SkillManager" toc={true}>
  Manager instance for loading and unloading skills. Access via
  [`add_skill()`][add-skill] and
  [`list_skills()`][list-skills] rather
  than directly.
</ParamField>

<ParamField path="PROMPT_SECTIONS" type="\%hashref | list | None" default="None" toc={true}>
  Class-level attribute. Subclasses can set this to declaratively define prompt sections
  instead of calling `prompt_add_section()` in the constructor.
</ParamField>

<ParamField path="basic_auth" type="\@arrayref / undef" toc={true}>
  Explicit `(username, password)` for HTTP Basic Auth on all endpoints. If not set,
  credentials are read from `SWML_BASIC_AUTH_USER` / `SWML_BASIC_AUTH_PASSWORD` env vars,
  or auto-generated on startup.
</ParamField>

<ParamField path="use_pom" type="$boolean" default="True" toc={true}>
  Enable Prompt Object Model for structured prompt building. Set to `False` to use
  plain text prompts only.
</ParamField>

<ParamField path="token_expiry_secs" type="$integer" default="3600" toc={true}>
  Expiration time in seconds for SWAIG function authentication tokens.
</ParamField>

<ParamField path="auto_answer" type="$boolean" default="True" toc={true}>
  Automatically add an `answer` verb before the AI verb in the SWML document.
</ParamField>

<ParamField path="record_call" type="$boolean" default="False" toc={true}>
  Enable call recording. When `True`, a `record_call` verb is added to the SWML document.
</ParamField>

<ParamField path="record_format" type="$scalar" default="mp4" toc={true}>
  Recording file format. Common values: `"mp4"`, `"wav"`.
</ParamField>

<ParamField path="record_stereo" type="$boolean" default="True" toc={true}>
  Record in stereo (separate channels for each party) when `True`.
</ParamField>

<ParamField path="default_webhook_url" type="$scalar / undef" toc={true}>
  Base URL for SWAIG function webhooks. If not set, the SDK auto-detects from the
  incoming request or uses `SWML_PROXY_URL_BASE`.
</ParamField>

<ParamField path="suppress_logs" type="$boolean" default="False" toc={true}>
  Suppress SDK log output. Useful in testing or when integrating with external logging.
</ParamField>

<ParamField path="enable_post_prompt_override" type="$boolean" default="False" toc={true}>
  Allow dynamic per-request override of the post-prompt configuration.
</ParamField>

<ParamField path="check_for_input_override" type="$boolean" default="False" toc={true}>
  Allow dynamic per-request override of input checking behavior.
</ParamField>

<ParamField path="config_file" type="$scalar / undef" toc={true}>
  Path to a JSON config file. If not provided, the SDK searches default paths.
  See [`ConfigLoader`][configloader].
</ParamField>

<ParamField path="native_functions" type="\@arrayref / undef" toc={true}>
  List of native SWAIG function names to enable at construction time (e.g.,
  `["check_time", "wait_for_user"]`). Can also be set later via
  [`set_native_functions()`][set-native-functions].
</ParamField>

<ParamField path="schema_path" type="$scalar / undef" toc={true}>
  Path to a custom SWML schema file for validation. If not provided, the SDK searches
  default paths automatically.
</ParamField>

<ParamField path="schema_validation" type="$boolean" default="True" toc={true}>
  Enable SWML schema validation. Disable with `False` or `SWML_SKIP_SCHEMA_VALIDATION=1`
  env var.
</ParamField>

## **Decorators**

### tool

```perl
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "assistant", route => "/assistant");
$agent->set_prompt_text("You are a helpful assistant.");

sub my_tool {
    my ($args, $raw_data) = @_;
    return SignalWire::SWAIG::FunctionResult->new("Done.");

}

$agent->serve();
```

The `define_tool()` method is the recommended way to define SWAIG functions. Use
`$agent->define_tool()` on a standalone function, or `$agent->define_tool()` on a method
inside a subclass. Both forms accept the same parameters.

When parameters are not explicitly provided, the decorator automatically infers the
JSON Schema from Perl type hints on the function signature.

#### Parameters

<ParamField path="name" type="$scalar / undef" toc={true}>
  Function name exposed to the AI. Defaults to the subroutine name.
</ParamField>

<ParamField path="description" type="$scalar / undef" toc={true}>
  What the function does. The AI reads this to decide when to call it. Defaults to the
  subroutine documentation, or `"Function {name}"` as a fallback.
</ParamField>

<ParamField path="parameters" type="\%hashref / undef" toc={true}>
  Explicit JSON Schema for function parameters. If omitted, the schema is
  automatically inferred from Perl type hints on the decorated function.
</ParamField>

<ParamField path="secure" type="$boolean" default="True" toc={true}>
  Require token validation on tool calls.
</ParamField>

<ParamField path="fillers" type="\%hashref / undef]" toc={true}>
  Filler phrases by language code, spoken while the function runs.
</ParamField>

<ParamField path="webhook_url" type="$scalar / undef" toc={true}>
  External webhook URL. If set, SignalWire calls this URL instead of executing locally.
</ParamField>

<ParamField path="required" type="\@arrayref / undef" toc={true}>
  Required parameter names. Auto-inferred from type hints when not specified.
</ParamField>

<ParamField path="**swaig_fields" type="$scalar" toc={true}>
  Additional SWAIG fields (e.g., `wait_file`, `meta_data`).
</ParamField>

## **Examples**

### Instance decorator with type inference

```perl {4}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "assistant", route => "/assistant");
$agent->set_prompt_text("You are a helpful assistant.");

sub check_order {
    my ($args, $raw_data) = @_;
    my $order_id = args.get("order_id");
    return SignalWire::SWAIG::FunctionResult->new("Order ${order_id} shipped March 28.");

}

$agent->serve();
```

### With explicit parameters

```perl
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "assistant", route => "/assistant");
$agent->set_prompt_text("You are a helpful assistant.");

sub search_products {
    my ($args, $raw_data) = @_;
    my $query = args.get("query");
    return SignalWire::SWAIG::FunctionResult->new("Found 3 results for '${query}'.");

}

$agent->serve();
```

### Class decorator (subclass)

```perl
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

package SupportAgent;
use Moo;
extends 'SignalWire::Agent::AgentBase';

    sub transfer_to_human {
        my ($args, $raw_data) = @_;
        return SignalWire::SWAIG::FunctionResult->new("Transferring now.").connect("+15551234567");

    }

my $agent = SupportAgent(name => "support", route => "/support");
$agent->serve();
```

### Typed parameters (auto-inferred schema)

```perl
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "assistant", route => "/assistant");
$agent->set_prompt_text("You are a helpful assistant.");

sub calculate_shipping {
    my ($args, $raw_data) = @_;
    my $cost = weight_kg * 2.50 if not express else weight_kg * 5.00;
    return SignalWire::SWAIG::FunctionResult->new("Shipping to ${destination}: $${cost:.2f}");

}

$agent->serve();
```

### Basic agent with a tool

```perl {4}
use SignalWire::Agent::AgentBase;
use SignalWire::SWAIG::FunctionResult;

my $agent = SignalWire::Agent::AgentBase->new(name => "support-agent", route => "/support");

$agent->add_language("English", "en-US", "rime.spore");
$agent->set_prompt_text("You are a friendly customer support $agent->");
$agent->add_hints(["SignalWire", "SWML", "SWAIG"]);
$agent->set_params({"temperature": 0.7, "end_of_speech_timeout": 1000});

sub lookup_order {
    my ($args, $raw_data) = @_;
    my $order_id = args.get("order_id");
    return SignalWire::SWAIG::FunctionResult->new("Order ${order_id} shipped on March 28.");

}

$agent->run();
```
