Connect the call to an Amazon Bedrock AI agent. Similar to
[`ai()`][ai] but uses Amazon Bedrock as
the LLM backend.

## **Parameters**

<ParamField path="prompt" type="$scalar / undef" toc={true}>
  The prompt configuration for the Bedrock agent.
</ParamField>

<ParamField path="SWAIG" type="$scalar / undef" toc={true}>
  SWAIG configuration for tool/function definitions.
</ParamField>

<ParamField path="ai_params" type="$scalar / undef" toc={true}>
  AI parameters for the Bedrock session.
</ParamField>

<ParamField path="global_data" type="$scalar / undef" toc={true}>
  Data accessible to the AI and SWAIG functions.
</ParamField>

<ParamField path="post_prompt" type="$scalar / undef" toc={true}>
  Post-prompt configuration.
</ParamField>

<ParamField path="post_prompt_url" type="$scalar / undef" toc={true}>
  URL to receive the post-prompt result.
</ParamField>

## **Returns**

`dict` -- Server response confirming the Bedrock session.

## **Example**

```perl {15}
use SignalWire::Relay::Client;

my $client = SignalWire::Relay::Client->new(
    project => "your-project-id",
    token => "your-api-token",
    host => "your-space.signalwire.com",
    contexts => ["default"],
)

$client->on_call(sub {
    my ($call) = @_;
    $call->answer();

    # Start an Amazon Bedrock AI agent
    my $result = $call->amazon_bedrock(
        my $prompt = {"text": "You are a helpful assistant."},
        my $ai_params = {"barge_confidence": 0.02},
    )

});
$client->run();
```
