Transfer a SIP call to an external SIP endpoint using the SIP REFER method.
Unlike [`transfer()`][transfer]
which transfers control within RELAY, `refer()` performs a SIP-level transfer
to an external endpoint.

<Info>
This method emits [`calling.call.refer`][calling-call-refer] events. See [Call Events][call-events] for payload details.
</Info>

## **Parameters**

<ParamField path="device" type="\%hashref" required={true} toc={true}>
  The target SIP device for the REFER.
</ParamField>

<Indent>
<ParamField path="device.type" type="$scalar" required={true} toc={true}>
  Device type. Typically `"sip"`.
</ParamField>

<ParamField path="device.params" type="\%hashref" required={true} toc={true}>
  SIP parameters including `uri` (the SIP URI to refer to).
</ParamField>
</Indent>

<ParamField path="status_url" type="$scalar / undef" toc={true}>
  URL to receive REFER status webhooks.
</ParamField>

## **Returns**

`dict` -- Server response confirming the refer operation.

## **Example**

```perl {15}
use SignalWire::Relay::Client;

my $client = SignalWire::Relay::Client->new(
    project => "your-project-id",
    token => "your-api-token",
    host => "your-space.signalwire.com",
    contexts => ["default"],
)

$client->on_call(sub {
    my ($call) = @_;
    $call->answer();
    $call->play([{"type": "tts", "text": "Transferring to another line."}]);

    my $result = $call->refer(
        my $device = {
            "type": "sip",
            "params": {"uri": "sip:support@example.com"},
        },
    )
    print "SIP REFER result: ${result}";

});
$client->run();
```
