Echo audio back to the caller. This is primarily useful for testing audio
quality, latency, and connectivity. The caller hears their own voice repeated
back to them.

<Info>
This method emits [`calling.call.echo`][calling-call-echo] events. See [Call Events][call-events] for payload details.
</Info>

## **Parameters**

<ParamField path="timeout" type="$scalar / undef" toc={true}>
  Maximum duration of the echo in seconds. The echo stops automatically after
  this timeout.
</ParamField>

<ParamField path="status_url" type="$scalar / undef" toc={true}>
  URL to receive echo status webhooks.
</ParamField>

## **Returns**

`dict` -- Server response confirming the echo operation.

## **Example**

```perl {15}
use SignalWire::Relay::Client;

my $client = SignalWire::Relay::Client->new(
    project => "your-project-id",
    token => "your-api-token",
    host => "your-space.signalwire.com",
    contexts => ["default"],
)

$client->on_call(sub {
    my ($call) = @_;
    $call->answer();
    $call->play([{"type": "tts", "text": "Starting echo test. You should hear yourself."}]);

    my $result = $call->echo(timeout => 30);
    print "Echo completed: ${result}";

    $call->play([{"type": "tts", "text": "Echo test complete."}]);
    $call->hangup();

});
$client->run();
```
