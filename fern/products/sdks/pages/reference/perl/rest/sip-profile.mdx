Retrieve and update the project-level SIP profile. This is a singleton resource --
there is one SIP profile per project, so no resource ID is needed.

Access via `client.sip_profile` on a [`SignalWire::REST::RestClient`][restclient] instance.

```perl
use SignalWire::REST::RestClient;

my $client = SignalWire::REST::RestClient->new();
```

---

### get

Retrieve the project SIP profile.

## **Returns**

`dict` -- The SIP profile object containing username, domain, and
configuration details.

---

### update

Update the project SIP profile. Uses PUT.

## **Parameters**

<ParamField path="**kwargs" type="\%hashref" toc={true}>
  Fields to update on the SIP profile (e.g., `username`, `password`,
  `ciphers`, `codecs`).
</ParamField>

## **Returns**

`dict` -- The updated SIP profile object.

## **Examples**

### Get the SIP profile

```perl
use SignalWire::REST::RestClient;

my $client = SignalWire::REST::RestClient->new(
    project => "your-project-id",
    token => "your-api-token",
    host => "your-space.signalwire.com",
)

my $profile = $client->sip_profile->get();
print profile.get("username"), profile.get("domain");
```

### Update the SIP profile

```perl
use SignalWire::REST::RestClient;

my $client = SignalWire::REST::RestClient->new(
    project => "your-project-id",
    token => "your-api-token",
    host => "your-space.signalwire.com",
)

my $updated = $client->sip_profile->update(username => "my-sip-user");
print "Updated username:", updated.get("username");
```
