Query conference log entries. Conference logs provide a read-only view of
conference activity in your project.

Access via `client.logs.conferences` on a [`SignalWire::REST::RestClient`][restclient] instance.

```perl
use SignalWire::REST::RestClient;

my $client = SignalWire::REST::RestClient->new();
```

<Note>
Conference logs only support `list()`. There is no `get()` method for
individual conference entries.
</Note>

### list

List conference log entries.

## **Parameters**

<ParamField path="**params" type="\%hashref" toc={true}>
  Optional query parameters for filtering and pagination.
</ParamField>

## **Returns**

`dict` -- JSON response containing a `data` list of conference log objects.

## **Example**

```perl
use SignalWire::REST::RestClient;

my $client = SignalWire::REST::RestClient->new(
    project => "your-project-id",
    token => "your-api-token",
    host => "your-space.signalwire.com",
)

my $confs = $client->logs->conferences->list();
for my $conf (@{confs.get("data", [])}) {
    print conf.get("name"), conf.get("status");
```
