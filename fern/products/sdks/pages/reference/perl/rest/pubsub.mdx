Generate tokens for the SignalWire PubSub (publish-subscribe) real-time messaging
service. PubSub tokens authenticate client-side connections to subscribe to
channels and publish messages.

Access via `client.pubsub` on a [`SignalWire::REST::RestClient`][restclient] instance.

```perl
use SignalWire::REST::RestClient;

my $client = SignalWire::REST::RestClient->new();
```

---

### create_token

Create a PubSub authentication token.

## **Parameters**

<ParamField path="channels" type="\@arrayref" toc={true}>
  List of channel names the token grants access to.
</ParamField>

<ParamField path="member_id" type="$scalar" toc={true}>
  Identifier for the connecting member.
</ParamField>

<ParamField path="ttl" type="$integer" toc={true}>
  Token time-to-live in seconds.
</ParamField>

<ParamField path="**kwargs" type="\%hashref" toc={true}>
  Additional token parameters (e.g., `state`, `permissions`).
</ParamField>

## **Returns**

`dict` -- The token object containing the JWT token string.

## **Example**

```perl
use SignalWire::REST::RestClient;

my $client = SignalWire::REST::RestClient->new(
    project => "your-project-id",
    token => "your-api-token",
    host => "your-space.signalwire.com",
)

my $token = $client->pubsub->create_token(
    channels => ["notifications", "updates"],
    member_id => "user-123",
    ttl => 3600,
)
print "PubSub token:", token.get("token");
```
