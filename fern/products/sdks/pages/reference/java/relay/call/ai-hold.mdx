Put an active AI agent session on hold. The AI stops processing conversation
while the call remains active. The caller may hear hold music or silence
depending on the configuration.

<Info>
Use [`aiUnhold()`][ai-unhold] to resume the AI session.
</Info>

## **Parameters**

<ParamField path="timeout" type="@Nullable String" toc={true}>
  Maximum hold duration. The AI session automatically resumes after this timeout.
</ParamField>

<ParamField path="prompt" type="@Nullable String" toc={true}>
  A prompt for the AI to speak before going on hold (e.g., "Please hold
  while I check on that.").
</ParamField>

## **Returns**

`Map<String, Object>` -- Server response confirming the AI hold.

## **Example**

```java {20}
import com.signalwire.sdk.relay.RelayClient;

var client = RelayClient.builder()
        .project("your-project-id")
        .token("your-api-token")
        .host("your-space.signalwire.com")
        .contexts(List.of("default"))
        .build();

async def handle_call(call):
    await call.answer()

    // Start an AI agent
    var action = await call.ai( {"text": "You are a support agent."}, );

    // After some event, put the AI on hold
    await call.ai_hold( "One moment please while I look that up.", "30", )

    // Do some processing, then resume with ai_unhold()

client.run();


```
