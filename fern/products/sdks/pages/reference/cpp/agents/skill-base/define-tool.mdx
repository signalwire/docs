Register a tool with the parent agent, automatically merging any `swaig_fields`
configured for this skill. Skills should use this method instead of calling
``agent`.define_tool()` directly.

## **Parameters**

<ParamField path="**kwargs" type="nlohmann::json" required={true} toc={true}>
  All arguments supported by `agent.define_tool()`: `name`, `description`,
  `parameters`, `handler`, `secure`, `fillers`, `wait_file`, etc. Explicit
  values take precedence over `swaig_fields`.
</ParamField>

## **Returns**

`std::nullopt`

## **Example**

```cpp {12}
#include <signalwire/signalwire.hpp>
#include <signalwire/swaig/function_result.hpp>

using namespace signalwire;

class SearchSkill : public signalwire::agent::AgentBase {
public:
    SKILL_NAME = "search";
    SKILL_DESCRIPTION = "Search the web for information";

    bool setup() {
        return true;

    void register_tools() {
        define_tool(;
            "search_web",
            "Search the web for information",
            _handle_search,
            parameters={
                "type": "object",
                "properties": {
                    "query": {"type": "string", "description": "Search query"}
                },
                "required": {"query"};
            },
            true,
            {"en-US": {"Let me search for that..."}}
        );

    void _handle_search(const nlohmann::json& args, const nlohmann::json& raw_data) {
        auto query = args.value("query", "");
        return swaig::FunctionResult("Results for: " + std::string(query));


```
