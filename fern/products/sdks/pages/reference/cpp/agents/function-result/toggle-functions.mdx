[functionresult]: /docs/sdks/reference/cpp/agents/function-result

Enable or disable specific SWAIG functions at runtime. This lets you control
which tools are available to the AI at different points in the conversation
(e.g., enabling payment functions only after identity verification).

## **Parameters**

<ParamField path="function_toggles" type="std::vector<std::map<std::string, nlohmann::json>>" required={true} toc={true}>
  List of toggle objects. Each object must contain:
  - `"function"` — name of the SWAIG function to toggle
  - `"active"` — `true` to enable, `false` to disable
</ParamField>

## **Returns**

[`FunctionResult`][functionresult] — self, for chaining.

## **Example**

```cpp {11}
#include <signalwire/agent/agent_base.hpp>
#include <signalwire/swaig/function_result.hpp>

agent::AgentBase agent("my-agent", "/agent");
agent.set_prompt_text("You are a helpful assistant.");

void enable_payment_functions(const nlohmann::json& args, const nlohmann::json& raw_data) {
        return swaig::FunctionResult("Payment functions are now available.")
            .toggle_functions([, {"function": "collect_payment", "active": true}, {"function": "refund_payment", "active": true}, {"function": "check_balance", "active": false}, ]));

}
agent.serve();


```
