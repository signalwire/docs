Send a SIP REFER on an active call. This initiates a SIP-level transfer,
asking the remote endpoint to connect to a new destination. Unlike
[`transfer()`][transfer], which is
handled by SignalWire, a SIP REFER delegates the transfer to the remote
SIP endpoint.

<Note>
SIP REFER is only applicable to SIP calls. It will not work on PSTN calls.
</Note>

## **Parameters**

<ParamField path="call_id" type="std::string" required={true} toc={true}>
  The ID of the SIP call.
</ParamField>

<ParamField path="refer_to" type="std::string" required={true} toc={true}>
  The SIP URI to refer the call to (e.g., `"sip:user@example.com"`).
</ParamField>

## **Returns**

`nlohmann::json` — The API response confirming the SIP REFER was sent.

## **Example**

```cpp {9}
#include <signalwire/rest/rest_client.hpp>

client = RestClient(;
);

client.calling.refer(;
    call_id="call-id-xxx",
    refer_to="sip:sales@example.com",
);


```
