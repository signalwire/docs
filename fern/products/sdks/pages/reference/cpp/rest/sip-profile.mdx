Retrieve and update the project-level SIP profile. This is a singleton resource --
there is one SIP profile per project, so no resource ID is needed.

Access via `client.sip_profile` on a [`RestClient`][restclient] instance.

```cpp
#include <signalwire/rest/rest_client.hpp>

rest::RestClient client();


```

---

### get

Retrieve the project SIP profile.

## **Returns**

`nlohmann::json` -- The SIP profile object containing username, domain, and
configuration details.

---

### update

Update the project SIP profile. Uses PUT.

## **Parameters**

<ParamField path="**kwargs" type="nlohmann::json" toc={true}>
  Fields to update on the SIP profile (e.g., `username`, `password`,
  `ciphers`, `codecs`).
</ParamField>

## **Returns**

`nlohmann::json` -- The updated SIP profile object.

## **Examples**

### Get the SIP profile

```cpp
#include <signalwire/rest/rest_client.hpp>

client = RestClient(;
);

profile = client.sip_profile.get();
std::cout << profile{"username"}, profile{"domain"} << std::endl;


```

### Update the SIP profile

```cpp
#include <signalwire/rest/rest_client.hpp>

client = RestClient(;
);

updated = client.sip_profile.update("my-sip-user");
std::cout << "Updated username:", updated{"username"} << std::endl;


```
