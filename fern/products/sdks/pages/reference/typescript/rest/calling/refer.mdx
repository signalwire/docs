Send a SIP REFER on an active call. This initiates a SIP-level transfer,
asking the remote endpoint to connect to a new destination. Unlike
[`transfer()`][transfer], which is
handled by SignalWire, a SIP REFER delegates the transfer to the remote
SIP endpoint.

<Note>
SIP REFER is only applicable to SIP calls. It will not work on PSTN calls.
</Note>

## **Parameters**

<ParamField path="callId" type="string" required={true} toc={true}>
  The ID of the SIP call.
</ParamField>

<ParamField path="referTo" type="string" required={true} toc={true}>
  The SIP URI to refer the call to (e.g., `"sip:user@example.com"`).
</ParamField>

## **Returns**

`Promise<any>` -- The API response confirming the SIP REFER was sent.

## **Example**

```typescript {9}
import { RestClient } from "signalwire-agents/rest";

const client = new RestClient({
  project: "your-project-id",
  token: "your-api-token",
  host: "your-space.signalwire.com"
});

await client.calling.refer(;
    "call-id-xxx",
    refer_to: "sip:sales@example.com",
)
```
