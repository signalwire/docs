The `FabricNamespace` provides access to all SignalWire Fabric resources through the
[`RestClient`][restclient]. It organizes sub-resources for managing AI agents,
SWML scripts, RELAY applications, call flows, conference rooms, subscribers,
SIP infrastructure, cXML resources, and authentication tokens.

Access via `client.fabric` on a [`RestClient`][restclient] instance.

```typescript
import { RestClient } from "signalwire-agents/rest";

const client = new RestClient({
  project: "your-project-id",
  token: "your-api-token",
  host: "your-space.signalwire.com"
});

const agents = await client.fabric.aiAgents.list();
for (const agent of agents.data ?? []) {
  console.log(`${agent.name}: ${agent.id}`);
}
```

<Note>
Fabric resources use two update strategies. Resources like `aiAgents`, `swmlWebhooks`,
`sipGateways`, and `cxmlWebhooks` use PATCH for partial updates. Resources like
`swmlScripts`, `relayApplications`, `callFlows`, `conferenceRooms`, `subscribers`,
and others use PUT for full replacement updates. The SDK handles this automatically.
</Note>

## **Sub-resource Properties**

All typed sub-resources are available as properties on `client.fabric`:

| Property | Description |
|----------|-------------|
| `aiAgents` | AI agent resources (PATCH updates) |
| `swmlScripts` | SWML script resources (PUT updates) |
| `relayApplications` | RELAY application resources (PUT updates) |
| `callFlows` | Call flow resources with versioning |
| `conferenceRooms` | Conference room resources (PUT updates) |
| `subscribers` | Subscriber resources with SIP endpoints |
| `sipEndpoints` | Top-level SIP endpoint resources |
| `cxmlScripts` | cXML script resources (PUT updates) |
| `cxmlApplications` | cXML application resources (read-only) |
| `swmlWebhooks` | SWML webhook resources (PATCH updates) |
| `sipGateways` | SIP gateway resources (PATCH updates) |
| `cxmlWebhooks` | cXML webhook resources (PATCH updates) |
| `freeswitchConnectors` | FreeSWITCH connector resources (PUT updates) |
| `resources` | Cross-type generic resource operations |
| `addresses` | Read-only fabric address lookup |
| `tokens` | Subscriber, guest, invite, and embed token creation |
