Register a callback for debug events received at the `/debug_events` endpoint.
Use as a decorator. Both sync and async handlers are supported.

<Note>
[`enableDebugEvents()`][enable-debug-events] must be called before events will be delivered.
</Note>

## **Parameters**

<ParamField path="handler" type="(arg0: string, arg1: dict) => undefined" required={true} toc={true}>
  Callback function with signature `(event_type: str, data: dict)`.

  - `event_type` -- Event label string (e.g., `"barge"`, `"llm_error"`, `"session_start"`, `"step_change"`)
  - `data` -- Full event payload including `call_id`, `label`, and event-specific fields
</ParamField>

## **Returns**

`Callable` -- The handler function, unchanged. This allows `onDebugEvent` to be used as a
decorator.

## **Example**

```typescript {7}
import { AgentBase } from "@signalwire/sdk";

agent = AgentBase(name="debug-agent", route="/debug")
agent.setPromptText("You are a helpful assistant.")
agent.enableDebugEvents(level=1)

@agent.on_debug_event
function handleDebug(eventType, data) {
    if event_type == "llm_error":
        console.log(`LLM error: ${data}`)
    elif event_type == "barge":
        console.log(`Barge detected: ${data["barge_elapsed_ms"]}ms`)
    elif event_type == "step_change":
        console.log(`Step changed to: ${data["step"]}`)

agent.run()
```
