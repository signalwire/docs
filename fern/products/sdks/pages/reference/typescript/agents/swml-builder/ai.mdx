Add an `ai` verb to the main section. Starts an AI-powered conversation on the call.

<Note>
`promptText` and `promptPom` are mutually exclusive. Provide one or the other to
set the AI system prompt, but not both.
</Note>

## **Parameters**

<ParamField path="promptText" type="string | undefined" toc={true}>
  Plain text system prompt for the AI. Mutually exclusive with `promptPom`.
</ParamField>

<ParamField path="promptPom" type={"Record<string, unknown>[] | undefined"} toc={true}>
  Prompt Object Model (POM) structure for the AI prompt. A list of section dictionaries
  with keys like `"section"`, `"body"`, and `"bullets"`. Mutually exclusive with
  `promptText`.
</ParamField>

<ParamField path="postPrompt" type="string | undefined" toc={true}>
  Instructions for summarizing the call after the AI conversation ends.
</ParamField>

<ParamField path="postPromptUrl" type="string | undefined" toc={true}>
  URL where the post-prompt summary is sent via webhook.
</ParamField>

<ParamField path="swaig" type={"Record<string, unknown> | undefined"} toc={true}>
  SWAIG (SignalWire AI Gateway) configuration with tool function definitions and
  defaults. Structure: `{"defaults": {"web_hook_url": "..."}, "functions": [...]}`.
</ParamField>

<ParamField path="options" type="Record<string, unknown>" toc={true}>
  Additional AI verb parameters passed directly into the verb config (e.g.,
  `hints`, `languages`, `params`, `global_data`).
</ParamField>

## **Returns**

`this` -- The builder instance for method chaining.

## **Example**

```typescript
import { SwmlBuilder } from 'signalwire-agents';

const builder = new SwmlBuilder();
builder.addVerb('answer', {});
builder.addVerb('ai', {
  prompt: { text: 'You are a helpful assistant.' },
  post_prompt_url: 'https://example.com/summary',
});
```
