## HIPAA Compliance Guide

> **Summary**: Guidelines for building HIPAA-compliant voice AI applications with the SignalWire Agents SDK.

> **Disclaimer**: This guide provides technical guidance for implementing security controls relevant to HIPAA compliance. It is not legal advice. Healthcare organizations must consult with qualified healthcare compliance experts and legal counsel to ensure their specific implementation meets all applicable HIPAA requirements. Compliance is a shared responsibility between SignalWire, your organization, and your implementation choices.

### Overview

The Health Insurance Portability and Accountability Act (HIPAA) establishes requirements for protecting Protected Health Information (PHI). When building voice AI agents that handle healthcare data, you must implement appropriate safeguards to protect PHI throughout the call lifecycle.

#### What is PHI in Voice AI?

In the context of voice AI agents, PHI may include:

- Patient names, addresses, phone numbers
- Medical record numbers and account numbers
- Appointment details and scheduling information
- Prescription information
- Lab results and diagnoses
- Insurance information
- Any health-related information combined with patient identifiers

#### Shared Responsibility Model

| Layer | Responsibility |
|-------|---------------|
| SignalWire Platform | Infrastructure security, encryption in transit, secure data centers |
| Your Application | Access controls, audit logging, PHI handling, prompt design |
| Your Organization | Policies, training, BAA management, incident response |

#### Audit Logging

HIPAA requires audit trails for PHI access. Implement comprehensive logging:

```python
import logging
from datetime import datetime
from signalwire_agents import AgentBase
from signalwire_agents.core.function_result import SwaigFunctionResult

# Configure structured logging
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)
audit_logger = logging.getLogger('hipaa_audit')

class HIPAACompliantAgent(AgentBase):
    def __init__(self):
        super().__init__(name="hipaa-agent")
        self.add_language("English", "en-US", "rime.spore")

    def log_phi_access(self, action, user_id, resource_type, success, details=None):
        """Log PHI access for HIPAA audit trail."""
        audit_logger.info(
            "PHI_ACCESS",
            extra={
                "timestamp": datetime.utcnow().isoformat(),
                "action": action,
                "user_id": user_id,
                "resource_type": resource_type,
                "success": success,
                "details": self.sanitize_for_log(details)
            }
        )

    def sanitize_for_log(self, data):
        """Remove PHI from log entries."""
        if data is None:
            return None

        # Fields to redact
        sensitive_fields = [
            'ssn', 'social_security',
            'dob', 'date_of_birth',
            'address', 'phone',
            'medical_record', 'mrn',
            'diagnosis', 'prescription'
        ]

        sanitized = dict(data) if isinstance(data, dict) else {"value": "[REDACTED]"}
        for field in sensitive_fields:
            if field in sanitized:
                sanitized[field] = "[REDACTED]"

        return sanitized
```

Required audit log events:

| Event | Required Fields |
|-------|-----------------|
| Call Start | Timestamp, call_id, caller_id (hashed), agent_name |
| PHI Access | Timestamp, action, resource_type, user_id, success/failure |
| Function Call | Timestamp, function_name, sanitized_args, result_status |
| Authentication | Timestamp, auth_type, success/failure, source_ip |
| Call End | Timestamp, call_id, duration, disposition |

#### Recording Retention

Call recordings containing PHI are considered part of the medical record. HIPAA does not specify a retention period for PHI itself—retention requirements come from state medical records laws, other federal regulations (e.g., Medicare conditions of participation), and organizational policies.

> **Note**: The HIPAA six-year retention requirement (45 CFR 164.530(j)) applies to administrative documentation—policies, procedures, training records, risk assessments, and audit logs—not to PHI or call recordings.

| Consideration | Recommendation |
|---------------|----------------|
| Retention Period | Follow state medical records laws and organizational policy (commonly 7-10 years for adults; longer for minors) |
| Storage | Encrypted at rest, access-controlled |
| Destruction | Secure deletion with audit trail when retention period expires |
| Access Logging | Log all recording access |

### PHI Handling Best Practices

#### Data Compartmentalization with global_data

One of the most effective ways to protect PHI is to keep it away from the LLM entirely. Use `global_data` to store sensitive information that your SWAIG functions need but the AI does not:

```python
class CompartmentalizedHealthcareAgent(AgentBase):
    def __init__(self):
        super().__init__(name="compartmentalized-healthcare")
        self.add_language("English", "en-US", "rime.spore")

    def verify_patient(self, args, raw_data):
        """Verify patient and store PHI in global_data, not in AI context."""
        dob = args.get("date_of_birth")
        ssn_last4 = args.get("ssn_last4")

        # Fetch full patient record from database
        patient = self.lookup_patient(dob, ssn_last4)

        if patient:
            # Store sensitive data in global_data - LLM cannot see this
            # Only your SWAIG functions can access it
            return (
                SwaigFunctionResult("I've verified your identity. How can I help?")
                .set_global_data({
                    "patient_id": patient["id"],
                    "mrn": patient["medical_record_number"],
                    "full_name": patient["full_name"],
                    "ssn": patient["ssn"],
                    "insurance_id": patient["insurance_id"],
                    "verified": True
                })
            )

        return SwaigFunctionResult("I couldn't verify your identity.")

    def schedule_appointment(self, args, raw_data):
        """Use stored PHI without exposing it to the LLM."""
        # Retrieve PHI from global_data - never sent to LLM
        global_data = raw_data.get("global_data", {})

        if not global_data.get("verified"):
            return SwaigFunctionResult("Please verify your identity first.")

        # Use the stored patient_id and MRN for scheduling
        patient_id = global_data.get("patient_id")
        mrn = global_data.get("mrn")

        # Schedule using real identifiers, but LLM only knows
        # "the verified patient" - not the actual PHI
        appointment = self.create_appointment(patient_id, args.get("date"))

        # Return confirmation without exposing PHI to AI
        return SwaigFunctionResult(
            f"Your appointment is confirmed for {args.get('date')}. "
            "You'll receive a confirmation at the contact information on file."
        )
```

The `global_data` pattern ensures:

| Data Location | LLM Access | SWAIG Function Access |
|---------------|------------|----------------------|
| Conversation context | Yes | Yes |
| Function arguments | Yes | Yes |
| Function response text | Yes | Yes |
| `global_data` | **No** | Yes |

#### Collecting PHI via IVR (Bypassing the LLM)

For maximum PHI protection, collect sensitive data using traditional IVR methods (DTMF or voice input) that bypass the LLM entirely. Use `prompt()` and `input()` SWML actions with `ai_response` to return control to the AI afterward:

```python
def collect_ssn(self, args, raw_data):
    """Collect SSN via DTMF - data never touches the LLM."""
    return (
        SwaigFunctionResult()
        # Use prompt() to speak and collect DTMF input
        .prompt(
            play="Please enter your 9-digit Social Security Number using your keypad.",
            input_type="digits",
            num_digits=9,
            timeout=10,
            # Store result in global_data under "ssn_input"
            result_name="ssn_input"
        )
        # After collection, return to AI with this response
        # The AI never sees the actual SSN digits
        .ai_response("Thank you for entering your information. Let me verify that.")
    )

def collect_dob_voice(self, args, raw_data):
    """Collect date of birth via voice - processed directly, not by LLM."""
    return (
        SwaigFunctionResult()
        .prompt(
            play="Please say your date of birth, including the month, day, and year.",
            input_type="speech",
            timeout=10,
            result_name="dob_input"
        )
        .ai_response("Thank you. I'm verifying your information now.")
    )

def collect_credit_card(self, args, raw_data):
    """Collect payment info via DTMF for PCI/HIPAA compliance."""
    return (
        SwaigFunctionResult()
        # Stop recording during payment collection
        .stop_record_call(control_id="main_recording")
        .prompt(
            play="Please enter your 16-digit card number using your keypad.",
            input_type="digits",
            num_digits=16,
            timeout=30,
            result_name="card_number"
        )
        .prompt(
            play="Please enter the 4-digit expiration date. For example, 0 3 2 6 for March 2026.",
            input_type="digits",
            num_digits=4,
            timeout=15,
            result_name="card_expiry"
        )
        .prompt(
            play="Please enter the 3 or 4 digit security code from the back of your card.",
            input_type="digits",
            num_digits=4,
            min_digits=3,
            timeout=15,
            result_name="card_cvv"
        )
        # Resume recording after payment collection
        .record_call(control_id="main_recording", stereo=True)
        # Return to AI - it only knows payment was collected, not the details
        .ai_response("Thank you. I'm processing your payment now.")
    )

def process_collected_data(self, args, raw_data):
    """Process the collected IVR data from global_data."""
    global_data = raw_data.get("global_data", {})

    # SSN was collected via DTMF, stored in global_data
    ssn = global_data.get("ssn_input")
    dob = global_data.get("dob_input")

    # Process the data - LLM never had access to it
    verified = self.verify_in_database(ssn, dob)

    if verified:
        return (
            SwaigFunctionResult("Your identity has been verified.")
            .set_global_data({"verified": True, "ssn": ssn})
        )

    return SwaigFunctionResult("Verification failed. Please try again.")
```

This approach provides several layers of protection:

| Collection Method | LLM Sees Data | Transcript Contains | Recording Contains |
|-------------------|---------------|---------------------|-------------------|
| Normal conversation | Yes | Yes | Yes |
| DTMF via `prompt()` | **No** | **No** | Tones only |
| Voice via `prompt()` | **No** | **No** | Yes (pause recording to exclude) |
| `global_data` storage | **No** | **No** | **No** |

#### Secure Function Design

Design SWAIG functions to minimize PHI in responses:

```python
def check_appointment(self, args, raw_data):
    """Check appointment - returns minimal PHI."""
    patient_id = args.get("patient_id")

    # Verify caller is authorized
    if not self.verify_caller_authorization(raw_data, patient_id):
        self.log_phi_access("appointment_check", patient_id, "appointment", False)
        return SwaigFunctionResult(
            "I was unable to verify your identity. "
            "Please contact us directly for assistance."
        )

    # Log the access
    self.log_phi_access("appointment_check", patient_id, "appointment", True)

    # Return minimal information
    appointment = self.get_next_appointment(patient_id)
    if appointment:
        # Don't include provider names or detailed location
        return SwaigFunctionResult(
            f"Your next appointment is on {appointment['date']} "
            f"at {appointment['time']}. "
            f"Please arrive 15 minutes early."
        )

    return SwaigFunctionResult("You have no upcoming appointments scheduled.")
```

```python
import re

def validate_patient_id(self, patient_id):
    """Validate patient ID format."""
    # Only allow alphanumeric, specific length
    if not patient_id:
        return False
    if not re.match(r'^[A-Z0-9]{8,12}$', patient_id):
        return False
    return True

def validate_date_of_birth(self, dob):
    """Validate DOB format."""
    try:
        # Expected format: MM/DD/YYYY
        if not re.match(r'^\d{2}/\d{2}/\d{4}$', dob):
            return False
        # Additional validation...
        return True
    except Exception:
        return False
```

```python
class RoleBasedHealthcareAgent(AgentBase):
```

```python
"""HIPAA-compliant healthcare appointment agent."""

import os
import logging
from datetime import datetime
from signalwire_agents import AgentBase
from signalwire_agents.core.function_result import SwaigFunctionResult

# Audit logging setup

class HIPAAHealthcareAgent(AgentBase):
```
