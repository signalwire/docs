SignalWire AI is a programmable, real-time conversational voice AI capability for building
**AI Agents**: software that talks with people in natural language over the phone, SIP, and the web.
An AI Agent listens, understands, responds in a natural voice, and takes action by calling your
own functions, all within a single conversation.

## What is SignalWire AI?

AI is a capability of the SignalWire platform, not a separate product you have to wire up. The
same platform that handles your voice, SIP, and messaging traffic also runs your AI Agents. The
result is a single system where an agent can answer a call, look something up, send a text, and
transfer to a human, all without leaving SignalWire.

You can build an AI Agent in three ways: with [SWML](/docs/swml), the
[Server SDK](/docs/server-sdks), or the [no-code dashboard](/docs/platform/ai/no-code-agents).

### Where it's used

AI Agents handle the voice conversations businesses have every day: customer support, appointment
scheduling, receptionists that route calls, lead qualification, surveys, and concierge services.
Because an agent can call your functions, it can complete real tasks during the call rather than
just reading a script. See the [capabilities guide](/docs/platform/ai/capabilities) for a deeper
tour of everything they have to offer.

## How SignalWire AI works

Under the hood, the **SignalWire AI Gateway ([SWAIG](/docs/swml/guides/swaig))** orchestrates the
services that make real-time voice AI possible. When a caller connects, SWAIG runs a continuous
conversation loop and coordinates speech recognition, the language model, voice synthesis, and any
functions your agent calls.

### The conversation loop

Each turn of the conversation flows through the same pipeline:

1. **Speech recognition.** The caller's speech is transcribed in real time. SignalWire
   uses Deepgram's Nova-3 model by default and detects when the caller has finished speaking so the
   agent can respond at the right moment.
2. **Language model.** The transcript, your prompt, and the conversation so far are sent
   to a large language model (default `gpt-4o-mini`) that decides what to say and whether to call a
   function.
3. **Function calls.** If the model needs to act, it calls a SWAIG function. SignalWire
   sends structured JSON to your webhook, your backend runs its logic, and the response feeds back
   into the conversation.
4. **Voice synthesis.** The model's reply is spoken back to the caller using a text-to-speech
   voice from providers such as ElevenLabs, Cartesia, Rime, Deepgram, Google, or Polly.

The loop repeats for every turn until the call ends.

### The building blocks

- **AI Agent**: the entity, created by SWML's `ai` method or the Server SDK, that conducts the
  conversation.
- **Prompt**: defines the agent's personality, goals, and instructions. Write it as plain text, as
  a structured Prompt Object Model, or as multi-step [Contexts](/docs/swml/reference/calling/ai/prompt)
  for staged conversations.
- **LLM**: the reasoning engine that generates responses, configurable per agent.
- **ASR / STT**: automatic speech recognition that turns caller audio into text.
- **TTS providers**: the voices that speak the agent's responses, selectable per language.
- **SWAIG functions**: developer-defined webhook functions, prebuilt native functions, and
  Model Context Protocol tools the agent can call to reach external systems and databases.
- **Datasphere**: SignalWire's built-in retrieval-augmented generation knowledge base, so
  answers can be grounded in your own documents. See the [capabilities guide](/docs/platform/ai/capabilities).
- **Post-prompt and conversation logging**: an end-of-call summary plus full conversation and
  function logs for analytics and records.

### Natural conversation features

Real conversations are messy, so the platform handles the hard parts for you.
[Barge-in](/docs/swml/reference/calling/ai/params#paramsenable_barge) lets callers interrupt the
agent mid-sentence, [end-of-utterance detection](/docs/swml/reference/calling/ai/params#paramsend_of_speech_timeout)
keeps the agent from cutting people off, [filler words](/docs/swml/reference/calling/ai/languages#languagesspeech_fillers)
cover brief processing pauses, and [background audio](/docs/swml/reference/calling/ai/params#paramsbackground_file)
provides ambient feedback. Together these keep the conversation feeling human.

## Why it's easier to build with SignalWire

Standing up a voice AI agent elsewhere usually means stitching together a telephony provider, a
speech-to-text service, an LLM, a text-to-speech service, and the servers to run it all. On
SignalWire, that stack is already integrated and serverless: you describe the agent, and the
platform runs it. You stay free to choose your own LLM and voice providers.

Here is a complete customer service agent that looks up accounts and transfers to a human. You can
build and call it in about ten minutes.

<Tabs>
<Tab title="SWML">

```yaml
version: 1.0.0
sections:
  main:
    - ai:
        prompt:
          text: |
            You are a customer service agent. Help customers with their questions and account needs.
            Use the available functions to look up information or transfer calls when needed.
        SWAIG:
          functions:
            - function: lookup_account
              description: Look up customer account information
              parameters:
                type: object
                properties:
                  account_id:
                    type: string
                    description: Customer account ID
              web_hook_url: https://example.com/account-lookup
            - function: transfer_to_support
              description: Transfer to human support agent
              web_hook_url: https://example.com/transfer-support
```

[SWML docs](/docs/swml) | [AI method reference](/docs/swml/reference/calling/ai)

</Tab>
<Tab title="Server SDK">

```python
from signalwire import AgentBase, FunctionResult

agent = AgentBase(name="support")
agent.prompt_add_section("Role", "You are a helpful customer service agent.")

@agent.tool(description="Look up customer account")
def lookup_account(account_id: str) -> FunctionResult:
    # Simulate database lookup
    customer = {"name": "John Doe", "status": "active"}
    return FunctionResult(f"Account: {customer['name']}, Status: {customer['status']}")

@agent.tool(description="Transfer to support")
def transfer_support() -> FunctionResult:
    return FunctionResult("Connecting you to support.").connect("+15551234567")

if __name__ == "__main__":
    agent.run()
```

## Example agents

The same pattern scales to any use case. Here is an appointment scheduling agent that checks
availability, books the appointment, and sends an SMS confirmation.

<Tabs>
<Tab title="SWML">

```yaml
version: 1.0.0
sections:
  main:
    - ai:
        prompt:
          text: |
            You are an appointment scheduling agent. Help customers book appointments,
            check availability, and send confirmations.
        SWAIG:
          functions:
            - function: check_availability
              description: Check if a time slot is available
              parameters:
                type: object
                properties:
                  date:
                    type: string
                    description: Date for appointment
                  time:
                    type: string
                    description: Preferred time
              web_hook_url: https://example.com/check-availability
            - function: book_appointment
              description: Book an appointment
              parameters:
                type: object
                properties:
                  name:
                    type: string
                    description: Customer name
                  phone:
                    type: string
                    description: Customer phone number
                  date:
                    type: string
                    description: Appointment date
                  time:
                    type: string
                    description: Appointment time
              web_hook_url: https://example.com/book-appointment
```

[SWML docs](/docs/swml) | [SWAIG functions guide](/docs/swml/guides/swaig)

</Tab>
<Tab title="Server SDK">

```python
from signalwire import AgentBase, FunctionResult
from datetime import datetime

appointments = []

agent = AgentBase(name="scheduler", route="/scheduler")
agent.prompt_add_section("Role", "You help customers schedule appointments.")
agent.prompt_add_section("Guidelines", """
- Collect customer name, date, and preferred time
- Confirm all details before booking
- Send SMS confirmation when booking is complete
""")
agent.add_language("English", "en-US", "rime.spore")

@agent.tool(description="Check if a time slot is available")
def check_availability(date: str, time: str) -> FunctionResult:
    for apt in appointments:
        if apt["date"] == date and apt["time"] == time:
            return FunctionResult(f"Sorry, {date} at {time} is not available.")
    return FunctionResult(f"{date} at {time} is available.")

@agent.tool(description="Book an appointment")
def book_appointment(
    name: str,
    phone: str,
    date: str,
    time: str
) -> FunctionResult:
    appointments.append({
        "name": name,
        "phone": phone,
        "date": date,
        "time": time,
        "booked_at": datetime.now().isoformat()
    })
    return (
        FunctionResult(f"Appointment booked for {name} on {date} at {time}.")
        .send_sms(
            to_number=phone,
            from_number="+15559876543",
            body=f"Your appointment is confirmed for {date} at {time}."
        )
    )

if __name__ == "__main__":
    agent.run()
```
