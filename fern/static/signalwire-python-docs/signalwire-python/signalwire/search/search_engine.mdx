<Indent>

Calculate final score using max-signal-wins approach.

The strongest signal (vector, keyword, filename, or metadata) becomes
the base score. Agreement from additional sources boosts it. This keeps
scores in an intuitive 0-1 range where 0.5 means 'decent match'
regardless of which source found the result.


</Indent>

<Indent>

Perform improved search with fast filtering and vector re-ranking

Strategy:
1. Fast candidate collection (filename, metadata, keywords)
2. Vector re-ranking on candidates only
3. Fallback to full vector search if few candidates

**Parameters:**

<ParamField path="query_vector" type="List[float]">
Embedding vector for the query
</ParamField>

<ParamField path="enhanced_text" type="str">
Processed query text for keyword search
</ParamField>

<ParamField path="count" type="int" default="3">
Number of results to return
</ParamField>

<ParamField path="similarity_threshold" type="float" default="0.0">
Minimum similarity score
</ParamField>

<ParamField path="tags" type="Optional[List[str]]" default="None">
Filter by tags
</ParamField>

<ParamField path="keyword_weight" type="Optional[float]" default="None">
Optional manual weight for keyword vs vector
</ParamField>

<ParamField path="original_query" type="Optional[str]" default="None">
Original query for exact matching
</ParamField>

**Returns:** `List[Dict[str, Any]]`

List of search results with scores and metadata


</Indent>
</Indent>
