# TTS Engine Reference

This document provides a comprehensive breakdown of all Text-to-Speech (TTS) engine, voice, and model combinations supported by mod_openai.

## Voice Format Convention

The general format for specifying voices is:
```
engine.voice:model
```

Where:
- `engine` - The TTS provider (e.g., elevenlabs, rime, openai)
- `voice` - The voice identifier (name or ID depending on engine)
- `model` - Optional model variant (not all engines support this)

### 2. Rime AI (`rime`)

**Format**: `rime.<speaker>:<model>`

**Models**:
| Model | Allowed Parameters |
|-------|-------------------|
| `mist` | pauseBetweenBrackets, lang, phonemizeBetweenBrackets, speedAlpha |
| `mistv2` | **Default** - Same as mist |
| `arcana` | repetition_penalty, temperature, top_p, max_tokens |

**Language Support** (for non-mist models):
- `eng` - English (default)
- `deu` - German
- `fra` - French
- `spa` - Spanish
- `cmn` - Chinese Mandarin
- `kor` - Korean
- `jpn` - Japanese
- `hin` - Hindi
- `ita` - Italian
- `por` - Portuguese (Brazil)
- `pol` - Polish
- `tur` - Turkish
- `rus` - Russian
- `nld` - Dutch

**Speed Options** (for rime):
- `slowest`, `slower`, `slow`, `normal`, `fast`, `faster`, `fastest`

**Examples**:
```
rime.spore:arcana
rime.speaker1:mistv2
rime.voice123:mist
```

```
openai.shimmer:tts-1
```

---

### 4. Azure Cognitive Services (`azure`)

**Format**: `azure.<voice>`

Azure TTS uses SSML-based configuration and does not have a separate model parameter.

**Voice Format**: Azure neural voice names (e.g., `en-US-RyanMultilingualNeural`)

**Configuration**:
- Requires `AZURE_TTS_KEY` environment variable or `azure_tts_key` parameter
- Requires `AZURE_TTS_HOST` environment variable or `azure_tts_host` parameter

**Examples**:
```
azure.en-US-RyanMultilingualNeural
azure.en-US-JennyNeural
azure.de-DE-ConradNeural
```

---

### 5. Deepgram Aura (`deepgram`)

**Format**: `deepgram.<voice>`

Deepgram uses URL-based voice specification; the voice is passed as part of the API URL rather than the JSON body.

**Configuration**:
- Requires `DEEPGRAM_KEY` environment variable

**Examples**:
```
deepgram.aura-asteria-en
deepgram.aura-luna-en
deepgram.aura-stella-en
```

---

### 6. Cartesia Sonic (`cartesia`)

**Format**: `cartesia.<voice_id>:<model>`

**Models**:
| Model | Description |
|-------|-------------|
| `sonic-english` | English-optimized |
| `sonic-multilingual` | Multilingual support |
| `sonic-2` | Version 2 |
| `sonic-3` | Version 3 |
| `sonic-turbo` | Low-latency turbo |
| `sonic-v3` | **Default** - Latest version |

**Voice**: Uses Cartesia voice UUIDs

**Additional Parameters**:
- `speed` (e.g., "normal", "slow", "fast", or numeric)
- `emotion` (supports auto-emotion feature)
- `max_emotion` (default: 30)

**Output Formats**:
- MP3 (container: mp3, encoding: mp3)
- WAV (container: wav, encoding: pcm_s16le)

**Examples**:
```
cartesia.a0e99841-438c-4a64-b679-ae501e7d6091:sonic-v3
cartesia.voice-uuid-here:sonic-turbo
cartesia.custom-voice:sonic-multilingual
```

---

### 7. Amazon TTS (`amazon`)

**Format**: `amazon.<model>.<voice>` or `amazon.<voice>`

**Billing SKUs**:
| SKU | Description |
|-----|-------------|
| `amazon_tts_standard` | Standard voices |
| `amazon_tts_neural` | Neural voices |
| `amazon_tts_generative` | Generative voices |

**Examples**:
```
amazon.neural.Joanna
amazon.generative.Matthew
amazon.Joanna
```

---

### 8. Google Cloud TTS (`gcloud`)

**Format**: `gcloud.<voice>`

Standard FreeSWITCH TTS interface - used as fallback engine.

**Billing SKUs**:
- `gcloud` (standard billing)
- `gcloud_cog` (cognitive billing)

**Examples**:
```
gcloud.en-US-Neural2-J
gcloud.en-US-Wavenet-D
gcloud.en-GB-Neural2-A
```
