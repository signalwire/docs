# Actions

Actions represent asynchronous operations that can be performed on calls. Unlike synchronous methods, actions return immediately and provide an object you can use to monitor and control the operation.

## Available Actions
- **[Detect][detect]** - Detect DTMF, fax tones, or answering machines
- **[Fax][fax]** - Send or receive fax documents
- **[Play][play]** - Play audio, TTS, or silence
- **[Prompt][prompt]** - Play media while collecting user input
- **[Record][record]** - Record call audio
- **[Send Digits][send-digits]** - Send DTMF digits
- **[Tap][tap]** - Stream call media to external endpoints

## Common Action Methods

All actions share common methods for control and monitoring:

### getCompleted

Returns `true` if the action has finished, `false` otherwise.

### getControlID

Returns the UUID that identifies the action.

### stop

Stops the action immediately.

### getResult

Returns the final result of the action (available after completion).
