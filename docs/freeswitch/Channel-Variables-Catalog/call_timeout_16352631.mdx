
# call_timeout 

 Created by  Ryan Harris, last modified by  John Boteler on 2022.06.14

## call_timeout (deprecated)

integer Controls how long (in seconds) to ring the B leg of a call when using the bridge application. The timeout is set on the A leg, and applies to any bridges that happen in the channel.

If you need to set a timeout on a call that has no A leg, use [originate\_timeout](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#originate_timeout).

If you need to set a timeout with enterprise bridging/originate, use [originate\_timeout](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#originate_timeout).

If you need to set the timeout on a per-leg basis (i.e. a different timeout for each destination), use the [leg\_timeout](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#leg_timeout) variable.

Beware that if you are not using `{`[ignore\_early\_media](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#ignore_early_media)`=true}`, `call_timeout` is no longer applicable as soon as early media signal is received.

  
**Default Value:** 60

**Usage**

```xml
<action application="set" data="call_timeout=20"/>
```

If a call timeout is to be specified against a group\_call() list, use the following format:

```xml
<action application="bridge" data="{originate_timeout=24}${group_call(sales@$${domain})}"/>
```
