
# hold_hangup_xfer_exten 

 Created by  Ryan Harris, last modified on 2018.02.07

## hold_hangup_xfer_exten

string Controls what happens to a calling (A) party when in a bridge state and the bridge ends while the called (B) party is on hold. If not set on leg B (ie. the default), then A leg is hung up. If it is set on leg B, then leg A is transferred to the given extension, as per [transfer\_after\_bridge](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#transfer_after_bridge).

**Usage**

```xml
<action application="set" data="hold_hangup_xfer_exten=1000:XML:default"/>
```
