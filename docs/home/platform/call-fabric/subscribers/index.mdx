# Subscribers Overview

<Subtitle>The foundation of user management in Call Fabric</Subtitle>

## What are Subscribers?

Subscribers are one of the most fundamental and valuable [Resources][resources] in SignalWire's Call Fabric platform. While you can think of a Subscriber as a "user," the concept is actually broader, encompassing comprehensive features and services for onboarding, management, and utilization of Call Fabric services.

Subscribers embody SignalWire's approach to **Programmable Unified Communications (PUC)**, enabling developers to quickly and securely onboard and manage large numbers of users while saving months of development and testing time.

## Technical Architecture

<AccordionGroup>
  <Accordion 
    title="ðŸ“ Resource Addressing" 
    description="Unified addressing system for all Call Fabric Resources"
    defaultOpen={true}
  >
    Subscribers use the unified Call Fabric addressing system:
    
    **Address Format**: `/<Context>/<Name>`
    
    ```
    /private/john-doe           # Default private context
    /public/support-agent       # Public context with alias
    /private/jane?channel=video # Video channel specification
    ```
    
    - **Default context**: `private` for security by default
    - **Multiple addresses**: Subscribers can have multiple Resource addresses
    - **Channel specification**: Support for audio/video channels (`?channel=audio`)
  </Accordion>

  <Accordion 
    title="ðŸ”— API Integration" 
    description="Complete REST API for Subscriber management"
  >
    **Full CRUD Operations**:
    - Create, read, update, delete Subscribers
    - Bulk operations for large deployments
    - Webhook integration for real-time events
    - Token-based authentication for secure access
    
    **Supported Operations**:
    ```bash
    POST   /api/fabric/subscribers     # Create subscriber
    GET    /api/fabric/subscribers     # List subscribers  
    PUT    /api/fabric/subscribers/:id # Update subscriber
    DELETE /api/fabric/subscribers/:id # Delete subscriber
    ```
  </Accordion>

  <Accordion 
    title="ðŸ” Authentication Tokens" 
    description="Multiple token types for secure access"
  >
    **Token Types**:
    - **Subscriber Tokens**: Basic authentication for individual users
    - **Guest Tokens**: Server-side API integration using existing API authorization
    - **Invite Tokens**: Secure user invitation workflows
    - **Refresh Tokens**: Automatic token renewal and session management
    
    Each token type provides different levels of access and is designed for specific use cases in your application architecture.
  </Accordion>
</AccordionGroup>

## Getting Started

Ready to implement Subscribers in your application? Follow these steps:

<Steps>

### Create Your First Subscriber

Navigate to the [SignalWire Dashboard][create-subscribers] and create your first Subscriber through the Resources pane.

### Configure Authentication

Set up user credentials, contact information, and Resource addresses for your Subscriber's communication needs.

### Set Up Contexts

Configure private and public contexts based on your security requirements and communication patterns.

### Integrate with Your Application  

Use the Browser SDK or REST API to connect your application with the Subscriber system and start handling authentication.

### Monitor and Scale

Use the unified logging system to monitor usage, debug issues, and scale your user base as your application grows.

</Steps>
