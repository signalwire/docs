# live_translate

Start live translation of the call. The translation will be sent to the specified webhook URL.

<APIField
  name="live_translate"
  type="object"
  required={true}
>
  An object that accepts the [`live_translate parameters`](#live_translate-parameters).
</APIField>

## **live_translate Parameters**

<APIField
  name="live_translate.action"
  type="string | object"
  required={true}
>
  The action to be performed.<br/>An object that contains **one** of the [`live_translate actions`](./action/index.mdx)<br/>**Possible Values:** [[`stop`](/swml/methods/live_translate/action/stop), [`start`](/swml/methods/live_translate/action/start), [`summarize`](/swml/methods/live_translate/action/summarize), [`inject`](/swml/methods/live_translate/action/inject)]
</APIField>

## Action Usage Context

| Action | Call Start | Live Call |
|--------|-----------|-----------|
| `start` | ✅ Primary use | ✅ Can start mid-call |
| `stop` | ❌ No session to stop | ✅ Designed for this |
| `summarize` | ❌ No content to summarize | ✅ Designed for this |
| `inject` | ❌ No session exists | ✅ Designed for this |

**Call Start:** The initial SWML document returned when a call first arrives.

**Live Call:** Actions sent to active calls via:
- The [Call Commands REST API](/rest/signalwire-rest/endpoints/calling/call-commands)
- SWML sections executed via [`transfer`](/swml/methods/transfer) or [`execute`](/swml/methods/execute) during a call

:::info ai_summary vs summarize action
- **`ai_summary: true`** (in `start`): Automatically generates summary when session **ends**
- **`summarize` action**: On-demand summary **during** an active session
:::
