# data_map Variable Scopes

Variable scopes available in SWAIG `data_map` contexts. These scopes are only accessible within AI agent SWAIG functions.

For detailed usage examples and best practices, see the [data_map Guide](/swml/guides/ai/swaig/functions/data_map).

## Variable Scopes

<APIField
  name="args"
  type="object"
>
  Function arguments passed to the SWAIG function. Contains the structured arguments defined by the function's argument schema.

  Properties are defined by the SWAIG function's `argument.properties` schema. Access them using dot notation: `args.key_name`

  **Example keys:** `args.target`, `args.type`, `args.location`, `args.customer_id`

  **Availability:** SWAIG function data_map contexts only

  **Example:**
  ```yaml
  - pattern: .*
    string: '${args.target}'
  ```
</APIField>

<APIField
  name="meta_data"
  type="object"
>
  Function-level metadata that persists across function calls within an AI conversation. Can be initialized in the function definition or modified using SWAIG actions.

  Properties are user-defined and persist for the duration of the AI session. Access them using dot notation: `meta_data.key_name`

  **Example keys:** `meta_data.table.support`, `meta_data.user_preferences`, `meta_data.context`

  **Availability:** SWAIG functions only

  **Example:**
  ```yaml
  meta_data:
    table:
      support: '+1XXXXXXXXXX'
      sales: '+1YYYYYYYYYY'

  # Access in data_map:
  string: '${meta_data.table.${lc:args.target}}'
  ```
</APIField>

<APIField
  name="global_data"
  type="object"
>
  Application-wide data accessible across all functions in an AI agent. Can be set using the `set_global_data` SWAIG action.

  Properties are user-defined and available throughout the AI application across all function calls and sessions. Access them using dot notation: `global_data.key_name`

  **Example keys:** `global_data.api_key`, `global_data.config`, `global_data.settings`

  **Availability:** SWAIG functions only

  **Example:**
  ```yaml
  # Access in data_map:
  string: '${global_data.api_endpoint}'
  ```
</APIField>

<APIField
  name="prompt_vars"
  type="object"
>
  Built-in template variables in data_map contexts that provide call and AI session information. These are automatically populated by the system.

  **Availability:** SWAIG data_map expressions

  **Properties:**
  - `prompt_vars.call_direction` (string) - Call direction (inbound/outbound)
  - `prompt_vars.caller_id_name` (string) - Caller name
  - `prompt_vars.caller_id_number` (string) - Caller number
  - `prompt_vars.local_date` (string) - Current local date
  - `prompt_vars.local_time` (string) - Current local time
  - `prompt_vars.time_of_day` (string) - Time of day (morning/afternoon/evening)
  - `prompt_vars.supported_languages` (string[]) - List of supported languages
  - `prompt_vars.default_language` (string) - Default language

  **Example:**
  ```yaml
  response: 'The current time is ${prompt_vars.local_time}'
  ```
</APIField>
