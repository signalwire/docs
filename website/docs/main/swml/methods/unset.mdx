Unset specified variables. The variables have been set either using the [`set`](./set.mdx) command
or as a byproduct of some other statements or methods (like [`record`](/swml/methods/record#variables))

<APIField
  name="unset"
  type="object"
  required={true}
>
  An object that contains the [`unset parameters`](#unset-parameters).
</APIField>

## **unset Parameters**

<APIField
  name="unset.vars"
  type="string | string[]"
  required={true}
>
  Names of the variables to unset.
</APIField>

## **Variable**

Any variable can be unset by this method.

## **Examples**

### Unset a single variable

```yaml andJson
version: 1.0.0
sections:
  main:
    - set:
        num_var: 1
    - play:
        url: 'say: The value of num_var is: ${num_var}.'
    - unset:
        vars: num_var
    - play:
        url: 'say: The value of num_var is ${num_var}.'
```

### Unset multiple variables

```yaml andJson
version: 1.0.0
sections:
  main:
    - set:
        systems:
          hr:
            - tracy
            - luke
          engineering:
            john: absent
        name: john
    - play:
        url: 'say: ${systems.hr}'
    - unset:
        vars:
          - systems
          - name
    # this play statement emits an error because `systems` is undefined
    # at this point so there's nothing for `play` to say.
    - play:
        url: 'say: ${systems}'
```
