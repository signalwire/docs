The `parameters` object is used to define the input data that will be passed to the function.

<APIField
  name="parameters"
  type="object"
>
  An object that contains the [`parameters`](#parameters)
</APIField>

## Parameters

The `parameters` object defines the function parameters that will be passed to the AI.

<APIField
  name="type"
  type="string"
  required={true}
>
  Defines the top-level type of the parameters. Must be set to `"object"`
</APIField>

<APIField
  name="properties"
  type="object"
  required={true}
>
  An object containing the [properties](#properties) definitions to be passed to the function
</APIField>

<APIField
  name="required"
  type="string[]"
>
  Array of required property names from the `properties` object
</APIField>

## Properties

The `properties` object defines the input data that will be passed to the function. It supports different types of parameters, each with their own set of configuration options.
The property name is a key in the `properties` object that is user-defined.

<APIField
  name="[key:string]"
  type="object"
  required={true}
>
  An object with dynamic property names, where:<ul><li>**Keys:** User-defined strings, that set the property name.</li><li>**Values:** Must be one of the valid [schema types](#schema-types). Learn more about valid schema types from the [JSON Schema documentation](https://json-schema.org/understanding-json-schema/reference)</li></ul>
</APIField>

<h3>Array Properties</h3>

<APIField name="type" type="string" required={true}>
  The type of parameter(s) the AI is passing to the function. Must be set to `"array"`
</APIField>

<APIField name="description" type="string">
  A description of the property
</APIField>

<APIField name="items" type="object" required={true}>
  An array of items. These items must be one of the valid [schema types](#schema-types)
</APIField>

<APIField name="default" type="array">
  The default array value
</APIField>

<APIField name="nullable" type="boolean" default="false">
  Whether the property can be null
</APIField>

#### Example

```yaml andJSON
parameters:
  type: object
  properties:
    property_name:
      type: array
      description: Array of strings and objects
      items:
        - type: string
          description: A single string value that must be one of the possible values
          default: "one"
          enum:
            - "one"
            - "two"
            - "three"
        - type: object
          description: An object with a required property, `desired_value`, that must be one of the possible values
          required:
            - desired_value
          properties:
            desired_value:
              type: string
              description: A single string value that must be one of the possible values
              enum:
                - "four"
                - "five"
                - "six"
```
