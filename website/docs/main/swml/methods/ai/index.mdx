Create an AI agent with a prompt. Since the text prompt is central to getting great results out of the AI,
it is highly recommended that you also read the [Prompting Best Practices][Prompting Best Practices] guide.

<APIField
  name="ai"
  type="object"
  required={true}
>
  An object that contains the [`ai parameters`](#ai-parameters).
</APIField>



## **ai Parameters**

<APIField
  name="prompt"
  type="object"
  required={true}
>
  Establishes the initial set of instructions and settings to configure the agent.
</APIField>

<APIField
  name="global_data"
  type="object"
>
  A powerful and flexible environmental variable which can accept arbitrary data that is set initially in the SWML script or from the SWML [`set_global_data` action][set_global_data action]. This data can be referenced **globally**. All contained information can be accessed and expanded within the prompt - for example, by using a template string.
</APIField>

<APIField
  name="hints"
  type="string"
>
  Hints help the AI agent understand certain words or phrases better. Words that can commonly be misinterpreted can be added to the hints to help the AI speak more accurately.
</APIField>

<APIField
  name="languages"
  type="object"
>
  An array of JSON objects defining supported languages in the conversation.
</APIField>

<APIField
  name="params"
  type="object"
>
  A JSON object containing parameters as key-value pairs.
</APIField>

<APIField
  name="post_prompt"
  type="object"
>
  The final set of instructions and configuration settings to send to the agent.
</APIField>

<APIField
  name="post_prompt_url"
  type="string"
>
  The URL to which to send status callbacks and reports. Authentication can also be set in the url in the format of `username:password@url`.
</APIField>

<APIField
  name="pronounce"
  type="object"
>
  An array of JSON objects to clarify the AI's pronunciation of words or expressions.
</APIField>

<APIField
  name="SWAIG"
  type="object"
>
  An array of JSON objects to create user-defined functions/endpoints that can be executed during the dialogue.
</APIField>
