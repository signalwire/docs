  name="play"
  type="object"

>
  An object that accepts the [`play parameters`](#parameters).

## **play Parameters** {#parameters}

<APIField
  name="auto_answer"
  type="boolean"
  default="true"
>
  If `true`, the call will automatically answer as the sound is playing. If `false`, you will start playing the audio during early media, additionally, you can instruct to answer the call with the [`answer method`][answer-method].
</APIField>

<APIField
  name="urls (or url)"
  type="string | string[]"
  required={true}
>
  Either a [playable sound](#playable-sounds), or an array of [playable sounds](#playable-sounds)<br/>Cannot pass both `url` and `urls` at the sametime. Authentication can also be set in the url in the format of `username:password@url`.
</APIField>

<APIField
  name="volume"
  type="number"
  default="0"
>
  Volume gain to apply to played URLs. Allowed values from `-40.0` to `40.0`.
</APIField>

<APIField
  name="say_voice"
  type="string"
  default="Polly.Salli"
>
  Voice to use with `say:` for text to speech.
</APIField>

<APIField
  name="say_language"
  type="string"
  default="en-US"
>
  Language to use with `say:` for text to speech.
</APIField>

<APIField
  name="say_gender"
  type="string"
  default="female"
>
  Gender to use with `say:` for text to speech.
</APIField>

<APIField
  name="status_url"
  type="string"
>
  HTTP or HTTPS URL to deliver play status events.
</APIField>

## **Variables**

Read by the method:

- **`say_voice:`** (in) - Optional voice to use for text to speech.
- **`say_language:`** (in) - Optional language to use for text to speech.
- **`say_gender:`** (in) - Optional gender to use for text to speech.

## **Examples**

### Playing a single URL

```yaml andJson
version: 1.0.0
sections:
  main:
    - play:
        url: 'https://cdn.signalwire.com/swml/audio.mp3'
```


### Playing multiple URLs

```yaml andJson
version: 1.0.0
sections:
  main:
    - play:
        urls:
          - 'https://cdn.signalwire.com/swml/audio.mp3'
          - 'say: this is something to say'
          - 'silence: 3.0'
          - 'ring:10.0:us'
```

### Playing multiple URLs with volume adjusted

```yaml andJson
version: 1.0.0
sections:
  main:
    - play:
        volume: 20
        urls:
          - 'https://cdn.signalwire.com/swml/audio.mp3'
          - 'say: this is something to say'
          - 'silence: 3.0'
          - 'ring:10.0:us'
```

### Specifying a voice to use for speaking

**Globally**

```yaml andJson
version: 1.0.0
sections:
  main:
    - set:
        say_voice: gcloud.en-US-Neural2-A
    - play:
        url: 'say:Hi, do I sound different?'
    - play:
        url: 'say:I don''t, do I?'
```

**For just one instance**

```yaml andJson
version: 1.0.0
sections:
  main:
    - play:
        url: 'say:Hi, do I sound different?'
        say_voice: gcloud.en-US-Neural2-A
    - play:
        url: 'say:I was down with the flu'
```
