# Calls

A call is a connection between SignalWire and another phone. 
**Outbound calls** are made from SignalWire numbers to other phone numbers. 
**Inbound calls** are made from other phone numbers to SignalWire numbers.

|                                                   Attribute |                                                                                                                                     |
| ----------------------------------------------------------: | ----------------------------------------------------------------------------------------------------------------------------------- |
|      `account_sid` <span class="optional-arg">string</span> | The unique identifier for the account that created this call.                                                                       |
|       `annotation` <span class="optional-arg">string</span> | The annotation for the call.                                                                                                        |
|      `answered_by` <span class="optional-arg">string</span> | Who/what the call was answered by. Possible values are `human` or `machine`.                                                        |
|      `api_version` <span class="optional-arg">string</span> | The version of the SignalWire API.                                                                                                  |
|      `caller_name` <span class="optional-arg">string</span> | The name of the caller. Only available if Caller ID lookup is enabled.                                                              |
|   `date_created` <span class="optional-arg">datetime</span> | The date, in RFC 2822 GMT format, this call was created.                                                                            |
|   `date_updated` <span class="optional-arg">datetime</span> | The date, in RFC 2822 GMT format, this call was updated.                                                                            |
|        `direction` <span class="optional-arg">string</span> | The direction of the call. Possible values are `inbound` or `outbound`.                                                             |
|         `duration` <span class="optional-arg">string</span> | The duration, in seconds, of the call.                                                                                              |
|       `end_time` <span class="optional-arg">datetime</span> | The time, in RFC 2822 GMT format, the call was terminated.                                                                          |
|   `forwarded_from` <span class="optional-arg">string</span> | The number this call was forwarded from.                                                                                            |
|             `from` <span class="optional-arg">string</span> | The number, in E.164 format, that initiated the call.                                                                               |
|   `formatted_from` <span class="optional-arg">string</span> | The formatted number that initiated the call.                                                                                       |
|  `parent_call_sid` <span class="optional-arg">string</span> | The unique identifier for the call that created this leg.                                                                           |
| `phone_number_sid` <span class="optional-arg">string</span> | **Outbound call**: the unique identifier for `OutgoingCallerId`. **Inbound call**: the unique identifier for `IncomingPhoneNumber`. |
|           `price` <span class="optional-arg">integer</span> | The charge for the call.                                                                                                            |
|       `price_unit` <span class="optional-arg">string</span> | The currency, in ISO 4127 format, for the price of the call.                                                                        |
|              `sid` <span class="optional-arg">string</span> | The unique identifier for the call.                                                                                                 |
|     `start_time` <span class="optional-arg">datetime</span> | The time, in RFC 2822 GMT format, the call began.                                                                                   |
|           `status` <span class="optional-arg">string</span> | The status of the call. See <a href="#status"><b>below</b></a> for all possible values.                                             |
| `subresource_uris` <span class="optional-arg">object</span> | A Map of available sub-resources.                                                                                                   |
|               `to` <span class="optional-arg">string</span> | The number, in E.164 format, that received the call.                                                                                |
|     `formatted_to` <span class="optional-arg">string</span> | The formatted number that received the call.                                                                                        |
|              `uri` <span class="optional-arg">string</span> | The URI for the call.                                                                                                               |


## `Status`

The `status` attribute has the following values:

|                                            Value |                                                       |
| -----------------------------------------------: | ----------------------------------------------------- |
|      `queued` <span class="optional-arg"></span> | The call is ready and waiting in line.                |
|     `ringing` <span class="optional-arg"></span> | The call is ringing.                                  |
| `in-progress` <span class="optional-arg"></span> | The call was picked up and is in progress.            |
|    `canceled` <span class="optional-arg"></span> | The call was terminated when ringing or queued.       |
|   `completed` <span class="optional-arg"></span> | The call was picked up and terminated with no issues. |
|        `busy` <span class="optional-arg"></span> | The caller received a busy signal.                    |
|      `failed` <span class="optional-arg"></span> | The call was not completed because of a failure.      |

|                                              Parameter |                                                                                                                                                   |

|     `ForwardedFrom` <span class="optional-arg"></span> | The number this call was forwarded from.                                                                                                          |

|      `CallDuration` <span class="optional-arg"></span> | The duration, in seconds, of the call.                                                                                                            |
|      `RecordingUrl` <span class="optional-arg"></span> | The URL of the recorded audio call.                                                                                                               |
|      `RecordingSid` <span class="optional-arg"></span> | The unique identifier for the audio recording.                                                                                                    |
| `RecordingDuration` <span class="optional-arg"></span> | The duration, in seconds of the recording.                                                                                                        |
|         `Timestamp` <span class="optional-arg"></span> | The timestamp, in RFC 2822 format, an event occurred.                                                                                             |
|    `CallbackSource` <span class="optional-arg"></span> | The source of the webhook.                                                                                                                        |

## RecordingStatusCallback Parameters

In addition to the [standard request parameters](/compatibility-api/cxml/voice#request-parameters), the following are parameters passed back to your application when SignalWire makes a request to the `RecordingStatusCallback` URL.

|                                              Parameter |                                                                                                        |
| -----------------------------------------------------: | ------------------------------------------------------------------------------------------------------ |
|      `RecordingSid` <span class="optional-arg"></span> | The unique identifier for the audio recording.                                                         |
|      `RecordingUrl` <span class="optional-arg"></span> | The URL of the recorded audio call.                                                                    |
|   `RecordingStatus` <span class="optional-arg"></span> | The status of the recording. See <a href="#recording-status"><b>below</b></a> for all possible values. |
| `RecordingDuration` <span class="optional-arg"></span> | The duration, in seconds of the recording.                                                             |
| `RecordingChannels` <span class="optional-arg"></span> | The number of channels in the recording. Possible values are `1` and `2`.                              |
|   `RecordingSource` <span class="optional-arg"></span> | The type of call that made the recording.                                                              |
|    `RecordingTrack` <span class="optional-arg"></span> | Which audio tracks are recorded. Can be `inbound`, `outbound`, or `both`.                              |

## `RecordingStatus`

The `RecordingStatus` attribute has the following values:

|                                            Value |                                                                   |
| -----------------------------------------------: | ----------------------------------------------------------------- |
| `in-progress` <span class="optional-arg"></span> | The recording of the call has begun.                              |
|   `completed` <span class="optional-arg"></span> | The recording of the call is completed and ready to access.       |
|      `failed` <span class="optional-arg"></span> | The recording of the call is not accessible because of a failure. |
