# Incoming Phone Numbers

IncomingPhoneNumbers represent an account's phone numbers that were purchased through SignalWire.

## Properties


|                                                          Attribute |                                                                                                                                                                                                                 |
| -----------------------------------------------------------------: | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
|             `account_sid` <span class="optional-arg">string</span> | The unique identifier for the account that is associated with this phone number.                                                                                                                                |
|    `address_requirements` <span class="optional-arg">string</span> | Whether or not a registered address with SignalWire is required. See <a href="#address_requirements"><b>below</b></a> for possible values.                                                                      |
|             `address_sid` <span class="optional-arg">string</span> | The unique identifier for the address associated with this phone number.                                                                                                                                        |
|             `api_version` <span class="optional-arg">string</span> | The version of the SignalWire API.                                                                                                                                                                              |
|                   `beta` <span class="optional-arg">boolean</span> | New numbers on SignalWire are marked as `beta`. Possible values are `true` or `false`.                                                                                                                          |
|            `capabilities` <span class="optional-arg">object</span> | Whether or not a number can receive calls and messages. Possible values are `Voice`, `SMS`, `MMS`, and `Fax`. Each of these values have a boolean value of `true` or `false`.                                   |
|          `date_created` <span class="optional-arg">datetime</span> | The date, in RFC 2822 format, this phone number was created.                                                                                                                                                    |
|          `date_updated` <span class="optional-arg">datetime</span> | The date, in RFC 2822 format, this phone number was updated.                                                                                                                                                    |
|   `emergency_address_sid` <span class="optional-arg">string</span> | The unique identifier of the address associated with E911 for this phone number.                                                                                                                                |
|        `emergency_status` <span class="optional-arg">string</span> | Whether the phone route has an active E911 address associated. Possible values are `Active` or `Inactive`.                                                                                                      |
|           `friendly_name` <span class="optional-arg">string</span> | A formatted version of the number.                                                                                                                                                                              |
|            `identity_sid` <span class="optional-arg">string</span> | The unique identifier for the identity associated with this phone number.                                                                                                                                       |
|                  `origin` <span class="optional-arg">string</span> | The origin of the phone number. SignalWire numbers are denoted as `signalwire` while hosted numbers are denoted as `hosted`.                                                                                    |
|            `phone_number` <span class="optional-arg">string</span> | The incoming number in E.164 format.                                                                                                                                                                            |
|                     `sid` <span class="optional-arg">string</span> | The unique identifier for this phone number.                                                                                                                                                                    |
|     `sms_application_sid` <span class="optional-arg">string</span> | The unique identifier for the application associated with SMS handling on this phone number. If `SmsApplicationSid` is present, the URLs on the application will be used and all other URLs will be ignored.    |
|     `sms_fallback_method` <span class="optional-arg">string</span> | Whether the request to `SmsFallbackUrl` is a `GET` or a `POST`. Default is `POST`.                                                                                                                              |
|        `sms_fallback_url` <span class="optional-arg">string</span> | The URL to request if errors occur when fetching `SmsUrl`.                                                                                                                                                      |
|              `sms_method` <span class="optional-arg">string</span> | Whether the request to `SmsUrl` is a `GET` or a `POST`. Default is `POST`.                                                                                                                                      |
|                 `sms_url` <span class="optional-arg">string</span> | The URL to request when an incoming SMS is received.                                                                                                                                                            |
|         `status_callback` <span class="optional-arg">string</span> | The URL to request to pass status updates to.                                                                                                                                                                   |
|  `status_callback_method` <span class="optional-arg">string</span> | Whether the request to `StatusCallback` is a `GET` or a `POST`. Default is `POST`.                                                                                                                              |
|               `trunk_sid` <span class="optional-arg">string</span> | The unique identifier for the Trunk associated with this phone number.                                                                                                                                          |
|                     `uri` <span class="optional-arg">string</span> | The URI for this number.                                                                                                                                                                                        |
|   `voice_application_sid` <span class="optional-arg">string</span> | The unique identifier for the application associated with call handling on this phone number. If `VoiceApplicationSid` is present, the URLs on the application will be used and all other URLs will be ignored. |
| `voice_caller_id_lookup` <span class="optional-arg">boolean</span> | Whether or not to look up a caller's name in the database. Possible values are `true` or `false`.                                                                                                               |
|   `voice_fallback_method` <span class="optional-arg">string</span> | Whether the request to `VoiceFallbackUrl` is a `GET` or a `POST`. Default is `POST`.                                                                                                                            |
|      `voice_fallback_url` <span class="optional-arg">string</span> | The URL to request if errors occur when fetching `Url`.                                                                                                                                                         |
|            `voice_method` <span class="optional-arg">string</span> | Whether the request to `Url` is a `GET` or a `POST`. Default is `POST`.                                                                                                                                         |
|               `voice_url` <span class="optional-arg">string</span> | The URL to request when an incoming call is received.                                                                                                                                                           |

## `address_requirements`

The `address_requirements` attribute has the following values:

|                                        Value |                                                                       |
| -------------------------------------------: | --------------------------------------------------------------------- |
|    `none` <span class="optional-arg"></span> | No address is required.                                               |
|     `any` <span class="optional-arg"></span> | An address is required, but can be anywhere in the world.             |
|   `local` <span class="optional-arg"></span> | An address is required and must be in the same country as the number. |
| `foreign` <span class="optional-arg"></span> | An address is required and must be outside the country of the number. |
