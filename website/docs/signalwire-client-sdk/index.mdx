# Overview

The SignalWire Client SDK is a JavaScript library that enables WebRTC-based voice, video, and chat applications for web browsers and React Native mobile apps. Built on WebSocket architecture, it provides real-time communication capabilities without additional plugins.

:::info This SDK is meant to replace the [Browser SDK](/sdks/browser-sdk) 

The SignalWire Client SDK should be used in place of the [Browser SDK](/sdks/browser-sdk) for all new projects. The new SDK maintains the API where possible while making architectural changes to support a unified way of interacting with SignalWire's services.

:::


### Create a Guest Token for that address

Guest tokens are a simple way to get access to a specific address.

```bash
curl -L -g 'https://{Your_Space_Name}.signalwire.com/api/fabric/guests/tokens' \
  -u '<Project ID>:<API Token>' \
  -H 'Content-Type: application/json' \
  -H 'Accept: application/json' \
  -d '{"allowed_addresses":["<Address UUID>"]}'
```

This [Create Guest Tokens](/rest/signalwire-rest/endpoints/fabric/guest-tokens-create) endpoint 
will give you a JSON response with a `token` property. To quickly get started, you can 
choose to run the endpoint directly from the 
[Guest Tokens documentation page](/rest/signalwire-rest/endpoints/fabric/guest-tokens-create).

The API credentials can be found in your dashboard. To learn how to find it, 
see [Your Credentials](/platform/dashboard/getting-started/your-signalwire-api-space) page.

### Walkthrough

<AccordionGroup>
<Accordion title="Initialize the SignalWire Client Instance">

After you get the token from the CURL request, you can initialize the client.

```javascript
const client = await SignalWire({
  token: "<token you got from the API>",
});
```

It is recommended to keep a persistent client instance for the lifetime of the application.

</Accordion>
<Accordion title="Dial to an Address">

When the user clicks the `Start Call` button, you can use the `dial` method to dial the call.

```javascript
const callSession = await client.dial({
  to: "<address for a SWML Resource or a phone number or a SIP address>",
  rootElement: $("#callContainer")!,
});
```

The `callSession` object is a [CallSession](/sdks/signalwire-client-sdk/call-session) object 
that you can use to manage the call.

After the call session is created, you can use the `start` method to start the call.

```javascript
callSession.start();
```

</Accordion>

<Accordion title="Listening for Events">

After the call session is created, you can use the `on` method to listen for events.

```javascript
callSession.on("call.joined", () => {
  console.log("Call joined");
  // UI Updates
});
```

</Accordion>

<Accordion title="Hangup the Call">

You can use the `hangup` method to hangup the call.

```javascript
callSession.hangup();
```

</Accordion>

</AccordionGroup>

### Next steps and variations

If you're coming from other providers, you might find our compatibility XML offering [cXML](/compatibility-api/cxml) easier. 
It can be created and dialed to the same as SWML. To get the full feature set of our platform, we still recommend using SWML.

Of course, scripts are not the only things to dial to. You can try dialing phone numbers directly, in e.164 format, or SIP addresses.

Similarly, you can create AI agents either through the dashboard, or directly in SWML.
